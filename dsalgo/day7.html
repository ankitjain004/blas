01Allocate Minimum Number of Pages Hard

Section · Binary Search
GFG
Problem

Books with page counts pages[] must be split contiguously among m students. Each student gets ≥1 book. Minimize the maximum pages any student carries.

Example

Input: pages = [12, 34, 67, 90], m = 2  ·  Output: 113

  1. lo = max = 90, hi = sum = 203.
  2. mid=146: split {12,34,67}=113, {90}=90 → 2 students ≤ 2 ✓ → ans=146, hi=145.
  3. mid=117: same split works → ans=117, hi=116.
  4. mid=103: {12,34}=46, {67}=67, {90}=90 → needs 3 ✗ → lo=104.
  5. mid=110: still needs 3 ✗ → lo=111. mid=113: {12,34,67}=113, {90} → 2 ✓ → ans=113, hi=112. Loop exits → return 113.
Mental Model
Binary search on the answer. The answer lies in [max(pages), sum(pages)]. For a candidate cap, greedily walk the books in order, starting a new student whenever the running sum would exceed cap. Feasible iff students used ≤ m.
Other Approaches
Brute Try every possible cap from max..sum → O((sum-max)·n).
DP partition O(n²·m) — overkill here.
Optimal Approach
  1. lo = max(pages); hi = sum(pages).
  2. While lo <= hi: pick mid; if feasible(mid) → record ans = mid, hi = mid - 1 (try tighter). Else lo = mid + 1.
  3. feasible(cap): walk books, accumulate, start new student when overflow; return studentsUsed ≤ m.

  L -- no --> Z["return ans"]
  L -- yes --> M["mid = (lo+hi)/2"]
  M --> F{"feasible(mid)?"}
  F -- yes --> A["ans = mid; hi = mid - 1"]

Code (Java)
/**
 * Minimum possible "max pages per student" when distributing books contiguously.
 * Trick: binary search on the ANSWER. The function feasible(cap) is monotone — once a
 * cap works, every larger cap also works — so we converge in O(n log(sum-max)).
 */
int allocateMinimumPages(int[] pagesPerBook, int totalStudents){
  if(totalStudents > pagesPerBook.length) return -1;             // can't give everyone a book

  int lo = 0, hi = 0;
  for(int pages : pagesPerBook){ lo = Math.max(lo, pages); hi += pages; }

  int answerCap = hi;                                            // sum always works as a fallback
  while(lo <= hi){
    int candidateCap = lo + (hi - lo) / 2;
    if(canSplitWithinCap(pagesPerBook, totalStudents, candidateCap)){
      answerCap = candidateCap;                                  // cap works → try tighter
      hi = candidateCap - 1;
    } else {
      lo = candidateCap + 1;                                     // cap too small → relax
    }
  }
  return answerCap;
}

private boolean canSplitWithinCap(int[] pages, int allowedStudents, int cap){
  int studentsUsed = 1;
  int runningPages  = 0;
  for(int pageCount : pages){
    if(runningPages + pageCount > cap){
      studentsUsed++;
      runningPages = pageCount;
      if(studentsUsed > allowedStudents) return false;
    } else {
      runningPages += pageCount;
    }
  }
  return true;
}
Time O(n · log(sum))
Space O(1)

02Aggressive Cows Hard

Section · Binary Search
GFG
Problem

Place k cows into stalls at positions stalls[] so the MINIMUM distance between any two cows is MAXIMIZED. Return that maximum-of-minimums.

Example

Input: stalls = [1, 2, 4, 8, 9], cows = 3  ·  Output: 3

  1. Sorted already. lo=1, hi=9-1=8.
  2. mid=4: place at 1; next ≥ 5 → 8; next ≥ 12 → none. Placed 2 < 3 ✗ → hi=3.
  3. mid=2: place at 1, 4, 8 → 3 ✓ → best=2, lo=3.
  4. mid=3: place at 1, 4, 8 → 3 ✓ → best=3, lo=4. Loop exits → return 3.
Mental Model
Binary search on the gap. Sort stalls. For a candidate gap d, greedily place a cow in the first stall, then in each subsequent stall ≥ prev+d. Feasible iff we placed all k cows.
Other Approaches
Brute Try every gap from 1..max → O(n·max).
Optimal Approach
  1. Sort stalls. lo = 1; hi = stalls[n-1] - stalls[0].
  2. Binary search: mid; if canPlace(mid) → lo = mid + 1; record answer = mid. Else hi = mid - 1.
  3. canPlace(d): place 1st cow; walk stalls; when stall - lastPlaced ≥ d → place. Return placed ≥ k.
Visual
flowchart LR
  S["sort stalls; lo=1, hi=max-min"] --> M["mid = candidate gap"]
  M --> F{"can place k cows with gap mid?"}
  F -- yes --> L["best = mid; lo = mid + 1 (try larger)"]
  F -- no --> R["hi = mid - 1 (gap too big)"]
  L --> M

Code (Java)
/**
 * Aggressive Cows — place k cows so the MINIMUM pairwise distance is maximized.
 * Trick: binary search on the gap d. canPlace(d) is monotone-decreasing in d (larger
 * gap → fewer cows fit), so we look for the largest d that still fits all k cows.
 */
int aggressiveCows(int[] stallPositions, int totalCows){
  Arrays.sort(stallPositions);
  int lo = 1, hi = stallPositions[stallPositions.length - 1] - stallPositions[0];
  int bestGap = 0;

  while(lo <= hi){
    int candidateGap = lo + (hi - lo) / 2;
    if(canPlaceAllCowsWithGap(stallPositions, totalCows, candidateGap)){
      bestGap = candidateGap;                                    // gap works → try larger
      lo = candidateGap + 1;
    } else {
      hi = candidateGap - 1;                                     // gap too big
    }
  }
  return bestGap;
}

private boolean canPlaceAllCowsWithGap(int[] stalls, int totalCows, int requiredGap){
  int cowsPlaced     = 1;
  int lastPlacedAt   = stalls[0];
  for(int i = 1; i < stalls.length; i++){
    if(stalls[i] - lastPlacedAt >= requiredGap){
      cowsPlaced++;
      lastPlacedAt = stalls[i];
      if(cowsPlaced >= totalCows) return true;
    }
  }
  return false;
}
Time O(n log(maxGap))
Space O(1)

03Sort a Stack Medium

Section · Stack & Queue
GFG
Problem

Sort a stack in ascending order (largest on top) using only stack operations and recursion / one auxiliary stack.

Example

Input A: stack [3, 1, 4, 2] (bottom → top) · Output: [1, 2, 3, 4] (largest on top)

  1. Pop 2; recursively sort [3, 1, 4] → [1, 3, 4].
  2. insertSorted([1, 3, 4], 2): top 4 > 2 hold; 3 > 2 hold; 1 ≤ 2 push 2; restore 3, 4.
  3. Stack now [1, 2, 3, 4].

Input B (edge): empty stack [] · Output: []

  1. Outer recursion sees empty → returns immediately; no inserts performed.
Mental Model
Pop the top off; recursively sort the rest; then insert the popped value into the now-sorted stack at the right spot. Insertion itself recursively bubbles upward until the value finds its place.
Other Approaches
Auxiliary stack (iterative) Pop into a helper stack while moving larger values back temporarily.
Optimal Approach
  1. sort(stack): if empty → return; x = pop; sort(stack); insertSorted(stack, x).
  2. insertSorted(stack, x): if empty or top ≤ x → push x. Else y = pop; insertSorted(stack, x); push y.
Visual
flowchart TD
  S["sort(stack)"] --> P["pop top x"]
  P --> R["sort(rest of stack)"]
  R --> I["insertSorted(stack, x)"]
  I --> C{"empty or top <= x?"}
  C -- yes --> A["push x"]
  C -- no --> B["pop y; insertSorted(stack, x); push y"]
Code (Java)
/**
 * Sort a stack in place using only the stack itself + recursion (no helper array).
 * Trick: classic "remove-one, recursively-sort-rest, insert-into-sorted" pattern. The
 * insert step itself recurses to bubble the value down to the correct depth.
 */
void sortStack(Deque<Integer> stack){
  if(stack.isEmpty()) return;
  int currentTop = stack.pop();
  sortStack(stack);
  insertInSortedOrder(stack, currentTop);
}

private void insertInSortedOrder(Deque<Integer> stack, int valueToInsert){
  if(stack.isEmpty() || stack.peek() <= valueToInsert){
    stack.push(valueToInsert);
    return;
  }
  int largerValue = stack.pop();                              // hold it aside
  insertInSortedOrder(stack, valueToInsert);                  // recurse on smaller stack
  stack.push(largerValue);                                    // put it back on top
}
Time O(n²)
Space O(n) recursion

04Rotten Oranges (BFS) Medium

Section · Stack & Queue Part-II
LeetCode
Problem

Grid cells: 0 empty, 1 fresh orange, 2 rotten. Every minute, rot spreads 4-directionally to adjacent fresh oranges. Return the minimum minutes to rot ALL fresh oranges, or -1 if some remain unreachable.

Example

Input A: grid = [[2,1,1],[1,1,0],[0,1,1]] · Output: 4

  1. Initial: rotten at (0,0), fresh count = 6.
  2. Min 1: rot (0,1), (1,0); fresh = 4.
  3. Min 2: rot (0,2), (1,1); fresh = 2.
  4. Min 3: rot (2,1); fresh = 1.
  5. Min 4: rot (2,2); fresh = 0 → return 4.

Input B (edge): grid = [[2,1,1],[0,1,1],[1,0,1]] · Output: -1

  1. The lone fresh orange at (2,0) is walled off by 0s → BFS finishes with fresh > 0 → return -1.
Mental Model
Multi-source BFS — every initially rotten orange enters the queue at level 0. Each BFS level = one minute. Track fresh count; if > 0 after BFS → return -1; else return last level reached.
Other Approaches
Iterative scan Pass over grid repeatedly, rotting neighbors until stable → O((mn)²) worst.
Optimal Approach
  1. Queue all initially rotten cells; count fresh.
  2. BFS level-by-level; for each level, rot fresh neighbors, decrement fresh, enqueue, increment minutes.
  3. Return fresh == 0 ? minutes : -1.
Visual
flowchart LR
  S["enqueue all rotten; count fresh"] --> L["BFS level"]
  L --> R["rot 4 neighbors; fresh--"]
  R --> T["minutes++"]
  T --> C{"queue empty?"}
  C -- no --> L
  C -- yes --> A["return fresh==0 ? minutes : -1"]
Code (Java)
/**
 * Minimum minutes for rot to spread to every fresh orange, via multi-source BFS.
 * Trick: every initially rotten cell starts in the BFS frontier at time 0 — so the
 * answer is simply how many BFS levels we run. Track fresh count to detect unreachable.
 */
int orangesRotting(int[][] grid){
  int rows = grid.length, cols = grid[0].length;
  Deque<int[]> bfsQueue = new ArrayDeque<>();
  int freshOranges = 0;

  for(int r = 0; r < rows; r++)
    for(int c = 0; c < cols; c++){
      if(grid[r][c] == 2) bfsQueue.offer(new int[]{ r, c });
      else if(grid[r][c] == 1) freshOranges++;
    }
  if(freshOranges == 0) return 0;

  int minutesElapsed = 0;
  int[][] fourDirections = { {-1,0}, {1,0}, {0,-1}, {0,1} };

  while(!bfsQueue.isEmpty() && freshOranges > 0){
    int levelSize = bfsQueue.size();
    for(int s = 0; s < levelSize; s++){
      int[] cell = bfsQueue.poll();
      for(int[] d : fourDirections){
        int nr = cell[0] + d[0], nc = cell[1] + d[1];
        if(nr < 0 || nc < 0 || nr >= rows || nc >= cols) continue;
        if(grid[nr][nc] != 1) continue;
        grid[nr][nc] = 2;                                       // rot it
        freshOranges--;
        bfsQueue.offer(new int[]{ nr, nc });
      }
    }
    minutesElapsed++;
  }
  return freshOranges == 0 ? minutesElapsed : -1;
}
Time O(m·n)
Space O(m·n)

05Min Stack (O(1) getMin) Medium

Section · Stack & Queue Part-II
LeetCode
Problem

Design a stack supporting push, pop, top, getMin all in O(1).

Example

Input A: ops push 5, push 3, push 7, getMin, pop, getMin · Output: 3, 5

  1. push 5: stack [5], min = 5.
  2. push 3: 3 < min → push encoded 2*3 - 5 = 1; stack [5, 1], min = 3.
  3. push 7: 7 ≥ min → push 7; stack [5, 1, 7], min = 3.
  4. getMin → 3.
  5. pop: top 7 ≥ min → min unchanged; stack [5, 1].
  6. pop would pop encoded 1: 1 < min → restore min = 2*3 - 1 = 5; getMin → 5.

Input B (edge): push Integer.MIN_VALUE, getMin · Output: -2147483648

  1. First push: stack stores it as long, min = MIN_VALUE; encoding never triggers (no prior min). Long internals avoid the 2x - min overflow.
Mental Model
Trick: store an ENCODED value when a new minimum arrives. Push 2·x - oldMin (which is < x). On pop, if top < currentMin → currentMin was x, restore old min = 2·currentMin - top. Uses O(1) extra space beyond the stack itself.
Other Approaches
Auxiliary stack Parallel stack storing the running min — O(n) extra space.
Pair-stored single stack Each entry = {value, minAfterThisPush}. Easier to read than the encoding trick and avoids overflow concerns entirely, at the cost of 2x memory per element. 
class MinStack {
  private final Deque<int[]> stack = new ArrayDeque<>();   // {value, minAfter}
  public void push(int x){
    int minAfter = stack.isEmpty() ? x : Math.min(x, stack.peek()[1]);
    stack.push(new int[]{ x, minAfter });
  }
  public void pop()    { stack.pop(); }
  public int  top()    { return stack.peek()[0]; }
  public int  getMin() { return stack.peek()[1]; }
}
Overflow pitfall ⚠ The encoded value 2x - min can overflow int when x = Integer.MIN_VALUE or when min is large positive. Always store stack values as long internally and cast back to int at the API boundary.
Optimal Approach
  1. push(x): if empty → push x, min = x. Else if x ≥ min → push x. Else push 2x - min; min = x.
  2. pop(): t = pop. If t < min → min = 2·min - t.
  3. top(): t = peek. Return t < min ? min : t.
  4. getMin(): return min.
Visual
flowchart TD
  P["push(x)"] --> C{"x < min?"}
  C -- yes --> E["push (2x - min); min = x"]
  C -- no --> A["push x"]
  Po["pop()"] --> T["t = pop"]
  T --> F{"t < min?"}
  F -- yes --> M["min = 2*min - t (restore previous min)"]
  F -- no --> N["min unchanged"]
Code (Java)
/**
 * MinStack with O(1) getMin and O(1) extra space using an encoding trick.
 * Trick: when a new minimum x arrives, push (2x - oldMin) — guaranteed < x and acts as
 * a "flag" on pop telling us to restore the previous min via 2*min - top.
 * NOTE: use long internally to avoid overflow on signed-int edge cases.
 */
class MinStack {
  private Deque<Long> stack = new ArrayDeque<>();
  private long        runningMin;

  public void push(int incomingValue){
    long x = incomingValue;
    if(stack.isEmpty()){
      stack.push(x);
      runningMin = x;
    } else if(x >= runningMin){
      stack.push(x);
    } else {
      stack.push(2 * x - runningMin);                          // encoded flag value
      runningMin = x;
    }
  }

  public void pop(){
    long top = stack.pop();
    if(top < runningMin) runningMin = 2 * runningMin - top;    // restore previous min
  }

  public int top(){
    long top = stack.peek();
    return (int) (top < runningMin ? runningMin : top);
  }

  public int getMin(){ return (int) runningMin; }
}
Time O(1) per op
Space O(n)

06Maximum of Minimums of Every Window Size Hard

Section · Stack & Queue Part-II
GFG
Problem

For every window size k = 1..n, return the maximum over all windows of size k of the window's MINIMUM.

Example

Input A: a = [10, 20, 30, 50, 10, 70, 30] · Output: [70, 70, 20, 10, 10, 10, 10]

  1. NSE/PSE give widest-window length per index (see Worked Example below).
  2. Bucket by len: 1→50, 2→70, 3→20, 4→10, 7→10.
  3. Back-fill right→left: ans[6]=10, ans[5]=10, ans[4]=10, ans[3]=20, ans[2]=70, ans[1]=70.

Input B (edge): a = [3, 1, 2] · Output: [3, 2, 1]

  1. a[0]=3: len=1 → bucket[1]=3. a[1]=1: len=3 → bucket[3]=1. a[2]=2: len=1 → bucket[1]=max(3,2)=3.
  2. After back-fill: ans = [3, 2, 1].
Mental Model
For each a[i], find the LARGEST window in which it is the minimum. That window length is len = nextSmaller[i] - prevSmaller[i] - 1. Then answer[len] = max(answer[len], a[i]). Finally back-fill: answer[len-1] = max(answer[len-1], answer[len]) (a min over length L also dominates L-1).
Other Approaches
Brute All windows of all sizes → O(n³) or O(n² log n).
Optimal Approach
  1. Monotonic stack: prevSmaller[i], nextSmaller[i] (strict-less indices, -1 / n if none).
  2. For each i: len = nextSmaller[i] - prevSmaller[i] - 1; answer[len] = max(answer[len], a[i]).
  3. Back-fill answer[i] = max(answer[i], answer[i+1]) from right to left.
Visual
flowchart TD
  E["for each a[i]"] --> P["len = nextSmaller[i] - prevSmaller[i] - 1"]
  P --> A["answer[len] = max(answer[len], a[i])"]
  A --> B["back-fill answer right to left"]
Code (Java)
/**
 * For each window size k in 1..n, return max over windows of (window minimum).
 * Trick: instead of iterating window sizes, ask "what is the WIDEST window in which
 * a[i] is the minimum?" — len = nextSmaller[i] - prevSmaller[i] - 1. Each value owns
 * exactly one window length and contributes to answer[len].
 */
int[] maxOfMinForEveryWindowSize(int[] numbers){
  int n = numbers.length;
  int[] prevSmaller = new int[n];
  int[] nextSmaller = new int[n];
  Deque<Integer> monotonicStack = new ArrayDeque<>();

  for(int i = 0; i < n; i++){
    while(!monotonicStack.isEmpty() && numbers[monotonicStack.peek()] >= numbers[i]) monotonicStack.pop();
    prevSmaller[i] = monotonicStack.isEmpty() ? -1 : monotonicStack.peek();
    monotonicStack.push(i);
  }
  monotonicStack.clear();
  for(int i = n - 1; i >= 0; i--){
    while(!monotonicStack.isEmpty() && numbers[monotonicStack.peek()] >= numbers[i]) monotonicStack.pop();
    nextSmaller[i] = monotonicStack.isEmpty() ? n : monotonicStack.peek();
    monotonicStack.push(i);
  }

  int[] answerByWindowSize = new int[n + 1];                  // 1-indexed
  for(int i = 0; i < n; i++){
    int widestWindowLen = nextSmaller[i] - prevSmaller[i] - 1;
    answerByWindowSize[widestWindowLen] = Math.max(answerByWindowSize[widestWindowLen], numbers[i]);
  }
  for(int k = n - 1; k >= 1; k--){                           // back-fill: longer also fills shorter
    answerByWindowSize[k] = Math.max(answerByWindowSize[k], answerByWindowSize[k + 1]);
  }
  return Arrays.copyOfRange(answerByWindowSize, 1, n + 1);
}
Time O(n)
Space O(n)
Worked Example — why answer[len] = max(answer[len], a[i])

For each index i, len = nextSmaller[i] − prevSmaller[i] − 1 is the widest window in which a[i] is the strict minimum (everything strictly between the two boundaries is ≥ a[i]). So inside that window the minimum is exactly a[i], making a[i] one candidate for "best minimum achievable in a window of size len". We take max because multiple indices may all produce the same len; the largest such value wins.

Take a = [10, 20, 30, 50, 10, 70, 30], n = 7.

i   a[i]   prevSmaller   nextSmaller   len = ns − ps − 1
0   10        −1              4              4
1   20         0              4              3
2   30         1              4              2
3   50         2              4              1
4   10        −1              7              7
5   70         4              7              2
6   30         4              7              2

Apply answer[len] = max(answer[len], a[i]) (1-indexed buckets):

  • len=1: candidate a[3]=50 → answer[1]=50
  • len=2: candidates a[2]=30, a[5]=70, a[6]=30 → answer[2]=70
  • len=3: candidate a[1]=20 → answer[3]=20
  • len=4: candidate a[0]=10 → answer[4]=10
  • len=7: candidate a[4]=10 → answer[7]=10
  • len=5, 6: empty so far → 0

Back-fill answer[k] = max(answer[k], answer[k+1]) from right to left:

k=6: max(0, 10)  = 10
k=5: max(0, 10)  = 10
k=4: max(10, 10) = 10
k=3: max(20, 10) = 20
k=2: max(70, 20) = 70
k=1: max(50, 70) = 70

final answer (window sizes 1..7) = [70, 70, 20, 10, 10, 10, 10]

Why this is correct: any window W of size k has some min a[m]; the widest "dominance range" of m has length L_m ≥ k, so a[m] is bucketed into answer[L_m] and the back-fill propagates it down to answer[k]. Hence every legitimate window-minimum is considered, and max picks the strongest.

NSE + PSE on a monotonic stack gives each element its “dominance range” — the widest window in which it is the min (or max). Once you have len[i] = nextSmaller[i] - prevSmaller[i] - 1, the same template solves a whole family:

07The Celebrity Problem Medium

Section · Stack & Queue Part-II
GFG
Problem

In a party of n people, a celebrity is known by everyone but knows no one. Using knows(a, b), find the celebrity (or -1) in O(n) calls.

Example

Input A: n=3, knows = [[0,1,0],[0,0,0],[0,1,0]] · Output: 1

  1. l=0, r=2. knows(0,2)=0 → r--, r=1.
  2. l=0, r=1. knows(0,1)=1 → l++, l=1. Loop ends, candidate=1.
  3. Verify: !knows(1, 0) & knows(0, 1) ✓; !knows(1, 2) & knows(2, 1) ✓ → return 1.

Input B (edge): n=2, knows = [[0,1],[1,0]] (mutual) · Output: -1

  1. Two-pointer narrows to candidate 1, but knows(1, 0) = 1 → candidate also knows someone → return -1.
Mental Model
Two-pointer elimination. Compare two people; knows(a, b) rules a out; !knows(a, b) rules b out. n-1 calls converges to one candidate; verify with one more sweep.
Other Approaches
Stack Push everyone, pop two at a time, push the survivor. Same complexity.
Brute Check every person against every other → O(n²).
Optimal Approach
  1. l = 0, r = n-1. While l < r: if knows(l, r) → l++; else r--.
  2. candidate = l. Verify: for every i ≠ candidate, must have !knows(candidate, i) AND knows(i, candidate).
Visual
flowchart LR
  S["l = 0, r = n-1"] --> C{"knows(l, r)?"}
  C -- yes --> A["l++ (l is not the celeb)"]
  C -- no --> B["r-- (r is not the celeb)"]
  A --> S
  B --> S
  S --> V["verify single candidate via full sweep"]
Code (Java)
/**
 * Find the celebrity (known by everyone, knows no one) in O(n) knows() calls.
 * Trick: each knows(a, b) call eliminates exactly ONE of {a, b} from celebrity-hood.
 * Linear two-pointer elimination converges in n-1 calls; one verification sweep
 * confirms (or returns -1).
 */
int findCelebrity(int n){
  int leftPointer = 0, rightPointer = n - 1;
  while(leftPointer < rightPointer){
    if(knows(leftPointer, rightPointer)) leftPointer++;        // l knows someone → l isn't celeb
    else                                  rightPointer--;       // l doesn't know r → r isn't celeb
  }
  int candidate = leftPointer;

  for(int person = 0; person < n; person++){

 * position with 'a'..'z' and probing the HashSet — O(L * 26) per word.

int ladderLength(String beginWord, String endWord, List<String> wordList){

  dictionary.remove(beginWord);

      char[] current = bfsQueue.poll().toCharArray();

    }
    stepsFromBegin++;

/**
 * Topological order of a DAG via Kahn's BFS algorithm.
 * Trick: the pop-order from Kahn IS a valid topological order. Same code as cycle
 * detection — we just record the order instead of counting.
 */
int[] topologicalSort(int vertexCount, List<List<Integer>> adjacency){
  int[] indegree = new int[vertexCount];
  for(int u = 0; u < vertexCount; u++)
    for(int v : adjacency.get(u)) indegree[v]++;

  Deque<Integer> queue = new ArrayDeque<>();
  for(int v = 0; v < vertexCount; v++) if(indegree[v] == 0) queue.offer(v);

  int[] order = new int[vertexCount];
  int   idx   = 0;
  while(!queue.isEmpty()){
    int u = queue.poll();
    order[idx++] = u;
    for(int v : adjacency.get(u))
      if(--indegree[v] == 0) queue.offer(v);
  }
  if(idx < vertexCount) throw new IllegalStateException("graph has a cycle");
  return order;
}

/**
 * Weighted Job Scheduling — pick non-overlapping jobs to maximize profit.
 * Trick: sort by end time + binary search. dp[i] = best using jobs[0..i]. For job i,
 * either skip it (dp[i-1]) or take it and add dp[k] where k is the last job that

    if(a[mid] <= target){ res = mid; lo = mid + 1; }
    else                  hi = mid - 1;
  }
  return res;
}

  for(int i = 0; i < q; i++) order[i] = i;
  Arrays.sort(order, (a, b) -> queries[a][1] - queries[b][1]);

  BitTrieNode root = new BitTrieNode();
  int[] answer = new int[q];
  int   numsCursor = 0;

  for(int idx : order){
    int x = queries[idx][0], m = queries[idx][1];
    while(numsCursor < nums.length && nums[numsCursor] <= m){
      insertIntoBitTrie(root, nums[numsCursor++]);
    }
    answer[idx] = root.any ? queryMaxXor(root, x) : -1;
  }
  return answer;
}

private void insertIntoBitTrie(BitTrieNode root, int value){
  BitTrieNode cur = root;
  for(int bit = 30; bit >= 0; bit--){
    int b = (value >> bit) & 1;
    if(cur.child[b] == null) cur.child[b] = new BitTrieNode();
    cur = cur.child[b];
  }
  root.any = true;
}

private int queryMaxXor(BitTrieNode root, int x){
  BitTrieNode cur = root;
  int xorResult = 0;
  for(int bit = 30; bit >= 0; bit--){
    int wantBit = 1 - ((x >> bit) & 1);
    if(cur.child[wantBit] != null){
      xorResult |= (1 << bit);
      cur = cur.child[wantBit];
    } else {
      cur = cur.child[1 - wantBit];
    }
  }
  return xorResult;
}
