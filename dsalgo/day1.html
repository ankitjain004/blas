01Set Matrix Zeroes Medium

Section · Arrays
LeetCode Article YouTube
Problem

Given an m × n matrix, if any element is 0, set its entire row and column to 0. Do it in-place with O(1) extra space.

Mental Model
Reuse the matrix as your memory. The first row & first column become "flag arrays" telling which rows/cols must be zeroed. Use one extra boolean for the first column itself to break ambiguity at matrix[0][0].
Other Approaches
Brute Mark every zero cell's row & column with a sentinel (like -1), then convert sentinels to 0. — O((R·C)·(R+C)).
Better Two boolean arrays row[R] and col[C]: mark in pass-1, zero in pass-2. — O(R·C) time, O(R+C) space.
Optimal Approach
  1. Use row[0] and col[0] as markers.
  2. Track col0=1 separately for the first column.
  3. Sweep cells (1..n-1, 1..m-1): if zero, mark matrix[i][0]=0 and matrix[0][j]=0.
  4. Second pass (inside-out): zero cells whose row/col marker is 0.
  5. Finally handle matrix[0][0] via row[0] flag and col0.
Visual
flowchart LR
  A["Scan matrix"] --> B{"cell == 0?"}
  B -- yes --> C["Mark row i 0 = 0
Mark col 0 j = 0"]
  B -- no --> A
  C --> A
  A --> D["Second pass i,j ≥ 1"]
  D --> E{"row marker 0
or col marker 0?"}
  E -- yes --> F["Set cell = 0"]
  E -- no --> D
  F --> G["Handle row 0 & col 0 last"]
Code (Java)
/**
 * Sets every entire row and column to 0 if any cell in that row/column is 0.
 * Uses the FIRST row and FIRST column themselves as O(1)-space markers.
 * Column 0 needs a separate boolean because it shares cell (0,0) with row 0.
 */
void setZeroes(int[][] matrix){
  int rowCount = matrix.length;
  int colCount = matrix[0].length;
  boolean firstColumnHasZero = false;

  // ---- Pass 1: scan & mark zeros into the first row / first column ----
  for(int row = 0; row < rowCount; row++){
    if(matrix[row][0] == 0) firstColumnHasZero = true;        // remember separately
    for(int col = 1; col < colCount; col++){
      if(matrix[row][col] == 0){
        matrix[row][0] = 0;                                   // mark the row
        matrix[0][col] = 0;                                   // mark the column
      }
    }
  }

  // ---- Pass 2: zero the cells, processing bottom-up so we don't overwrite markers too early ----
  for(int row = rowCount - 1; row >= 0; row--){
    for(int col = colCount - 1; col >= 1; col--){             // skip column 0 here
      if(matrix[row][0] == 0 || matrix[0][col] == 0) matrix[row][col] = 0;
    }
    if(firstColumnHasZero) matrix[row][0] = 0;                // handle column 0 last
  }
}
Time O(R·C)
Space O(1)

02Rotate Matrix by 90° Medium

Section · Arrays Part-II
LeetCode
Problem

Rotate an n × n matrix clockwise by 90° in-place.

Mental Model
Transpose, then reverse each row. A 90° clockwise rotation is equivalent to flipping over the main diagonal, then mirroring left-right.
Other Approaches
Brute Allocate a new matrix and write R[i][j] = M[n-1-j][i]; copy back. — O(n²) time, O(n²) extra space.
Optimal Approach
  1. Transpose: swap m[i][j] with m[j][i] for j > i.
  2. Reverse each row in place.
Visual
flowchart LR
  M[Original Matrix] --> T[Transpose
swap m i j with m j i]
  T --> R[Reverse each row]
  R --> O[Rotated 90° CW]
Code (Java)
/**
 * Rotates an n x n matrix 90 degrees clockwise, in place.
 * Trick: a 90-CW rotation == transpose along the main diagonal, then reverse each row.
 */
void rotate(int[][] matrix){
  int size = matrix.length;

  // ---- 1) Transpose: swap matrix[row][col] with matrix[col][row] for col > row ----
  for(int row = 0; row < size; row++){
    for(int col = row + 1; col < size; col++){
      int swapTemp = matrix[row][col];
      matrix[row][col] = matrix[col][row];
      matrix[col][row] = swapTemp;
    }
  }

  // ---- 2) Reverse each row (mirror left-right) ----
  for(int row = 0; row < size; row++){
    int leftIdx = 0, rightIdx = size - 1;
    while(leftIdx < rightIdx){
      int swapTemp = matrix[row][leftIdx];
      matrix[row][leftIdx]  = matrix[row][rightIdx];
      matrix[row][rightIdx] = swapTemp;
      leftIdx++;
      rightIdx--;
    }
  }
}
Time O(n²)
Space O(1)

03Search in a 2D Matrix Medium

Section · Arrays Part-III
LeetCode
Problem

Each row sorted, first integer of each row > last integer of previous row. Search for target.

Mental Model
Treat the 2D matrix as a flattened sorted array. Index k maps to (k/C, k%C). Now it's pure binary search.
Other Approaches
Brute Linear scan of all R·C cells. — O(R·C).
Better Binary search the first column to pick the right row, then binary search inside that row. — O(log R + log C).
Optimal Approach
  1. lo=0, hi=R*C-1.
  2. While lo ≤ hi: mid=(lo+hi)/2; element = m[mid/C][mid%C].
  3. Compare and shrink the window.
Visual
flowchart TD
  S["lo=0, hi=R*C-1"] --> L{"lo <= hi?"}
  L -- no --> N["Return false"]
  L -- yes --> M["mid = (lo+hi)/2
val = m mid/C  mid%C"]
  M --> C{"val == target?"}
  C -- yes --> Y["Return true"]
  C -- "val < target" --> R["lo = mid+1"]
  C -- "val > target" --> H["hi = mid-1"]
  R --> L
  H --> L
Code (Java)
/**
 * Searches for `target` in a row-sorted matrix where each row's first element
 * is greater than the previous row's last element. Treats the matrix as a single
 * sorted array of size rows*cols and binary-searches it.
 */
boolean searchInSortedMatrix(int[][] matrix, int target){
  int rowCount = matrix.length;
  int colCount = matrix[0].length;

  int low  = 0;
  int high = rowCount * colCount - 1;

  while(low <= high){
    int midIndex = (low + high) >>> 1;                       // unsigned shift = avoids overflow
    int midRow   = midIndex / colCount;                      // map 1-D index back to 2-D coords
    int midCol   = midIndex % colCount;
    int midValue = matrix[midRow][midCol];

    if(midValue == target)      return true;
    else if(midValue < target)  low  = midIndex + 1;
    else                        high = midIndex - 1;
  }
  return false;
}
Time O(log(R·C))
Space O(1)

04Two Sum Easy

Section · Arrays Part-IV
LeetCode
Problem

Return indices i, j such that a[i] + a[j] == target.

Mental Model
"Have I seen the complement?" While walking the array, ask the hash map if target − current already passed by. Trade memory for one linear scan.
Other Approaches
Brute Two nested loops checking every pair. — O(n²) time, O(1) space.
Better Sort + two pointers (only when returning values, not indices). — O(n log n) time, O(1) space.
Optimal Approach
  1. Single pass with HashMap of value → index.
  2. For each a[i]: if map.contains(target − a[i]) return pair.
  3. Else put a[i] → i and continue.
Visual
flowchart LR
  A["i = 0"] --> B{"map has
target - a i ?"}
  B -- yes --> R["Return map.get , i"]
  B -- no --> P["map.put a i -> i"]
  P --> N["i++"]
  N --> B
Code (Java)
/**
 * Returns the indices of two numbers in `nums` that add up to `target`.
 * Walks the array once, asking the hash map: "have I already seen the complement?"
 */
int[] twoSum(int[] nums, int target){
  Map<Integer, Integer> valueToIndex = new HashMap<>();

  for(int currentIndex = 0; currentIndex < nums.length; currentIndex++){
    int currentValue = nums[currentIndex];
    int complement   = target - currentValue;             // what we still need

    if(valueToIndex.containsKey(complement)){
      int complementIndex = valueToIndex.get(complement);
      return new int[]{ complementIndex, currentIndex };
    }
    valueToIndex.put(currentValue, currentIndex);         // remember for future lookups
  }
  return new int[]{ -1, -1 };                             // no pair found
}
Time O(n)
Space O(n)

05Reverse a Linked List Easy

Section · Linked List
LeetCode
Problem

Given head of a singly linked list, return the reversed list.

Mental Model
Three pointers, one snip at a time. prev is the new tail-side, curr is the worker, next is your safety net before flipping the arrow.
Other Approaches
Brute Push all node values onto a stack, then re-assign by popping. — O(n) time, O(n) space.
Better Recursive flip (shown below as the second code block). — O(n) time, O(n) stack.
Optimal Approach
  1. Init prev=null, curr=head.
  2. Loop: save next=curr.next; flip curr.next=prev; advance prev=curr; curr=next.
  3. Return prev.
Visual
flowchart LR
  N1[1] --> N2[2] --> N3[3] --> N4[null]
  R4[null] --> R3[1] 
  R5[2] --> R3
  R6[3] --> R5
Code · Iterative (Java)
/**
 * Reverses a singly-linked list iteratively.
 * Maintains 3 pointers: `previousNode` (already reversed part),
 * `currentNode` (the one being flipped), `nextNode` (safety save before we lose it).
 */
ListNode reverseListIterative(ListNode head){
  ListNode previousNode = null;
  ListNode currentNode  = head;

  while(currentNode != null){
    ListNode nextNode = currentNode.next;   // save next BEFORE we overwrite the pointer
    currentNode.next  = previousNode;       // flip the arrow backwards
    previousNode      = currentNode;        // shift the "reversed-so-far" tail forward
    currentNode       = nextNode;           // advance the worker
  }
  return previousNode;                       // previousNode ends up as the new head
}
Time O(n)
Space O(1)
Code · Recursive (Java)
Recursive intuition: trust recursion to reverse head.next onwards; when it returns, the node right after head is the new tail — point it back to head, then sever head.next.
/**
 * Reverses a linked list recursively.
 * Trust the recursion to reverse everything from `head.next` onward. Once it returns,
 * the node directly after `head` is now the last node of that reversed sub-list -- so
 * make IT point back to `head`, and `head` itself becomes the new tail (next = null).
 */
ListNode reverseListRecursive(ListNode head){
  // Base case: empty list or single node is already "reversed".
  if(head == null || head.next == null) return head;

  ListNode newHead = reverseListRecursive(head.next);  // recursion handles the rest

  head.next.next = head;   // the node that USED to be after head now points BACK at head
  head.next      = null;   // head is now the tail of the reversed list
  return newHead;
}
Time O(n)
Space O(n) stack

06Intersection of Two Linked Lists Easy

Section · Linked List Part-II
LeetCode
Problem

Find the node where two singly-linked lists merge (or null).

Mental Model
Equalize the runways. Two pointers; when one hits the end, jump to the OTHER list's head. After at most m+n steps they either meet at the intersection or both reach null together.
Other Approaches
Brute For every node in A, scan all of B for equality. — O(m·n).
Better Put all A nodes in a hash set; walk B. — O(m+n) time, O(m) space. Or: compute lengths, advance the longer pointer by |m-n|, then walk together. — O(m+n) time, O(1) space.
Optimal Approach
  1. a = headA, b = headB.
  2. Loop until a == b: if a==null set a=headB else a=a.next; same for b with headA.
  3. Return a.
Visual
flowchart LR
  A[a walks A then B] --> X((Meet at intersection))
  B[b walks B then A] --> X
Code (Java)
/**
 * Returns the node where two singly-linked lists meet, or null if they never do.
 * Idea: each pointer walks its own list, then jumps to the OTHER list's head.
 * After at most (lenA + lenB) steps the two pointers must align -- either at the
 * intersection node, or both at null simultaneously.
 */
ListNode getIntersectionNode(ListNode headA, ListNode headB){
  if(headA == null || headB == null) return null;

  ListNode walkerA = headA;
  ListNode walkerB = headB;

  // Loop terminates because both walkers traverse exactly lenA + lenB nodes total.
  while(walkerA != walkerB){
    walkerA = (walkerA == null) ? headB : walkerA.next;   // hop onto list B at the end
    walkerB = (walkerB == null) ? headA : walkerB.next;   // hop onto list A at the end
  }
  return walkerA;   // intersection node, or null if no intersection
}
Time O(m+n)
Space O(1)

07Rotate a Linked List by k Medium

Section · Linked List & Arrays
LeetCode
Problem

Rotate a linked list to the right by k places.

Mental Model
Make it a circle, then cut. Connect tail to head, walk to the new tail at len − k%len − 1, and break the circle there.
Other Approaches
Brute Rotate by 1 in a loop, k times. — O(n·k).
Better Same idea but reduce k = k % L first. — O(n + k%L).
Optimal Approach
  1. Compute length L; connect tail to head (cycle).
  2. k %= L; new tail at index L - k - 1.
  3. New head = newTail.next; newTail.next = null.
Visual
flowchart LR
  H[head] --> N1 --> N2 --> N3 --> T[tail]
  T --> H
  Cut[Cut at L - k - 1] --> NewH[New head & tail]
Code (Java)
/**
 * Rotates a singly-linked list to the RIGHT by k positions.
 * Strategy: turn the list into a circle, find the new tail, then break the circle.
 */
ListNode rotateRight(ListNode head, int k){
  if(head == null || head.next == null || k == 0) return head;

  // ---- 1) Walk to the tail and count length along the way ----
  int listLength = 1;
  ListNode oldTail = head;
  while(oldTail.next != null){
    oldTail = oldTail.next;
    listLength++;
  }

  // ---- 2) Close the loop into a circle ----
  oldTail.next = head;

  // ---- 3) The new tail is at position (length - k%length - 1) from the original head ----
  int rotateBy        = k % listLength;
  int stepsToNewTail  = listLength - rotateBy - 1;
  ListNode newTail    = head;
  while(stepsToNewTail-- > 0) newTail = newTail.next;

  // ---- 4) The node after the new tail is the new head; cut the circle there ----
  ListNode newHead = newTail.next;
  newTail.next     = null;
  return newHead;
}
Time O(n)
Space O(1)

08N Meetings in One Room Easy

Section · Greedy Algorithm
GFG
Problem

Maximum number of non-overlapping meetings, given start[] and end[].

Mental Model
Always pick the meeting that ends first. Earliest finish time leaves the most room for the rest. Classic exchange-argument greedy.
Other Approaches
Brute Try every subset, keep the largest non-overlapping one. — O(2ⁿ).
Better Sorting by start time fails (counter-examples exist) — only end-time greedy is correct.
Optimal Approach
  1. Pair (start, end, idx); sort by end ascending.
  2. Track lastEnd. If start > lastEnd, take it; update lastEnd.
Visual
flowchart TD
  S[Sort meetings by end time] --> L[Iterate sorted]
  L --> C{start > lastEnd?}
  C -- yes --> P[Pick · update lastEnd · count++]
  C -- no --> K[Skip]
  P --> L
  K --> L
Code (Java)
/**
 * Returns the maximum number of non-overlapping meetings that fit in one room.
 * Greedy proof: among all valid choices for the FIRST meeting, picking the one
 * that ends earliest leaves the largest possible suffix for the rest -- so always
 * take the meeting that finishes soonest.
 */
int maxMeetings(int[] startTimes, int[] endTimes){
  int meetingCount = startTimes.length;

  // We sort INDICES (not times) so we can still look up matching start/end pairs.
  Integer[] meetingsByEndTime = new Integer[meetingCount];
  for(int i = 0; i < meetingCount; i++) meetingsByEndTime[i] = i;
  Arrays.sort(meetingsByEndTime, (a, b) -> endTimes[a] - endTimes[b]);

  // The earliest-ending meeting is always part of an optimal answer.
  int meetingsHeld = 1;
  int lastEndTime  = endTimes[meetingsByEndTime[0]];

  for(int i = 1; i < meetingCount; i++){
    int meetingIdx = meetingsByEndTime[i];
    if(startTimes[meetingIdx] > lastEndTime){               // strictly after previous end
      meetingsHeld++;
      lastEndTime = endTimes[meetingIdx];
    }
  }
  return meetingsHeld;
}
Time O(n log n)
Space O(n)
Deep Dive · How the Comparator Lambda Actually Works
Parameter names are just labels. The sort algorithm always hands your lambda two elements in a fixed order — call them slot 1 (first arg) and slot 2 (second arg). What controls ascending vs descending is which slot ends up as the first argument to Integer.compare(...), not what you named them.
The Contract

The sort calls your comparator and reads only the sign of the returned number:

  • negative → "the FIRST argument I gave you should come first"
  • positive → "the SECOND argument I gave you should come first"
  • zero → equal, doesn't matter

Integer.compare(x, y) is conceptually just x - y:

  • Integer.compare(3, 7) → -4 (negative)
  • Integer.compare(7, 3) → +4 (positive)
Why Swapping Inputs Flips The Order

Suppose sort hands your lambda 3 (slot 1) and 7 (slot 2):

Case A (a,b) -> Integer.compare(a, b) → returns -4 → sort keeps 3 before 7 → ascending
Case B (a,b) -> Integer.compare(b, a) → returns +4 → sort puts 7 before 3 → descending

Sort doesn't know you swapped anything. It only knows it gave you (3, 7) and you returned +4, so it dutifully swaps the pair. Computing the comparison of the wrong pair from sort's perspective inverts the sign of every answer → reverses the whole order.

All Four Variations Side-by-Side
Lambda Slot 1 named First arg to compare Result
(a, b) -> compare(a[0], b[0])aslot 1ascending
(a, b) -> compare(b[0], a[0])aslot 2descending
(b, a) -> compare(a[0], b[0])bslot 2descending
(b, a) -> compare(b[0], a[0])bslot 1ascending
Mental shortcut. Ignore the names entirely. Compare the order of arguments inside compare(...) against the order in the lambda's parameter list:
  • Same order → ascending
  • Flipped order → descending

09Subset Sums Easy

Section · Recursion
GFG
Problem

Return sums of all 2ⁿ subsets of an array.

Mental Model
At each index: pick or not-pick. A binary recursion tree of depth n generates every subset exactly once. Carry the running sum down.
Other Approaches
Brute Bitmask iterate 0..2ⁿ-1; for each mask, sum the chosen elements. — O(2ⁿ · n).
Optimal Approach
  1. DFS(i, sum): if i==n add sum.
  2. Recurse with sum + a[i] (pick) and with sum (not pick).
Visual
flowchart TD
  R["i=0 sum=0"] --> P["pick a 0"]
  R --> NP["skip a 0"]
  P --> P1["pick a 1"]
  P --> P2["skip a 1"]
  NP --> P3["pick a 1"]
  NP --> P4["skip a 1"]
Code (Java)
/**
 * Generates the sums of every one of the 2^n subsets of `nums`.
 * Pure pick / skip recursion: at each index decide whether to include `nums[index]`.
 */
void collectSubsetSums(int index, int runningSum, int[] nums, List<Integer> allSums){
  if(index == nums.length){             // leaf: every element decided
    allSums.add(runningSum);
    return;
  }
  // Branch 1: PICK nums[index]  -- add it to the running sum
  collectSubsetSums(index + 1, runningSum + nums[index], nums, allSums);
  // Branch 2: SKIP nums[index]  -- leave the running sum unchanged
  collectSubsetSums(index + 1, runningSum,               nums, allSums);
}
Time O(2ⁿ)
Space O(n) recursion

10All Permutations of an Array Medium

Section · Recursion & Backtracking
LeetCode
Problem

Return all n! permutations of distinct integers.

Mental Model
Swap-in-place backtracking. Fix position i: try every j ≥ i in that slot, recurse, then swap back. No extra used[] array needed.
Other Approaches
Brute Generate all nⁿ sequences and filter unique permutations. — wildly wasteful.
Better Maintain a used[] boolean array + a path list, append/pop on backtrack. — O(n!·n) time, O(n) extra.
Optimal Approach
  1. backtrack(i): if i==n save snapshot.
  2. For j in [i, n): swap(i,j), recurse(i+1), swap back.
Visual
flowchart TD
  A[fix index 0
try 0,1,2] --> B[fix index 1
try remaining]
  B --> C[fix index 2]
  C --> L[Leaf -> record permutation]
Code (Java)
/**
 * Generates every permutation of `nums` via swap-in-place backtracking.
 * `prefixLength` is the number of slots already fixed; positions [0..prefixLength)
 * hold the chosen prefix, positions [prefixLength..n) hold the remaining pool to try.
 */
void permute(int prefixLength, int[] nums, List<List<Integer>> allPermutations){
  if(prefixLength == nums.length){
    // All slots filled -- record an immutable snapshot of nums.
    List<Integer> permutationSnapshot = new ArrayList<>(nums.length);
    for(int value : nums) permutationSnapshot.add(value);
    allPermutations.add(permutationSnapshot);
    return;
  }

  // Try each remaining candidate in the slot at index `prefixLength`.
  for(int candidateIdx = prefixLength; candidateIdx < nums.length; candidateIdx++){
    swap(nums, prefixLength, candidateIdx);            // place candidate into slot
    permute(prefixLength + 1, nums, allPermutations);  // recurse on the smaller subproblem
    swap(nums, prefixLength, candidateIdx);            // backtrack: restore for next try
  }
}

/** In-place swap helper. */
void swap(int[] arr, int i, int j){
  int temp = arr[i];
  arr[i]   = arr[j];
  arr[j]   = temp;
}
Time O(n · n!)
Space O(n)

11Nth Root of M Medium

Section · Binary Search
GFG
Problem

Find an integer x such that x^n = m, else -1.

Mental Model
Binary search the answer space. The function f(x) = xⁿ is monotonic, so the candidate range [1, m] is searchable. Use a "multiply with overflow guard" instead of pow.
Other Approaches
Brute Try x = 1..m and check xⁿ == m. — O(m · n).
Better Same linear scan but stop early once xⁿ > m. — still O(m^(1/n) · n).
Optimal Approach
  1. lo=1, hi=m; while lo ≤ hi compute mid.
  2. Compute midⁿ with early break if it exceeds m.
  3. Shrink: equal → answer; less → lo=mid+1; greater → hi=mid-1.
Visual
flowchart TD
  S["lo=1, hi=m"] --> L{"lo <= hi?"}
  L -- yes --> M["mid · compute mid^n with cap"]
  M --> C{"compare with m"}
  C -- "==" --> A["Return mid"]
  C -- "<" --> R["lo = mid+1"]
  C -- ">" --> H["hi = mid-1"]
  R --> L
  H --> L
  L -- no --> N["Return -1"]
Code (Java)
/**
 * Computes base^exponent but caps the result at (cap + 1) once the cap is exceeded.
 * Lets us compare big powers against `cap` without ever overflowing.
 */
long powerWithCap(long base, int exponent, long cap){
  long result = 1;
  for(int step = 0; step < exponent; step++){
    result *= base;
    if(result > cap) return cap + 1;       // early break -- we only care if it exceeded cap
  }
  return result;
}

/**
 * Returns the integer x such that x^n == m, or -1 if no such integer exists.
 * Binary searches the answer range [1..m] because x -> x^n is monotonically increasing.
 */
int nthRoot(int n, int m){
  long low = 1, high = m;
  while(low <= high){
    long midCandidate  = (low + high) / 2;
    long midToThePower = powerWithCap(midCandidate, n, m);

    if(midToThePower == m)       return (int) midCandidate;
    else if(midToThePower < m)   low  = midCandidate + 1;
    else                         high = midCandidate - 1;
  }
  return -1;
}
Time O(n · log m)
Space O(1)

12Implement Max Heap Medium

Section · Heaps
GFG
Problem

Implement a binary max-heap supporting insert(x), extractMax(), and peek() on a fixed-capacity array.

Mental Model
A heap is a complete binary tree flattened into an array where, for max-heap, every parent ≥ its children. Index i's parent is (i-1)/2, children are 2i+1, 2i+2. Fix violations by swimming up on insert and sinking down on extract.
Other Approaches
Brute Plain sorted array — O(n) insert, O(1) extract. Slow inserts.
Better Self-balancing BST (TreeSet) — O(log n) for both, but heavier constant factor than a heap.
Optimal Approach
  1. insert: place value at end (size++), then sift-up while it is greater than its parent.
  2. extractMax: save data[0], move last element to index 0, decrement size, then sift-down swapping with the larger child while heap property breaks.
  3. peek: return data[0].
Visual
flowchart TD
  I["insert(x): place at end"] --> SU["sift-up while x > parent"]
  E["extractMax: take data[0]"] --> M["move last to index 0"]
  M --> SD["sift-down while child > node"]
Code (Java)
class MaxHeap{
  private final int[] data;
  private int size = 0;

  MaxHeap(int capacity){ data = new int[capacity]; }

  int  peek(){ return data[0]; }
  int  size(){ return size; }

  void insert(int value){
    data[size++] = value;
    siftUp(size - 1);
  }

  int extractMax(){
    int max = data[0];
    data[0] = data[--size];                       // move last to root
    siftDown(0);
    return max;
  }

  // Bubble node up while it is larger than its parent.
  private void siftUp(int i){
    while(i > 0){
      int parent = (i - 1) / 2;
      if(data[i] <= data[parent]) break;
      swap(i, parent);
      i = parent;
    }
  }

  // Push node down, swapping with the LARGER child while heap property is broken.
  private void siftDown(int i){
    while(true){
      int left = 2 * i + 1, right = 2 * i + 2, largest = i;
      if(left  < size && data[left]  > data[largest]) largest = left;
      if(right < size && data[right] > data[largest]) largest = right;
      if(largest == i) break;
      swap(i, largest);
      i = largest;
    }
  }

  private void swap(int i, int j){ int t = data[i]; data[i] = data[j]; data[j] = t; }
}
Time O(log n) insert/extract
Space O(n)

14Next Smaller Element Medium

Section · Stack and Queue Part-II
GFG
Problem

For each element, find the first smaller element to its right (or -1 if none).

Mental Model
Monotonic increasing stack walking right → left. While the top is ≥ current, it can't be anyone's "next smaller" anymore — pop it. The new top (if any) is the answer for the current index; then push the current value so it can serve future indices.
Other Approaches
Brute For each i, scan j > i linearly until a smaller value is found. — O(n²) time.

  2. Pop while stack.top() ≥ nums[i] — those values are useless going forward.
  3. nextSmaller[i] = stack.isEmpty() ? -1 : stack.peek(), then push nums[i].

flowchart TD

  P --> I

/**
 * For each i, returns the FIRST element strictly smaller than nums[i] to the right,
 * or -1 if no such element exists. Uses a monotonic-increasing stack walked right-to-left.
 */
int[] nextSmallerElement(int[] nums){
  int n = nums.length;
  int[] nextSmaller = new int[n];

  // Stack of values seen to the right of `i`. Kept monotonically INCREASING from top down,
  // so the top is always the smallest "still useful" candidate for any future i.
  Deque<Integer> candidateStack = new ArrayDeque<>();

  for(int i = n - 1; i >= 0; i--){
    int currentValue = nums[i];

    // Anything >= current can never be the "next smaller" for indices to the LEFT of i,
    // because i itself is closer and smaller-or-equal. Pop them.
    while(!candidateStack.isEmpty() && candidateStack.peek() >= currentValue){
      candidateStack.pop();
    }

    nextSmaller[i] = candidateStack.isEmpty() ? -1 : candidateStack.peek();
    candidateStack.push(currentValue);            // current becomes a candidate for i-1, i-2, ...
  }
  return nextSmaller;
}

/**

 * Walks right-to-left, picking up one word at a time and appending to a builder.

String reverseWords(String input){
  input = input.trim();                                       // drop leading / trailing spaces
  StringBuilder reversedSentence = new StringBuilder();

  int cursor = input.length() - 1;
  while(cursor >= 0){

    // Step 1: skip any spaces between words.
    while(cursor >= 0 && input.charAt(cursor) == ' ') cursor--;

16Z-Function (Pattern Matching) Hard

Section · String Part-II
CP-Algorithms
Problem

For string s, compute z[i] = longest substring starting at i that matches a prefix of s. Use p + "$" + t for pattern matching.

Mental Model
Maintain a Z-box [L, R]. If i is inside the current box, you can reuse the previously computed z[i − L] as a head start instead of recomputing from zero.
Other Approaches
Brute For each i, compare prefix vs s[i..] char by char from scratch. — O(n²).
Better KMP failure function for pattern matching, also O(n+m); Z-function is more uniform and elegant.
Optimal Approach
  1. Init L=R=0.
  2. For i in 1..n-1: if i < R, z[i] = min(R-i, z[i-L]).
  3. Extend while characters match; if i + z[i] > R, update L=i, R=i+z[i].
Visual
flowchart TD
  S["Z-box L..R"] --> C{"i < R?"}
  C -- yes --> M["z i = min(R-i, z i-L )"]
  C -- no --> Z["z i = 0"]
  M --> E["Extend while s z = s i+z"]
  Z --> E
  E --> U{"i + z i > R?"}
  U -- yes --> Up["Update L=i, R=i+z i"]
Code (Java)
/**
 * Computes the Z-array of `s`: zArray[i] = length of the longest substring starting
 * at i that matches a prefix of s. The famous "Z-box" trick reuses prior work to
 * achieve overall O(n) time.
 */
int[] zFunction(String s){
  int n = s.length();
  int[] zArray = new int[n];

  // [zBoxLeft, zBoxRight) is the rightmost-ending "match window" found so far,
  // i.e. s[zBoxLeft..zBoxRight) equals s[0..zBoxRight - zBoxLeft).
  int zBoxLeft = 0, zBoxRight = 0;

  for(int i = 1; i < n; i++){

    // If i is INSIDE the current Z-box we can warm-start using a mirrored value.
    if(i < zBoxRight){
      int mirroredIdx        = i - zBoxLeft;
      int remainingInBox     = zBoxRight - i;
      zArray[i] = Math.min(remainingInBox, zArray[mirroredIdx]);
    }

    // Try to extend the match character by character beyond the warm-started value.
    while(i + zArray[i] < n && s.charAt(zArray[i]) == s.charAt(i + zArray[i])){
      zArray[i]++;
    }

    // If we extended past the existing right boundary, slide the Z-box.
    if(i + zArray[i] > zBoxRight){
      zBoxLeft  = i;
      zBoxRight = i + zArray[i];
    }
  }
  return zArray;
}
Time O(n)
Space O(n)

17Inorder Traversal (Iterative) Easy

Section · Binary Tree

Return inorder (Left → Root → Right) traversal without recursion.

Other Approaches

Better Morris traversal: thread the tree using right pointers; O(n) time, O(1) space (mutates and restores).

  3. Pop, output, then curr=popped.right.
Visual
flowchart TD
  R[curr = root] --> L{curr != null?}
  L -- yes --> P[push curr · curr = curr.left]
  P --> L
  L -- no --> S{stack empty?}
  S -- no --> Pop[pop · output · curr = pop.right]
  Pop --> L
  S -- yes --> D[Done]
Code (Java)
/**
 * Iterative inorder traversal: Left -> Root -> Right.
 * Use an explicit stack to simulate the call stack of the recursive version.
 */
List<Integer> inorderTraversal(TreeNode root){
  List<Integer> inorderResult = new ArrayList<>();
  Deque<TreeNode> ancestorStack = new ArrayDeque<>();
  TreeNode currentNode = root;

  while(currentNode != null || !ancestorStack.isEmpty()){

    // 1) Dive as far LEFT as possible, remembering each ancestor on the stack.
    while(currentNode != null){
      ancestorStack.push(currentNode);
      currentNode = currentNode.left;
    }

    // 2) The deepest unvisited ancestor is the next node in inorder.
    currentNode = ancestorStack.pop();
    inorderResult.add(currentNode.val);

    // 3) Now explore its right subtree (which itself starts with another left dive).
    currentNode = currentNode.right;
  }
  return inorderResult;
}
Time O(n)
Space O(h)

/**
 * BFS level-order traversal. Outer loop processes ONE level per iteration by
 * snapshotting the queue size before draining it.
 */
List<List<Integer>> levelOrder(TreeNode root){
  List<List<Integer>> levels = new ArrayList<>();
  if(root == null) return levels;

  Deque<TreeNode> bfsQueue = new ArrayDeque<>();
  bfsQueue.offer(root);

  while(!bfsQueue.isEmpty()){

      if(node.left  != null) bfsQueue.offer(node.left);
      if(node.right != null) bfsQueue.offer(node.right);
    }
    levels.add(currentLevelValues);
  }
  return levels;
}

19Maximum Path Sum (Binary Tree) Hard

Section · Binary Tree Part-III
LeetCode
Problem

A path is any sequence of nodes connected by parent-child edges (need not pass through the root). Return the maximum sum among all paths. Node values can be negative.

Mental Model
Each node has two roles. (1) As an internal bend point a path goes left → node → right — its sum is node.val + leftGain + rightGain and is a candidate for the global answer. (2) As a contributor to a parent only one side can be extended upward, so it returns node.val + max(left, right). Negative gains are clamped to 0.
Other Approaches
Brute Enumerate every pair (u, v) and compute the path sum via LCA. — O(n²) or worse.
Optimal Approach
  1. DFS each node; recursively get leftGain, rightGain, each clamped at 0.
  2. Update a global best with node.val + leftGain + rightGain (path bends here).
  3. Return node.val + max(leftGain, rightGain) upward — only one branch can be reused by an ancestor.
Visual
flowchart TD
  N["node"] --> L["leftGain = max(0, dfs(left))"]
  N --> R["rightGain = max(0, dfs(right))"]
  L --> U["update best: node.val + leftGain + rightGain"]
  R --> U
  U --> Ret["return node.val + max(leftGain, rightGain)"]
Code (Java)
// Tracks the best answer across the whole tree (a path may bend at any node).
int globalBestPathSum;

/**
 * Returns the maximum sum of any node-to-node path in the tree.
 * Each node has TWO roles: it can be the "bend point" of a path (uses both child gains),
 * OR it can be a contributor to its parent (then only one side may be reused).
 */
int maxPathSum(TreeNode root){
  globalBestPathSum = Integer.MIN_VALUE;
  maxDownwardGain(root);
  return globalBestPathSum;
}

/**
 * Returns the maximum sum of a path that STARTS at `node` and goes strictly DOWNWARD.
 * Side-effect: updates `globalBestPathSum` with the best path that BENDS at `node`.
 */
private int maxDownwardGain(TreeNode node){
  if(node == null) return 0;

  // Discard negative subtrees -- not taking them is strictly better.
  int leftGain  = Math.max(0, maxDownwardGain(node.left));
  int rightGain = Math.max(0, maxDownwardGain(node.right));

  // "Bend at this node" candidate uses BOTH children plus the node's own value.
  int bendingPathSum = node.val + leftGain + rightGain;
  globalBestPathSum  = Math.max(globalBestPathSum, bendingPathSum);

  // To a parent, this subtree can contribute only the BIGGER side.
  return node.val + Math.max(leftGain, rightGain);
}
Time O(n)
Space O(h) recursion

20Populate Next Right Pointers Medium

Section · Binary Search Tree
LeetCode
Problem

Perfect binary tree — link each node's next to its right neighbor on the same level. Use O(1) extra space.

Mental Model
Build level k+1 using level k's already-wired next chain. A "level head" pointer walks down-left; a "current" sweeps along the chain at the parent level, wiring the children below: cur.left.next = cur.right; cur.right.next = cur.next?.left.
Other Approaches
Brute Standard BFS by level using a queue, link adjacent nodes within each level. — O(n) time, O(width) space.
Optimal Approach
  1. levelHead = root; while levelHead.left != null:
  2. curr = levelHead; while curr != null: wire curr.left.next = curr.right and curr.right.next = curr.next != null ? curr.next.left : null; advance curr = curr.next.
  3. Descend: levelHead = levelHead.left.
Visual
flowchart LR
  P1["Parent"] -->|next| P2["Parent2"]
  P1 --> CL1["L1"]
  P1 --> CR1["R1"]
  P2 --> CL2["L2"]
  P2 --> CR2["R2"]
  CL1 -->|wire| CR1 -->|wire| CL2 -->|wire| CR2
Code (Java)
/**
 * Wires `next` pointers so each node points to its right neighbor on the same level.
 * Works in O(1) extra space by using already-wired `next` pointers on level k
 * to traverse and wire level k+1.
 */
Node connect(Node root){
  if(root == null) return null;

  Node leftmostOfCurrentLevel = root;

  // Stop when we reach the leaves -- they have no children to wire.
  while(leftmostOfCurrentLevel.left != null){

    // Sweep across the current level using the next-chain that's already there.
    Node parentNode = leftmostOfCurrentLevel;
    while(parentNode != null){
      // Wire the two children of THIS parent.
      parentNode.left.next  = parentNode.right;

      // Wire across siblings: rightChild -> leftChildOfNextParent (if any).
      parentNode.right.next = (parentNode.next != null) ? parentNode.next.left : null;

      parentNode = parentNode.next;
    }

    // Drop down one level (perfect tree, so leftmost is always reachable via .left).
    leftmostOfCurrentLevel = leftmostOfCurrentLevel.left;
  }
  return root;
}

/**

int floor(TreeNode root, int key){

Node dllHead;       // will become the head of the resulting doubly-linked list

/**
 * Inorder DFS. Each visited node is wired to the previously visited one, building
 * the DLL in sorted order one link at a time.
 */
void inorderStitch(Node node){
  if(node == null) return;

  inorderStitch(node.left);

  if(dllPrevious == null){
    dllHead = node;                // very first node visited becomes the DLL head
  } else {
    dllPrevious.right = node;      // prev -> node
    node.left         = dllPrevious; // node -> prev

  dllPrevious = node;              // advance the "previously visited" marker

  inorderStitch(node.right);
}

Node treeToDoublyLinkedList(Node root){
  dllHead = null;
  dllPrevious = null;

  inorderStitch(root);

  if(dllHead != null){             // close the cycle: head <-> tail
    dllHead.left      = dllPrevious;
    dllPrevious.right = dllHead;
  }
  return dllHead;
}

/**
 * Returns a deep copy of the connected undirected graph reachable from `originalNode`.
 * DFS that creates the clone BEFORE recursing into neighbors, so cycles don't loop forever.
 */
Node cloneGraph(Node originalNode){
  if(originalNode == null) return null;

  // Already cloned? Reuse to preserve graph identity (no duplicates, cycles preserved).
  if(originalToClone.containsKey(originalNode)){

}

24Strongly Connected Components (Kosaraju) Hard

Section · Graph Part-II
GFG
Problem

In a directed graph, find all strongly connected components — maximal sets of vertices where every pair is mutually reachable.

Mental Model
Two DFS passes around a transpose. The 1st DFS records vertices in finish-time order on a stack. After reversing all edges, the 2nd DFS popping in that order naturally peels off one SCC at a time — because the last-finished vertex must lie in a sink SCC of the original graph (= source SCC of the reverse).

  2. Transpose: build the reverse graph (flip every edge).
  3. Pass 2: pop from the stack; if unvisited in the transpose, DFS it in the transpose — every vertex visited in this single DFS forms one SCC.

List<List<Integer>> kosarajuSCC(int n, List<Integer>[] graph){

  Arrays.fill(visited, false);
  List<List<Integer>> components = new ArrayList<>();
  while(!finishOrder.isEmpty()){
    int v = finishOrder.pop();
    if(visited[v]) continue;
    List<Integer> component = new ArrayList<>();
    dfsCollect(v, transpose, visited, component);
    components.add(component);
  }
  return components;
}

private void dfsFinish(int u, List<Integer>[] graph, boolean[] visited, Deque<Integer> finishOrder){
  visited[u] = true;
  for(int v : graph[u])
    if(!visited[v]) dfsFinish(v, graph, visited, finishOrder);
  finishOrder.push(u);                        // push AFTER children done
}

private void dfsCollect(int u, List<Integer>[] transpose, boolean[] visited, List<Integer> component){
  visited[u] = true;
  component.add(u);
  for(int v : transpose[u])
    if(!visited[v]) dfsCollect(v, transpose, visited, component);
}
Time O(V + E)
Space O(V + E)

25Maximum Product Subarray Medium

Section · Dynamic Programming
LeetCode
Problem

Find the contiguous subarray with the largest product.

Mental Model
Track both max AND min so far. A negative number flips them — today's min × negative becomes tomorrow's max. Reset on zero.
Other Approaches
Brute Two nested loops compute every subarray's product. — O(n²).
Better Sweep from left and right separately, multiplying running product (reset on zero); take the max. — O(n) time, O(1) space.
Optimal Approach
  1. Maintain curMax, curMin, ans.
  2. For each x: if x < 0 swap curMax/curMin.
  3. curMax = max(x, curMax*x); curMin = min(x, curMin*x); update ans.
Visual
flowchart LR
  X[next x] --> N{x < 0?}
  N -- yes --> SW[swap curMax · curMin]
  N -- no --> U[Update]
  SW --> U
  U --> A[ans = max ans, curMax]
  A --> X
Code (Java)
/**
 * Largest product of any contiguous subarray.
 * Track BOTH the running max AND the running min ending at the current index --
 * because a future negative number can flip the smallest (most negative) into the largest.
 */
int maxProduct(int[] nums){
  int runningMax = nums[0];           // best product of any subarray ending HERE
  int runningMin = nums[0];           // worst product of any subarray ending HERE
  int globalBest = nums[0];           // overall answer

  for(int i = 1; i < nums.length; i++){
    int currentValue = nums[i];

    // A negative value swaps the roles: yesterday's min times negative is today's max.
    if(currentValue < 0){
      int swapTemp = runningMax;
      runningMax   = runningMin;
      runningMin   = swapTemp;
    }

    // Either extend the previous subarray or start fresh at the current value.
    runningMax = Math.max(currentValue, runningMax * currentValue);
    runningMin = Math.min(currentValue, runningMin * currentValue);

    globalBest = Math.max(globalBest, runningMax);
  }
  return globalBest;
}
Time O(n)
Space O(1)

26Minimum Path Sum in a Grid Medium

Section · Dynamic Programming Part-II
LeetCode
Problem

Given an m × n non-negative grid, find the path from top-left to bottom-right with minimum sum, moving only right or down.

Mental Model
Each cell asks: "who is my cheaper parent — the one above, or the one to the left?" Then it adds its own cost. Define dp[i][j] = min sum to reach (i,j); recurrence dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1]). Borders only have one parent.
Other Approaches
Brute Recursion exploring both moves from (0,0). — O(2^(m+n)) time.
Better Memoized recursion (top-down DP). — O(m·n) time, O(m·n) space + recursion stack.
Optimal Approach
  1. Iterate row by row. Cell (0,0) is the base case.
  2. For top row, only "left" neighbor exists; for left column, only "above".
  3. Otherwise dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1]).
  4. Answer is dp[m-1][n-1]. Optionally compress to a 1-D array since each row needs only the previous row.
Visual
flowchart TD
  C["Cell (i,j)"] --> U["From above: dp[i-1][j]"]
  C --> L["From left:  dp[i][j-1]"]
  U --> M["min(above, left)"]
  L --> M
  M --> A["dp[i][j] = grid[i][j] + min"]
Code (Java)
/**
 * Minimum sum path from grid[0][0] to grid[m-1][n-1], moving only right or down.
 * dp[col] = min sum to reach the current row at column `col`. Updated in place
 * row by row: BEFORE assignment dp[col] holds the value from the row above;
 * AFTER assignment it holds the value for the current row.
 */
int minPathSum(int[][] grid){
  int rowCount = grid.length;
  int colCount = grid[0].length;
  int[] minSumToReach = new int[colCount];

  // Top-left corner is the base case.
  minSumToReach[0] = grid[0][0];
  // Top row: only the "come from left" option is available.
  for(int col = 1; col < colCount; col++){
    minSumToReach[col] = minSumToReach[col - 1] + grid[0][col];
  }

  // Process each subsequent row, reusing minSumToReach as a rolling 1-D DP.
  for(int row = 1; row < rowCount; row++){
    minSumToReach[0] += grid[row][0];                 // first column: only "come from above"

    for(int col = 1; col < colCount; col++){
      int costFromAbove = minSumToReach[col];         // value still from PREVIOUS row
      int costFromLeft  = minSumToReach[col - 1];     // value already updated for THIS row
      minSumToReach[col] = grid[row][col] + Math.min(costFromAbove, costFromLeft);
    }
  }
  return minSumToReach[colCount - 1];
}
Time O(m·n)
Space O(n) rolling row

27Implement Trie (Prefix Tree) Medium

Section · Trie
LeetCode
Problem

Insert, search, startsWith — all on lowercase strings.

Mental Model
A 26-ary tree where each edge is a character. Walk character by character creating nodes lazily; mark a node isEnd when a complete word terminates there.
Other Approaches
Brute A HashSet of all words: insert/search are O(L), but startsWith requires scanning every word — O(N·L).
Better A sorted array + binary search for prefix range — fast lookup but slow inserts (O(N) shift).
Optimal Approach
  1. Each node has children[26] and isEnd.
  2. insert: walk/create per char, mark end.
  3. search/startsWith: walk; missing child means false. search additionally requires isEnd.
Visual
flowchart TD
  Root((root)) --> a((a))
  Root --> t((t))
  a --> p((p))
  p --> pE(("p · isEnd: app"))
  pE --> l((l))
  l --> e(("e · isEnd: apple"))
Code (Java)
class Trie{
  static class TrieNode{
    TrieNode[] children = new TrieNode[26];
    boolean isEndOfWord;
  }
  TrieNode root = new TrieNode();

  void insert(String word){
    TrieNode node = root;
    for(char ch : word.toCharArray()){
      int idx = ch - 'a';
      if(node.children[idx] == null) node.children[idx] = new TrieNode();
      node = node.children[idx];
    }
    node.isEndOfWord = true;
  }

  // Walks the trie following `s`. Returns the final node, or null if any char is missing.
  private TrieNode walk(String s){
    TrieNode node = root;
    for(char ch : s.toCharArray()){
      int idx = ch - 'a';
      if(node.children[idx] == null) return null;
      node = node.children[idx];
    }
    return node;
  }

  boolean search(String word)      { TrieNode node = walk(word);   return node != null && node.isEndOfWord; }
  boolean startsWith(String prefix){ return walk(prefix) != null; }
}
Time O(L) per op
Space O(N·L)
