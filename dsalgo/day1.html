01Set Matrix Zeroes Medium

Section · Arrays
LeetCode YouTube
Problem

Given an m × n matrix, if any element is 0, set its entire row and column to 0. Do it in-place with O(1) extra space.

Example

Input A: [[1,1,1],[1,0,1],[1,1,1]] · Output: [[1,0,1],[0,0,0],[1,0,1]]

  1. Pass 1: zero at (1,1) → mark row[1]=true, col[1]=true using first row/col as flags.
  2. Pass 2: for each cell (i,j) with i,j≥1, zero it if its row or column flag is set.
  3. Finally zero first row/col based on the original-firstRow/firstCol booleans captured at start.

Input B (edge — zeros in first row/col): [[0,1,2,0],[3,4,5,2],[1,3,1,5]] · Output: [[0,0,0,0],[0,4,5,0],[0,3,1,0]]

  1. firstRowZero=true, firstColZero=false.
  2. Mark col[3] via (0,3)=0; cols 0 and 3 get zeroed in pass 2.
  3. Finally row 0 zeroed entirely because firstRowZero.
Mental Model
Reuse the matrix as your memory. The first row & first column become "flag arrays" telling which rows/cols must be zeroed. Use one extra boolean for the first column itself to break ambiguity at matrix[0][0].
Other Approaches
Brute Mark every zero cell's row & column with a sentinel (like -1), then convert sentinels to 0. — O((R·C)·(R+C)).
Better Two boolean arrays row[R] and col[C]: mark in pass-1, zero in pass-2. — O(R·C) time, O(R+C) space.
Optimal Approach
  1. Use row[0] and col[0] as markers.
  2. Track col0=1 separately for the first column.
  3. Sweep cells (1..n-1, 1..m-1): if zero, mark matrix[i][0]=0 and matrix[0][j]=0.
  4. Second pass (inside-out): zero cells whose row/col marker is 0.
  5. Finally handle matrix[0][0] via row[0] flag and col0.

/**
 * Sets every entire row and column to 0 if any cell in that row/column is 0.
 * Uses the FIRST row and FIRST column themselves as O(1)-space markers.
 * Column 0 needs a separate boolean because it shares cell (0,0) with row 0.
 */
void setZeroes(int[][] matrix){
  int m = matrix.length;
  int n = matrix[0].length;
  boolean col0 = false;

  // ---- Pass 1: scan & mark zeros into the first row / first column ----
  for(int i = 0; i < m; i++){
    if(matrix[i][0] == 0) col0 = true;                        // remember separately
    for(int j = 1; j < n; j++){
      if(matrix[i][j] == 0){
        matrix[i][0] = 0;                                     // mark the row
        matrix[0][j] = 0;                                     // mark the column
      }
    }
  }

  // ---- Pass 2: zero the cells, processing bottom-up so we don't overwrite markers too early ----
  for(int i = m - 1; i >= 0; i--){
    for(int j = n - 1; j >= 1; j--){                          // skip column 0 here
      if(matrix[i][0] == 0 || matrix[0][j] == 0) matrix[i][j] = 0;
    }
    if(col0) matrix[i][0] = 0;                                // handle column 0 last
  }
}
Time O(R·C)
Space O(1)

02Rotate Matrix by 90° Medium

Problem

Rotate an n × n matrix clockwise by 90° in-place.

Example

Input A: [[1,2,3],[4,5,6],[7,8,9]] · Output: [[7,4,1],[8,5,2],[9,6,3]]

  1. Transpose (swap m[i][j] with m[j][i] for i<j) → [[1,4,7],[2,5,8],[3,6,9]].
  2. Reverse each row → [[7,4,1],[8,5,2],[9,6,3]]. ✓

Optimal Approach

  2. Reverse each row in place.

/**
 * Rotates an n x n matrix 90 degrees clockwise, in place.
 * Trick: a 90-CW rotation == transpose along the main diagonal, then reverse each row.
 */
void rotate(int[][] matrix){
  int n = matrix.length;

  // ---- 1) Transpose: swap matrix[i][j] with matrix[j][i] for j > i ----
  for(int i = 0; i < n; i++){
    for(int j = i + 1; j < n; j++){
      int tmp = matrix[i][j];
      matrix[i][j] = matrix[j][i];
      matrix[j][i] = tmp;
    }
  }

  // ---- 2) Reverse each row (mirror left-right) ----
  for(int i = 0; i < n; i++){
    int l = 0, r = n - 1;
    while(l < r){
      int tmp = matrix[i][l];
      matrix[i][l]  = matrix[i][r];
      matrix[i][r] = tmp;
      l++;
      r--;
    }
  }
}
Time O(n²)
Space O(1)

Problem

Each row sorted, first integer of each row > last integer of previous row. Search for target.

Example

Input A: matrix=[[1,3,5,7],[10,11,16,20],[23,30,34,60]], target=11 · Output: true

  1. Treat as flat sorted array of length 3·4=12; binary search on [0..11].

  3. Compare and shrink the window.

/**
 * Searches for `target` in a row-sorted matrix where each row's first element
 * is greater than the previous row's last element. Treats the matrix as a single
 * sorted array of size rows*cols and binary-searches it.
 */
boolean searchInSortedMatrix(int[][] matrix, int target){
  int m = matrix.length;
  int n = matrix[0].length;

  int lo  = 0;
  int hi = m * n - 1;

  while(lo <= hi){
    int mid = (lo + hi) >>> 1;                                // unsigned shift = avoids overflow
    int r   = mid / n;                                      // map 1-D index back to 2-D coords
    int c   = mid % n;
    int v   = matrix[r][c];

    if(v == target)      return true;
    else if(v < target)  lo  = mid + 1;
    else                 hi = mid - 1;
  }
  return false;
}

04Two Sum Easy

Section · Arrays Part-IV
LeetCode
Problem

Return indices i, j such that a[i] + a[j] == target.

Example

Input A: nums=[2,7,11,15], target=9 · Output: [0,1]

  1. i=0, need=9-2=7; map empty → store {2:0}.
  2. i=1, need=9-7=2; map has 2→0 → return [0,1].

Input B (edge — duplicate value): nums=[3,3], target=6 · Output: [0,1]

  1. i=0: store {3:0}.
  2. i=1, need=3; map has 3→0 → return [0,1] (we look up before inserting, so the same index is never reused).
Mental Model
"Have I seen the complement?" While walking the array, ask the hash map if target − current already passed by. Trade memory for one linear scan.
Other Approaches
Brute Two nested loops checking every pair. — O(n²) time, O(1) space.
Better Sort + two pointers (only when returning values, not indices). — O(n log n) time, O(1) space.
Optimal Approach
  1. Single pass with HashMap of value → index.
  2. For each a[i]: if map.contains(target − a[i]) return pair.
  3. Else put a[i] → i and continue.

/**
 * Returns the indices of two numbers in `nums` that add up to `target`.
 * Walks the array once, asking the hash map: "have I already seen the complement?"
 */
int[] twoSum(int[] nums, int target){
  Map<Integer, Integer> map = new HashMap<>();

  for(int i = 0; i < nums.length; i++){
    int x = nums[i];
    int need = target - x;                                 // what we still need

    if(map.containsKey(need)){
      int j = map.get(need);
      return new int[]{ j, i };
    }
    map.put(x, i);                                         // remember for future lookups
  }
  return new int[]{ -1, -1 };                             // no pair found
}
Time O(n)
Space O(n)

Problem

Given head of a singly linked list, return the reversed list.

Example

Input A: 1→2→3→4→5 · Output: 5→4→3→2→1

  1. prev=null, curr=1 → save next=2; flip 1.next=null; prev=1, curr=2.
  2. Repeat: after node 2 → 2→1; node 3 → 3→2→1; node 4 → 4→3→2→1; node 5 → 5→4→3→2→1.

  2. Single: one iteration flips 1.next=null, prev=1; return 1.

Three pointers, one snip at a time. prev is the new tail-side, curr is the worker, next is your safety net before flipping the arrow.

Optimal Approach
  1. Init prev=null, curr=head.
  2. Loop: save next=curr.next; flip curr.next=prev; advance prev=curr; curr=next.
  3. Return prev.

/**
 * Reverses a singly-linked list iteratively.
 * Maintains 3 pointers: `prev` (already reversed part),
 * `cur` (the one being flipped), `next` (safety save before we lose it).
 */
ListNode reverseListIterative(ListNode head){
  ListNode prev = null;
  ListNode cur  = head;

  while(cur != null){
    ListNode next = cur.next;   // save next BEFORE we overwrite the pointer
    cur.next  = prev;           // flip the arrow backwards
    prev      = cur;            // shift the "reversed-so-far" tail forward
    cur       = next;           // advance the worker
  }
  return prev;                   // prev ends up as the new head
}
Time O(n)
Space O(1)
Code · Recursive (Java)
Recursive intuition: trust recursion to reverse head.next onwards; when it returns, the node right after head is the new tail — point it back to head, then sever head.next.
/**
 * Reverses a linked list recursively.
 * Trust the recursion to reverse everything from `head.next` onward. Once it returns,
 * the node directly after `head` is now the last node of that reversed sub-list -- so
 * make IT point back to `head`, and `head` itself becomes the new tail (next = null).
 */
ListNode reverseListRecursive(ListNode head){
  // Base case: empty list or single node is already "reversed".
  if(head == null || head.next == null) return head;

  ListNode newHead = reverseListRecursive(head.next);  // recursion handles the rest

  head.next.next = head;   // the node that USED to be after head now points BACK at head
  head.next      = null;   // head is now the tail of the reversed list
  return newHead;
}
Time O(n)
Space O(n) stack

06Intersection of Two Linked Lists Easy

Section · Linked List Part-II
LeetCode
Problem

Find the node where two singly-linked lists merge (or null).

Example

Input A: A: 4→1→8→4→5   B: 5→6→1→8→4→5 (intersect at node 8) · Output: node 8

  1. Two pointers pA, pB walk their lists; when one ends, jump to the other's head.
  2. Each pointer travels exactly lenA + lenB steps, so they align at the intersection: meet at 8.

Input B (edge — no intersection): disjoint lists · Output: null

  1. Both pointers walk lenA+lenB steps and simultaneously become null → return null. No infinite loop.
Mental Model
Equalize the runways. Two pointers; when one hits the end, jump to the OTHER list's head. After at most m+n steps they either meet at the intersection or both reach null together.
Other Approaches
Brute For every node in A, scan all of B for equality. — O(m·n).
Better Put all A nodes in a hash set; walk B. — O(m+n) time, O(m) space. Or: compute lengths, advance the longer pointer by |m-n|, then walk together. — O(m+n) time, O(1) space.
Optimal Approach
  1. a = headA, b = headB.
  2. Loop until a == b: if a==null set a=headB else a=a.next; same for b with headA.
  3. Return a.

flowchart LR
  A[a walks A then B] --> X((Meet at intersection))
  B[b walks B then A] --> X

Code (Java)
/**
 * Returns the node where two singly-linked lists meet, or null if they never do.
 * Idea: each pointer walks its own list, then jumps to the OTHER list's head.
 * After at most (lenA + lenB) steps the two pointers must align -- either at the
 * intersection node, or both at null simultaneously.
 */
ListNode getIntersectionNode(ListNode headA, ListNode headB){
  if(headA == null || headB == null) return null;

  ListNode a = headA;
  ListNode b = headB;

  // Loop terminates because both walkers traverse exactly lenA + lenB nodes total.
  while(a != b){
    a = (a == null) ? headB : a.next;   // hop onto list B at the end
    b = (b == null) ? headA : b.next;   // hop onto list A at the end
  }
  return a;   // intersection node, or null if no intersection
}
Time O(m+n)
Space O(1)

Problem

Rotate a linked list to the right by k places.

Example

Input A: 1→2→3→4→5, k=2 · Output: 4→5→1→2→3

  1. Compute length n=5; close into a ring (tail.next = head).
  2. New tail is at position n - k%n - 1 = 5-2-1 = 2 (0-indexed) → node 3.
  3. New head = 3.next = 4; cut 3.next=null → 4→5→1→2→3.

  2. k %= L; new tail at index L - k - 1.
  3. New head = newTail.next; newTail.next = null.

/**
 * Rotates a singly-linked list to the RIGHT by k positions.
 * Strategy: turn the list into a circle, find the new tail, then break the circle.
 */
ListNode rotateRight(ListNode head, int k){
  if(head == null || head.next == null || k == 0) return head;

  // ---- 1) Walk to the tail and count length along the way ----
  int n = 1;
  ListNode tail = head;
  while(tail.next != null){
    tail = tail.next;
    n++;
  }

  // ---- 2) Close the loop into a circle ----
  tail.next = head;

  // ---- 3) The new tail is at position (length - k%length - 1) from the original head ----
  int steps        = n - k % n - 1;
  ListNode newTail = head;
  while(steps-- > 0) newTail = newTail.next;

  // ---- 4) The node after the new tail is the new head; cut the circle there ----
  ListNode newHead = newTail.next;
  newTail.next     = null;
  return newHead;
}
Time O(n)
Space O(1)

08N Meetings in One Room Easy

Problem

Maximum number of non-overlapping meetings, given start[] and end[].

Example

/**
 * Returns the maximum number of non-overlapping meetings that fit in one room.
 * Greedy proof: among all valid choices for the FIRST meeting, picking the one
 * that ends earliest leaves the largest possible suffix for the rest -- so always
 * take the meeting that finishes soonest.
 */
int maxMeetings(int[] start, int[] end){
  int n = start.length;

  // We sort INDICES (not times) so we can still look up matching start/end pairs.
  Integer[] order = new Integer[n];
  for(int i = 0; i < n; i++) order[i] = i;
  Arrays.sort(order, (a, b) -> end[a] - end[b]);

  // The earliest-ending meeting is always part of an optimal answer.
  int cnt = 1;
  int last = end[order[0]];

  for(int i = 1; i < n; i++){
    int j = order[i];
    if(start[j] > last){                                    // strictly after previous end
      cnt++;
      last = end[j];
    }
  }
  return cnt;
}
Time O(n log n)
Space O(n)
Deep Dive · How the Comparator Lambda Actually Works
Parameter names are just labels. The sort algorithm always hands your lambda two elements in a fixed order — call them slot 1 (first arg) and slot 2 (second arg). What controls ascending vs descending is which slot ends up as the first argument to Integer.compare(...), not what you named them.
The Contract

The sort calls your comparator and reads only the sign of the returned number:

  • negative → "the FIRST argument I gave you should come first"
  • positive → "the SECOND argument I gave you should come first"
  • zero → equal, doesn't matter

Integer.compare(x, y) is conceptually just x - y:

  • Integer.compare(3, 7) → -4 (negative)
  • Integer.compare(7, 3) → +4 (positive)
Why Swapping Inputs Flips The Order

Suppose sort hands your lambda 3 (slot 1) and 7 (slot 2):

Case A (a,b) -> Integer.compare(a, b) → returns -4 → sort keeps 3 before 7 → ascending
Case B (a,b) -> Integer.compare(b, a) → returns +4 → sort puts 7 before 3 → descending

Sort doesn't know you swapped anything. It only knows it gave you (3, 7) and you returned +4, so it dutifully swaps the pair. Computing the comparison of the wrong pair from sort's perspective inverts the sign of every answer → reverses the whole order.

All Four Variations Side-by-Side
Lambda Slot 1 named First arg to compare Result
(a, b) -> compare(a[0], b[0])aslot 1ascending
(a, b) -> compare(b[0], a[0])aslot 2descending
(b, a) -> compare(a[0], b[0])bslot 2descending
(b, a) -> compare(b[0], a[0])bslot 1ascending
Mental shortcut. Ignore the names entirely. Compare the order of arguments inside compare(...) against the order in the lambda's parameter list:
  • Same order → ascending
  • Flipped order → descending

/**
 * Generates every permutation of `nums` via swap-in-place backtracking.
 * `k` is the number of slots already fixed; positions [0..k)
 * hold the chosen prefix, positions [k..n) hold the remaining pool to try.
 */
void permute(int k, int[] nums, List<List<Integer>> res){
  if(k == nums.length){
    // All slots filled -- record an immutable snapshot of nums.
    List<Integer> perm = new ArrayList<>(nums.length);
    for(int x : nums) perm.add(x);
    res.add(perm);
    return;
  }

  // Try each remaining candidate in the slot at index `k`.
  for(int j = k; j < nums.length; j++){
    swap(nums, k, j);            // place candidate into slot
    permute(k + 1, nums, res);   // recurse on the smaller subproblem
    swap(nums, k, j);            // backtrack: restore for next try
  }
}

/** In-place swap helper. */

  int t = arr[i];
  arr[i]   = arr[j];
  arr[j]   = t;

/**

 * Lets us compare big powers against `cap` without ever overflowing.

  return res;

int nthRoot(int n, int m){

    else if(p < m)   lo  = mid + 1;

  }

class MaxHeap{
  private final int[] data;
  private int size = 0;

  MaxHeap(int capacity){ data = new int[capacity]; }

  int  peek(){ return data[0]; }
  int  size(){ return size; }

  void insert(int value){
    data[size++] = value;
    siftUp(size - 1);
  }

  int extractMax(){
    int max = data[0];
    data[0] = data[--size];                       // move last to root
    siftDown(0);
    return max;
  }

  // Bubble node up while it is larger than its parent.
  private void siftUp(int i){
    while(i > 0){
      int parent = (i - 1) / 2;
      if(data[i] <= data[parent]) break;
      swap(i, parent);
      i = parent;
    }
  }

  // Push node down, swapping with the LARGER child while heap property is broken.
  private void siftDown(int i){
    while(true){
      int left = 2 * i + 1, right = 2 * i + 2, largest = i;
      if(left  < size && data[left]  > data[largest]) largest = left;
      if(right < size && data[right] > data[largest]) largest = right;
      if(largest == i) break;
      swap(i, largest);
      i = largest;
    }
  }

  private void swap(int i, int j){ int t = data[i]; data[i] = data[j]; data[j] = t; }
}

 * For each i, returns the FIRST element strictly smaller than nums[i] to the right,

  // Stack of values seen to the right of `i`. Kept monotonically INCREASING from top down,
  // so the top is always the smallest "still useful" candidate for any future i.

/**
 * Computes the Z-array of `s`: zArray[i] = length of the longest substring starting
 * at i that matches a prefix of s. The famous "Z-box" trick reuses prior work to
 * achieve overall O(n) time.
 */
int[] zFunction(String s){
  int n = s.length();
  int[] z = new int[n];

  // [l, r) is the rightmost-ending "match window" found so far,
  // i.e. s[l..r) equals s[0..r - l).
  int l = 0, r = 0;

  for(int i = 1; i < n; i++){

    // If i is INSIDE the current Z-box we can warm-start using a mirrored value.
    if(i < r){
      int rem = r - i;
      z[i] = Math.min(rem, z[i - l]);
    }

    // Try to extend the match character by character beyond the warm-started value.
    while(i + z[i] < n && s.charAt(z[i]) == s.charAt(i + z[i])){
      z[i]++;
    }

    // If we extended past the existing right boundary, slide the Z-box.
    if(i + z[i] > r){
      l = i;
      r = i + z[i];
    }
  }
  return z;
}

 * Use an explicit stack to simulate the call stack of the recursive version.

List<Integer> inorderTraversal(TreeNode root){

    cur = cur.right;

/**
 * Returns the maximum sum of any node-to-node path in the tree.
 * Each node has TWO roles: it can be the "bend point" of a path (uses both child gains),

/**
 * Returns the maximum sum of a path that STARTS at `node` and goes strictly DOWNWARD.
 * Side-effect: updates `best` with the best path that BENDS at `node`.
 */
private int dfs(TreeNode node){
  if(node == null) return 0;

  // Discard negative subtrees -- not taking them is strictly better.
  int l = Math.max(0, dfs(node.left));
  int r = Math.max(0, dfs(node.right));

  // "Bend at this node" candidate uses BOTH children plus the node's own value.
  int bend = node.val + l + r;
  best     = Math.max(best, bend);

  // To a parent, this subtree can contribute only the BIGGER side.
  return node.val + Math.max(l, r);
}

/**
 * Wires `next` pointers so each node points to its right neighbor on the same level.
 * Works in O(1) extra space by using already-wired `next` pointers on level k
 * to traverse and wire level k+1.

Node connect(Node root){
  if(root == null) return null;

  Node head = root;

  // Stop when we reach the leaves -- they have no children to wire.
  while(head.left != null){

    // Sweep across the current level using the next-chain that's already there.
    Node cur = head;
    while(cur != null){
      // Wire the two children of THIS parent.
      cur.left.next  = cur.right;

      // Wire across siblings: rightChild -> leftChildOfNextParent (if any).
      cur.right.next = (cur.next != null) ? cur.next.left : null;

      cur = cur.next;
    }

    // Drop down one level (perfect tree, so leftmost is always reachable via .left).
    head = head.left;
  }
  return root;
}

    }
  }

 * Inorder DFS. Each visited node is wired to the previously visited one, building
 * the DLL in sorted order one link at a time.

  inorderStitch(node.left);

  if(prev == null){
    head = node;                   // very first node visited becomes the DLL head
  } else {

    node.left  = prev;             // node -> prev
  }
  prev = node;                     // advance the "previously visited" marker

  inorderStitch(node.right);
}

Node treeToDoublyLinkedList(Node root){
  head = null;
  prev = null;

  inorderStitch(root);

  if(head != null){                // close the cycle: head <-> tail
    head.left  = prev;
    prev.right = head;
  }
  return head;
}

Map<Node, Node> map = new HashMap<>();

/**
 * Returns a deep copy of the connected undirected graph reachable from `node`.
 * DFS that creates the clone BEFORE recursing into neighbors, so cycles don't loop forever.
 */
Node cloneGraph(Node node){
  if(node == null) return null;

  // Already cloned? Reuse to preserve graph identity (no duplicates, cycles preserved).
  if(map.containsKey(node)){
    return map.get(node);

  Node copy = new Node(node.val);
  map.put(node, copy);

  for(Node nb : node.neighbors){
    copy.neighbors.add(cloneGraph(nb));
  }
  return copy;
}

List<List<Integer>> kosarajuSCC(int n, List<Integer>[] graph){
  boolean[] visited = new boolean[n];
  Deque<Integer> finishOrder = new ArrayDeque<>();

  // Pass 1 — record finish order on the original graph
  for(int v = 0; v < n; v++)
    if(!visited[v]) dfsFinish(v, graph, visited, finishOrder);

  // Build transpose (reverse all edges)
  List<Integer>[] transpose = new List[n];
  for(int i = 0; i < n; i++) transpose[i] = new ArrayList<>();
  for(int u = 0; u < n; u++)
    for(int v : graph[u]) transpose[v].add(u);

  // Pass 2 — pop in reverse finish order, DFS on transpose
  Arrays.fill(visited, false);
  List<List<Integer>> components = new ArrayList<>();
  while(!finishOrder.isEmpty()){
    int v = finishOrder.pop();
    if(visited[v]) continue;
    List<Integer> component = new ArrayList<>();
    dfsCollect(v, transpose, visited, component);
    components.add(component);
  }
  return components;
}

private void dfsFinish(int u, List<Integer>[] graph, boolean[] visited, Deque<Integer> finishOrder){
  visited[u] = true;
  for(int v : graph[u])
    if(!visited[v]) dfsFinish(v, graph, visited, finishOrder);
  finishOrder.push(u);                        // push AFTER children done
}

private void dfsCollect(int u, List<Integer>[] transpose, boolean[] visited, List<Integer> component){
  visited[u] = true;
  component.add(u);
  for(int v : transpose[u])
    if(!visited[v]) dfsCollect(v, transpose, visited, component);
}

 * because a future negative number can flip the smallest (most negative) into the largest.

 * Minimum sum path from grid[0][0] to grid[m-1][n-1], moving only right or down.
 * dp[j] = min sum to reach the current row at column `j`. Updated in place
 * row by row: BEFORE assignment dp[j] holds the value from the row above;
 * AFTER assignment it holds the value for the current row.
 */
int minPathSum(int[][] grid){
  int m = grid.length;
  int n = grid[0].length;
  int[] dp = new int[n];

  // Top-left corner is the base case.
  dp[0] = grid[0][0];
  // Top row: only the "come from left" option is available.
  for(int j = 1; j < n; j++){
    dp[j] = dp[j - 1] + grid[0][j];
  }

  // Process each subsequent row, reusing dp as a rolling 1-D DP.
  for(int i = 1; i < m; i++){
    dp[0] += grid[i][0];                              // first column: only "come from above"

    for(int j = 1; j < n; j++){
      int up   = dp[j];                               // value still from PREVIOUS row
      int left = dp[j - 1];                           // value already updated for THIS row
      dp[j] = grid[i][j] + Math.min(up, left);
    }
  }
  return dp[n - 1];
