01Next Permutation Medium

Section · Arrays
LeetCode Article
Problem

Rearrange numbers into the lexicographically next greater permutation. If no such permutation, rearrange to the lowest (sorted ascending). Do it in place.

Mental Model
Find the rightmost "ascent", swap with the just-larger value to its right, then reverse the suffix. The suffix after the swap is in descending order — reversing it makes it the smallest possible tail → smallest jump up.
Other Approaches
Brute Generate all permutations, sort, find input, return next. — O(n! · n).
Better Use C++ STL next_permutation. — Hides the algorithm but is the same complexity.
Optimal Approach
  1. Scan from right, find first i where nums[i] < nums[i+1] ("pivot"). If none → reverse all and return.
  2. Scan from right, find first j where nums[j] > nums[pivot].
  3. Swap pivot with j.
  4. Reverse suffix starting at pivot+1.
Visual
flowchart TD
  S["scan right → pivot i (first i with nums[i] < nums[i+1])"] --> P{"pivot found?"}
  P -- no --> R["reverse whole array"]
  P -- yes --> J["scan right → j with nums[j] > nums[pivot]"]
  J --> X["swap(pivot, j)"]
  X --> V["reverse suffix from pivot+1"]
Code (Java)
/**
 * Next lexicographic permutation in place.
 * Trick: the rightmost ascending pair (pivot) is the position we need to nudge upward.
 * Swap pivot with the smallest value greater than it in the suffix, then reverse the
 * suffix so it becomes the smallest possible tail (yielding the *next* perm).
 */
void nextPermutation(int[] numbers){
  int length = numbers.length;

  // 1) find the rightmost index 'pivotIdx' where numbers[pivotIdx] < numbers[pivotIdx+1]
  int pivotIdx = length - 2;
  while(pivotIdx >= 0 && numbers[pivotIdx] >= numbers[pivotIdx + 1]) pivotIdx--;

  if(pivotIdx >= 0){
    // 2) find the rightmost value strictly greater than numbers[pivotIdx]
    int successorIdx = length - 1;
    while(numbers[successorIdx] <= numbers[pivotIdx]) successorIdx--;
    // 3) swap them — now the suffix is still descending
    swap(numbers, pivotIdx, successorIdx);
  }
  // 4) reverse the suffix to make it ascending = smallest possible tail
  reverseRange(numbers, pivotIdx + 1, length - 1);
}

private void swap(int[] a, int i, int j){ int t = a[i]; a[i] = a[j]; a[j] = t; }
private void reverseRange(int[] a, int left, int right){
  while(left < right){ swap(a, left++, right--); }
}
Time O(n)
Space O(1)

02Merge Two Sorted Arrays Without Extra Space Hard

Section · Arrays Part-II
LeetCode Article
Problem

Two sorted arrays A[n], B[m]. Modify them so that A contains the smaller n and B the larger m, each sorted.

Mental Model
Gap method (Shell-sort-style). Compare elements spaced gap = ceil((n+m)/2) apart across the conceptual combined array; halve the gap each pass. No extra memory and faster than swap-and-bubble.
Other Approaches
Brute Copy both into a new array, sort, split back. — O((n+m) log(n+m)) time, O(n+m) space.
Better Two pointers from A[n-1] down and B[0] up: swap any A>B pair, then sort A and B independently. — O((n+m) log) due to per-pass sort.
Optimal Approach (Gap)
  1. Total n + m. Start gap = ceil((n+m)/2).
  2. For each pair (i, i+gap) across the conceptual joined array: if out of order, swap.
  3. Index into A or B based on whether index < n.
  4. Halve the gap (ceil) until gap == 0.
Visual
flowchart TD
  S["gap = ceil((n+m)/2)"] --> L{"gap > 0?"}
  L -- no --> D["done"]
  L -- yes --> P["for i = 0..n+m-1-gap"]
  P --> C{"value(i) > value(i+gap)?"}
  C -- yes --> X["swap across A/B boundary"]
  C -- no --> P
  X --> P
  P --> H["gap = ceil(gap/2); if was 1 → 0"]
  H --> L
Code (Java)
/**
 * Merges two sorted arrays in-place with O(1) extra space using the GAP method.
 * Trick: conceptually concatenate A+B. Compare entries 'gap' positions apart;
 * halving the gap each pass guarantees the combined array becomes sorted.
 * 'valueAt' / 'swapAt' route an index to whichever real array it falls in.
 */
void mergeWithoutExtraSpace(int[] arrayA, int[] arrayB){
  int sizeA = arrayA.length, sizeB = arrayB.length;
  int combined = sizeA + sizeB;

  for(int gap = ceilDiv(combined, 2); gap > 0; gap = (gap == 1 ? 0 : ceilDiv(gap, 2))){
    for(int leftIdx = 0; leftIdx + gap < combined; leftIdx++){
      int rightIdx = leftIdx + gap;
      if(valueAt(leftIdx, arrayA, arrayB, sizeA) > valueAt(rightIdx, arrayA, arrayB, sizeA)){
        swapAt(leftIdx, rightIdx, arrayA, arrayB, sizeA);
      }
    }
  }
}

private int  ceilDiv(int x, int y){ return (x + y - 1) / y; }
private int  valueAt(int idx, int[] a, int[] b, int sizeA){
  return idx < sizeA ? a[idx] : b[idx - sizeA];
}
private void swapAt(int i, int j, int[] a, int[] b, int sizeA){
  int vi = valueAt(i, a, b, sizeA);
  int vj = valueAt(j, a, b, sizeA);
  if(i < sizeA) a[i] = vj; else b[i - sizeA] = vj;
  if(j < sizeA) a[j] = vi; else b[j - sizeA] = vi;
}
Time O((n+m) log(n+m))
Space O(1)

03Majority Element (> N/2 times) Easy

Section · Arrays Part-III
LeetCode Article
Problem

Find the element appearing more than ⌊n/2⌋ times. Assume one always exists.

Mental Model
Boyer-Moore voting. Pair each occurrence of the majority with one non-majority element — the majority still has surplus left. Track a candidate and a counter; reset when count hits zero.
Other Approaches
Brute Count every element with nested loops. — O(n²).
Better HashMap of frequencies. — O(n) time, O(n) space.
Optimal Approach (Boyer-Moore)
  1. candidate = null, count = 0.
  2. For each x: if count == 0 → candidate = x. Then count += (x == candidate ? 1 : -1).
  3. At the end, candidate is the majority (when one is guaranteed).
Visual
flowchart TD
  S["candidate=?, count=0"] --> L["for each x"]
  L --> Z{"count == 0?"}
  Z -- yes --> A["candidate = x"]
  Z -- no --> S2["keep candidate"]
  A --> U["count += (x==candidate?+1:-1)"]
  S2 --> U
  U --> L
  L --> R["return candidate"]
Code (Java)
/**
 * Boyer-Moore majority vote (works only when a >n/2 majority is guaranteed).
 * Trick: each non-majority element can "cancel" exactly one majority occurrence.
 * Since majority appears >n/2 times, it survives all cancellations.
 */
int majorityElement(int[] numbers){
  int leadingCandidate = 0;
  int voteBalance      = 0;

  for(int currentValue : numbers){
    if(voteBalance == 0){
      // no current candidate → adopt this value as our new candidate
      leadingCandidate = currentValue;
    }
    // matching value supports the candidate; differing value cancels one vote
    voteBalance += (currentValue == leadingCandidate) ? 1 : -1;
  }
  return leadingCandidate;
}
Time O(n)
Space O(1)

04Longest Consecutive Sequence Medium

Section · Arrays Part-IV
LeetCode Article
Problem

Given an unsorted array of integers, return the length of the longest sequence of consecutive integers (e.g., {100,4,200,1,3,2} → 4 for 1,2,3,4).

Mental Model
HashSet + "start a streak only at the smallest element". If x-1 is in the set, skip — we'll count this streak when we reach the smallest member. This caps total work at O(n).
Other Approaches
Brute For each x, scan to count x+1, x+2, ... using contains-search. — O(n³) with arrays, O(n²) with set.
Better Sort, then walk counting consecutive runs. — O(n log n).
Optimal Approach
  1. Insert all numbers into a HashSet.
  2. For each x in the set: skip if x-1 in set.
  3. Else walk x, x+1, x+2, ... while in set and count the streak.
  4. Track the longest streak.
Visual
flowchart TD
  S["set = HashSet(nums)"] --> L["for each x in set"]
  L --> C{"contains x-1?"}
  C -- yes --> L
  C -- no --> W["count = 0; while contains x → count++, x++"]
  W --> U["best = max(best, count)"]
  U --> L
Code (Java)
/**
 * Longest run of consecutive integers — O(n) amortised.
 * Trick: only ATTEMPT to extend a streak from its smallest element. If x-1 is in the
 * set, x is not a start. This ensures each element is visited at most twice overall.
 */
int longestConsecutive(int[] numbers){
  Set<Integer> valueSet = new HashSet<>();
  for(int value : numbers) valueSet.add(value);

  int longestStreak = 0;
  for(int candidateStart : valueSet){
    // only attempt to grow a streak from its smallest element
    if(valueSet.contains(candidateStart - 1)) continue;

    int currentValue  = candidateStart;
    int currentLength = 0;
    while(valueSet.contains(currentValue)){
      currentValue++;
      currentLength++;
    }
    longestStreak = Math.max(longestStreak, currentLength);
  }
  return longestStreak;
}
Time O(n)
Space O(n)

05Merge Two Sorted Linked Lists Easy

Section · Linked List
LeetCode Article
Problem

Merge two sorted singly linked lists into one sorted list. Reuse the existing nodes (no copies).

Mental Model
Use a dummy head; weave the two lists with a tail pointer. Always extend the tail to the smaller of the two current heads, then advance that list. The dummy simplifies the "where does the merged list start?" edge case.
Other Approaches
Brute Push everything into an array, sort, rebuild. — O((n+m) log(n+m)), loses the in-place advantage.
Optimal Approach
  1. Create dummy sentinel. tail = dummy.
  2. While both lists non-empty: attach whichever head is smaller, advance that list, advance tail.
  3. Attach the remaining non-empty list (it's already sorted).
  4. Return dummy.next.
Visual
flowchart LR
  D["dummy"] --> T["tail"]
  A["a list head"] --> X["pick smaller"]
  B["b list head"] --> X
  X --> T2["tail.next = picked; advance picked list; tail = tail.next"]
  T2 --> A
  T2 --> B
Code (Java)
/**
 * Merges two sorted linked lists into one sorted list (re-using nodes, in place).
 * Trick: a 'dummyHead' sentinel removes the special-case for "is this the first node?"
 * and a single tail pointer keeps the splice cost O(1) per step.
 */
ListNode mergeTwoLists(ListNode headA, ListNode headB){
  ListNode dummyHead = new ListNode(0);    // sentinel so we always have a previous node
  ListNode tailNode  = dummyHead;

  while(headA != null && headB != null){
    if(headA.val <= headB.val){
      tailNode.next = headA;               // attach smaller, advance that list
      headA = headA.next;
    } else {
      tailNode.next = headB;
      headB = headB.next;
    }
    tailNode = tailNode.next;
  }
  // exactly one of headA/headB is null; the other is already sorted → splice it on
  tailNode.next = (headA != null) ? headA : headB;
  return dummyHead.next;
}
Time O(n+m)
Space O(1)

06Starting Point of Loop in Linked List Medium

Section · Linked List Part-II
LeetCode Article
Problem

If a linked list has a cycle, return the node where the cycle begins. Else return null.

Mental Model
Floyd's Algorithm — Phase 2. After slow & fast meet inside the cycle, restart one pointer at head and walk both at speed 1. They meet exactly at the cycle's entry. (Math: distance from head to cycle start equals distance from meeting point to cycle start when traversed around.)
Other Approaches
Brute HashSet of visited nodes; first repeat is the loop start. — O(n) time, O(n) space.
Optimal Approach
  1. Run Floyd's until slow == fast (or fast falls off → no cycle, return null).
  2. Reset slow = head.
  3. Move both 1 step at a time until they collide again — that node is the cycle start.
Visual
flowchart LR
  H[head] --> A[A] --> B[B] --> S["cycle start"] --> C[C] --> D[D] --> S
  P1["phase 1: slow/fast meet inside cycle"] --> P2["phase 2: slow=head; both step 1 → meet at S"]
Code (Java)
/**
 * Returns the node where the cycle begins, or null if no cycle.
 * Trick: once slow and fast collide inside the cycle, the distance from head to the
 * cycle entry equals the distance from the collision point to the entry — so two
 * walkers moving at speed 1 (from head and from collision) meet exactly at the entry.
 */
ListNode detectCycle(ListNode head){
  ListNode slowWalker = head, fastWalker = head;

  // ---- phase 1: detect any meeting inside the cycle ----
  while(fastWalker != null && fastWalker.next != null){
    slowWalker = slowWalker.next;
    fastWalker = fastWalker.next.next;
    if(slowWalker == fastWalker){
      // ---- phase 2: walk one from head, one from meeting; they meet at the entry ----
      ListNode entryWalker = head;
      while(entryWalker != slowWalker){
        entryWalker = entryWalker.next;
        slowWalker  = slowWalker.next;
      }
      return entryWalker;
    }
  }
  return null;                              // fast fell off → no cycle
}
Time O(n)
Space O(1)

073 Sum Medium

Section · Linked List & Arrays
LeetCode Article
Problem

Return all unique triplets (a, b, c) in the array such that a + b + c = 0.

Mental Model
Sort, fix one, two-pointer the rest. A sorted array makes the "needed sum" monotonic — moving left/right pointers always trims toward the target. Skip duplicates at every level to avoid repeated triplets.
Other Approaches
Brute Three nested loops + HashSet for dedup. — O(n³).
Better Fix first two with loops, find third via HashSet. — O(n²) time, O(n) set.
Optimal Approach
  1. Sort.
  2. Fix index i (skip if duplicate of previous).
  3. Two pointers l = i+1, r = n-1; compute sum.
  4. If sum < 0 → l++; if > 0 → r--; if 0 → record, advance both past duplicates.
Visual
flowchart TD
  S["sort"] --> I["fix i"]
  I --> P["l = i+1, r = n-1"]
  P --> C{"sum vs 0"}
  C -- "<" --> A["l++"]
  C -- ">" --> B["r--"]
  C -- "=" --> R["record triple, advance past dupes"]
  A --> C
  B --> C
  R --> C
Code (Java)
/**
 * All unique triplets that sum to zero.
 * Trick: sort, fix the smallest element, then converge with two pointers from both ends.
 * Skipping duplicates at each of the three levels eliminates repeated triplets.
 */
List<List<Integer>> threeSum(int[] numbers){
  Arrays.sort(numbers);
  List<List<Integer>> triplets = new ArrayList<>();
  int length = numbers.length;

  for(int firstIdx = 0; firstIdx < length - 2; firstIdx++){
    if(firstIdx > 0 && numbers[firstIdx] == numbers[firstIdx - 1]) continue;   // skip dup first
    int leftPointer  = firstIdx + 1;
    int rightPointer = length - 1;
    while(leftPointer < rightPointer){
      int tripletSum = numbers[firstIdx] + numbers[leftPointer] + numbers[rightPointer];
      if(tripletSum == 0){
        triplets.add(Arrays.asList(numbers[firstIdx], numbers[leftPointer], numbers[rightPointer]));
        leftPointer++; rightPointer--;
        while(leftPointer < rightPointer && numbers[leftPointer]  == numbers[leftPointer - 1])  leftPointer++;
        while(leftPointer < rightPointer && numbers[rightPointer] == numbers[rightPointer + 1]) rightPointer--;
      } else if(tripletSum < 0){
        leftPointer++;
      } else {
        rightPointer--;
      }
    }
  }
  return triplets;
}
Time O(n²)
Space O(1) aux

08Job Sequencing Problem Medium

Section · Greedy Algorithm
GFG Article
Problem

Each job has a deadline d and profit p. Schedule at most one job per time-slot to maximize total profit; every job takes 1 unit of time.

Mental Model
Sort by profit DESC and greedily place each job into the latest free slot ≤ its deadline. Latest slot keeps earlier slots open for less-flexible (smaller-deadline) jobs we'll see next.
Other Approaches
Brute Try every permutation and pick the best schedule. — Factorial blow-up.
Better Use a Disjoint-Set "find next free slot" — O(N α(N)); more compact than a boolean scan.
Optimal Approach
  1. Sort jobs by profit descending.
  2. Let maxDeadline = max d; create boolean slotTaken[maxDeadline+1].
  3. For each job, scan slots from min(maxDeadline, d) down to 1; first free slot → place; record profit.
  4. If none free → drop the job.
Visual
flowchart TD
  S["sort jobs by profit DESC"] --> L["for each job (deadline d, profit p)"]
  L --> F["scan slot from min(maxDeadline,d) down to 1"]
  F --> C{"slot free?"}
  C -- yes --> P["take slot, profit += p"]
  C -- no --> F
  P --> L
Code (Java)
/**
 * Greedy job sequencing: maximise profit with unit-time jobs and per-job deadlines.
 * Trick: sort by profit DESC so higher-profit jobs get first pick; place each job in
 * the LATEST still-free slot ≤ its deadline, leaving earlier slots for tighter jobs.
 */
int[] jobSequencing(int[][] jobs){      // jobs[i] = {jobId, deadline, profit}
  // sort by profit descending
  Arrays.sort(jobs, (a, b) -> Integer.compare(b[2], a[2]));

  int latestDeadline = 0;
  for(int[] job : jobs) latestDeadline = Math.max(latestDeadline, job[1]);

  boolean[] slotTaken = new boolean[latestDeadline + 1];   // slotTaken[1..latestDeadline]
  int scheduledJobsCount = 0;
  int accumulatedProfit  = 0;

  for(int[] job : jobs){
    int deadline = job[1];
    int profit   = job[2];
    // scan slots from the latest possible one down to 1 — first free slot wins
    for(int slotIdx = Math.min(latestDeadline, deadline); slotIdx >= 1; slotIdx--){
      if(!slotTaken[slotIdx]){
        slotTaken[slotIdx] = true;
        scheduledJobsCount++;
        accumulatedProfit += profit;
        break;
      }
    }
  }
  return new int[]{scheduledJobsCount, accumulatedProfit};
}
Time O(N · maxDeadline)
Space O(maxDeadline)

09Combination Sum I Medium

Section · Recursion
LeetCode Article
Problem

Given a set of distinct positive integers and a target, return every unique combination summing to target. The same number can be picked unlimited times.

Mental Model
Pick-or-move backtracking. At each index either "pick this number" (don't advance index — repeats allowed) or "skip to next index". Recursion's natural ordering gives unique combinations without dedup.
Other Approaches
Brute Try all 2ⁿ subsets and check sums. — Exponential, misses repeats.
Optimal Approach
  1. Recurse with (index, remaining, currentList).
  2. Base: if remaining == 0 → save copy of currentList.
  3. If index == n or remaining < 0 → return.
  4. If nums[index] ≤ remaining → pick: recurse with (index, remaining - nums[index]), then pop.
  5. Skip: recurse with (index + 1, remaining).
Visual
flowchart TD
  N["state (i, rem)"] --> Z{"rem == 0?"}
  Z -- yes --> S["save current"]
  Z -- no --> E{"i < n && rem >= nums[i]?"}
  E -- yes --> P["pick: recurse(i, rem-nums[i])"]
  P --> K["skip: recurse(i+1, rem)"]
  E -- no --> K
Code (Java)
/**
 * All combinations of 'candidates' (unlimited use) summing to 'targetSum'.
 * Trick: 'pick or move on' at each index, KEEPING the index when picking (allows reuse).
 * Moving on means we'll never revisit prior values → combinations stay unique.
 */
List<List<Integer>> combinationSum(int[] candidates, int targetSum){
  List<List<Integer>> allCombinations = new ArrayList<>();
  exploreCombinations(0, targetSum, candidates, new ArrayList<>(), allCombinations);
  return allCombinations;
}

private void exploreCombinations(int currentIdx, int remainingTarget,
                                 int[] candidates, List<Integer> currentCombination,
                                 List<List<Integer>> allCombinations){
  if(remainingTarget == 0){
    allCombinations.add(new ArrayList<>(currentCombination));     // snapshot the winning combo
    return;
  }
  if(currentIdx == candidates.length || remainingTarget < 0) return;

  // option A: pick candidates[currentIdx] (and stay at same index → allows reuse)
  if(candidates[currentIdx] <= remainingTarget){
    currentCombination.add(candidates[currentIdx]);
    exploreCombinations(currentIdx, remainingTarget - candidates[currentIdx],
                        candidates, currentCombination, allCombinations);
    currentCombination.remove(currentCombination.size() - 1);     // backtrack
  }
  // option B: skip this candidate entirely → advance index
  exploreCombinations(currentIdx + 1, remainingTarget,
                      candidates, currentCombination, allCombinations);
}
Time O(2^t)
Space O(t) recursion

10Sudoku Solver Hard

Section · Recursion & Backtracking
LeetCode Article
Problem

Solve a partially-filled 9×9 Sudoku in place. Each digit 1-9 appears exactly once per row, column, and 3×3 sub-grid.

Mental Model
Backtrack cell-by-cell: try 1-9, recurse, undo on failure. Validity is a constant-time check per digit (scan row, column, and 3×3 block). Returning a boolean lets the recursion short-circuit when a solution is found.
Other Approaches
Brute Bit-mask the whole board state and enumerate — astronomically slow without pruning.
Better Pre-build bitmasks for each row / column / box → O(1) "is digit available" — clearer to scale, but same recursion shape.
Optimal Approach
  1. Find the next empty cell. If none → board is solved → return true.
  2. For digit d = 1..9: if placing d is valid (row/col/box clean), place it.
  3. Recurse. If solved → return true. Else undo (set to '.').
  4. If no digit works → return false (caller backtracks).
Visual
flowchart TD
  S["find next empty cell"] --> N{"found?"}
  N -- no --> W["solved → true"]
  N -- yes --> D["for d in 1..9"]
  D --> V{"valid in row/col/box?"}
  V -- yes --> P["place d, recurse"]
  P --> R{"recurse solved?"}
  R -- yes --> W
  R -- no --> U["undo"]
  U --> D
  V -- no --> D
  D --> F["return false (caller backtracks)"]
Code (Java)
/**
 * Solves a 9x9 Sudoku in place using backtracking.
 * Trick: O(1) validity check per (digit, cell) by scanning the row, column, and
 * 3x3 box — small enough that the recursion's branch factor stays manageable.
 */
boolean solveSudoku(char[][] board){
  for(int row = 0; row < 9; row++){
    for(int col = 0; col < 9; col++){
      if(board[row][col] != '.') continue;
      for(char candidateDigit = '1'; candidateDigit <= '9'; candidateDigit++){
        if(!isPlacementValid(board, row, col, candidateDigit)) continue;
        board[row][col] = candidateDigit;                     // try
        if(solveSudoku(board)) return true;                   // succeed → propagate up
        board[row][col] = '.';                                // backtrack
      }
      return false;                                           // no digit fits → backtrack
    }
  }
  return true;                                                // no empty cells → solved
}

private boolean isPlacementValid(char[][] board, int row, int col, char digit){
  int boxRowStart = (row / 3) * 3;
  int boxColStart = (col / 3) * 3;
  for(int i = 0; i < 9; i++){
    if(board[row][i] == digit) return false;                  // row conflict
    if(board[i][col] == digit) return false;                  // col conflict
    if(board[boxRowStart + i / 3][boxColStart + i % 3] == digit) return false;   // box conflict
  }
  return true;
}
Time O(9^empties) worst
Space O(81) recursion

11Single Element in Sorted Array Medium

Section · Binary Search
LeetCode Article
Problem

Sorted array where every element appears exactly twice except one. Find the lone element in O(log n).

Mental Model
Pairs sit at (even, odd) indices BEFORE the single element and (odd, even) AFTER it. Binary search using this parity invariant: check if mid still has its pair partner; if yes, the unique element is to the right.
Other Approaches
Brute Linear scan checking neighbours. — O(n).
Better XOR all elements (pairs cancel, single remains). — O(n), still linear.
Optimal Approach
  1. lo = 0, hi = n - 2 (avoid out-of-bounds for mid+1).
  2. mid = (lo+hi)/2. Force mid to even via mid -= mid & 1.
  3. If nums[mid] == nums[mid+1] → pair intact, unique is right: lo = mid + 2.
  4. Else → unique is at mid or left: hi = mid - 2.
  5. Return nums[lo].
Visual
flowchart TD
  S["lo=0, hi=n-2"] --> L{"lo < hi?"}
  L -- no --> R["return nums[lo]"]
  L -- yes --> M["mid = (lo+hi)/2; mid -= mid & 1 → even"]
  M --> C{"nums[mid] == nums[mid+1]?"}
  C -- yes --> A["lo = mid + 2"]
  C -- no --> B["hi = mid"]
  A --> L
  B --> L
Code (Java)
/**
 * Lone element in a sorted array where everything else appears twice.
 * Trick: before the lone element, pairs occupy (even, odd) index pairs;
 * after it, they occupy (odd, even). Binary search using this parity flip.
 */
int singleNonDuplicate(int[] sortedNumbers){
  int leftIdx = 0, rightIdx = sortedNumbers.length - 2;
  while(leftIdx < rightIdx){
    int midIdx = (leftIdx + rightIdx) / 2;
    midIdx -= (midIdx & 1);                       // ensure midIdx is even → pair starts here normally
    if(sortedNumbers[midIdx] == sortedNumbers[midIdx + 1]){
      // pair invariant intact → lone element lies strictly to the right
      leftIdx = midIdx + 2;
    } else {
      // pair invariant broken → lone element is at midIdx or to its left
      rightIdx = midIdx;
    }
  }
  return sortedNumbers[leftIdx];
}
Time O(log n)
Space O(1)

/**
 * Top k most frequent elements via a size-k min-heap on frequency.
 * Trick: a min-heap of (value, frequency) of size at most k always exposes the
 * smallest-frequency kept candidate at its root — easy to evict for newcomers.
 */
int[] topKFrequent(int[] numbers, int k){
  // 1) count frequencies
  Map<Integer, Integer> frequencyByValue = new HashMap<>();
  for(int value : numbers) frequencyByValue.merge(value, 1, Integer::sum);

  // 2) min-heap of (value, frequency) ordered by frequency ascending

  int[] topKValues = new int[k];
  for(int i = 0; i < k; i++) topKValues[i] = topKMinHeap.poll()[0];

/**
 * Stack-on-a-Queue using a single queue and an O(n) push.
 * Trick: after enqueuing x, cycle every prior element to the back so x sits at front.
 * Now pop()/top() are simple queue front operations — O(1).

class StackUsingQueue {

14Sliding Window Maximum Hard

Section · Stack & Queue Part-II
LeetCode Article
Problem

Given array nums and window size k, return the max within every window of size k.

Mental Model
Monotonic deque of indices, kept strictly decreasing by value. The front is always the current window's max. When a new element arrives, pop smaller-or-equal indices from the back (they can never be the max again).
Other Approaches
Brute For each window scan k elements. — O(n·k).
Better Max-heap of (value, index); pop stale roots. — O(n log k).
Optimal Approach
  1. Deque of indices, front = max of current window.
  2. For each index i: while back's value ≤ nums[i] → pop back.
  3. Push i to back.
  4. If front index ≤ i - k → pop front (out of window).
  5. Once i ≥ k-1, record nums[front] as the window's max.
Visual
flowchart TD
  L["for each i in nums"] --> P["while deque back's value <= nums[i]: pop back"]
  P --> A["push i to back"]
  A --> E["if front idx <= i - k: pop front"]
  E --> R{"i >= k-1?"}
  R -- yes --> O["output nums[front]"]
  R -- no --> L
  O --> L
Code (Java)
/**
 * Sliding window maximum in O(n) using a monotonic deque of indices.
 * Trick: the deque is kept strictly decreasing by value. Smaller-or-equal trailing
 * indices can NEVER be the max while this larger element is still in the window,
 * so we discard them as soon as we see the new larger value.
 */
int[] maxSlidingWindow(int[] numbers, int windowSize){
  int length = numbers.length;
  int[] windowMaxes = new int[length - windowSize + 1];
  Deque<Integer> monotonicIndexDeque = new ArrayDeque<>();   // stores INDICES, values strictly decreasing

  for(int currentIdx = 0; currentIdx < length; currentIdx++){
    // 1) evict smaller-or-equal trailing values — they can't be the max while current is around
    while(!monotonicIndexDeque.isEmpty()
          && numbers[monotonicIndexDeque.peekLast()] <= numbers[currentIdx]){
      monotonicIndexDeque.pollLast();
    }
    monotonicIndexDeque.addLast(currentIdx);

    // 2) evict front if it has slid out of the window
    if(monotonicIndexDeque.peekFirst() <= currentIdx - windowSize){
      monotonicIndexDeque.pollFirst();
    }

    // 3) once we have a full window, front is always its max
    if(currentIdx >= windowSize - 1){
      windowMaxes[currentIdx - windowSize + 1] = numbers[monotonicIndexDeque.peekFirst()];
    }
  }
  return windowMaxes;
}

Space O(k)

15Longest Substring Without Repeating Characters Medium

Problem

Given a string, return the length of the longest substring containing only distinct characters.

  2. For each right: if map.containsKey(s[right]) and map[s[right]] ≥ left → left = map[s[right]] + 1.

/**
 * Longest substring with all distinct characters — one pass O(n).
 * Trick: keep a HashMap of char → last seen index. When the new char's last position
 * is inside the current window, jump the left edge to right after that position.
 */
int lengthOfLongestSubstring(String text){
  Map<Character, Integer> lastSeenIndexOf = new HashMap<>();
  int windowLeftIdx = 0;
  int longestLength = 0;

      windowLeftIdx = lastSeenIndexOf.get(currentChar) + 1;

    lastSeenIndexOf.put(currentChar, windowRightIdx);
    longestLength = Math.max(longestLength, windowRightIdx - windowLeftIdx + 1);
  }
  return longestLength;
}

 * Minimum front-insertions to make 's' a palindrome.
 * Trick: the longest palindromic PREFIX of s equals the longest prefix of s that is
 * also a suffix of reverse(s). KMP's LPS on (s + '#' + reverse(s)) gives that length.

int minCharsToAddFront(String text){

  int[] longestPrefixSuffix = buildLps(separatorBlocked);
  int longestPalindromicPrefixLen = longestPrefixSuffix[longestPrefixSuffix.length - 1];

      currentPrefixLen++;
      lps[i++] = currentPrefixLen;
    } else if(currentPrefixLen > 0){
      currentPrefixLen = lps[currentPrefixLen - 1];
    } else {
      lps[i++] = 0;
    }
  }
  return lps;
}

17Postorder Traversal (Iterative) Medium

LeetCode Article
Problem

Return the postorder (left → right → root) traversal of a binary tree, iteratively.

Mental Model
Produce reverse postorder (root → right → left) with one stack, then flip. Reverse postorder is just a modified preorder where we push LEFT first, so right pops next.
Other Approaches
Recursion Trivial 3 lines but uses implicit stack. — O(h).
Two-stack variant Same insight, made explicit — do a mirrored preorder (Node → Right → Left) into a second stack, then drain. Reversing that sequence yields Left → Right → Node = postorder. 
List<Integer> postorderTwoStacks(TreeNode root){
  List<Integer> postorderOutput = new ArrayList<>();
  if(root == null) return postorderOutput;

  Deque<TreeNode> workStack   = new ArrayDeque<>();
  Deque<TreeNode> outputStack = new ArrayDeque<>();
  workStack.push(root);

  while(!workStack.isEmpty()){
    TreeNode node = workStack.pop();
    outputStack.push(node);                                  // collect in REVERSE postorder
    if(node.left  != null) workStack.push(node.left);        // mirrored preorder: push left first
    if(node.right != null) workStack.push(node.right);       // … so right is processed first
  }
  while(!outputStack.isEmpty()) postorderOutput.add(outputStack.pop().val);  // drain → true postorder
  return postorderOutput;
}
Optimal Approach
  1. Push root onto stack. Maintain output list.
  2. While stack non-empty: pop, push value to front of output (or add + reverse later).
  3. Push LEFT first, then RIGHT (so right comes out next).

  L -- yes --> R["reverse output → postorder"]
  L -- no --> O["pop → add value to output"]
  O --> A["push left (if any)"]
  A --> B["push right (if any)"]
  B --> L

/**
 * Iterative postorder traversal using one stack.
 * Trick: producing root→right→left with a stack is easy (just push left first so right
 * pops next). Reverse the result list and you have left→right→root = postorder.
 */
List<Integer> postorderTraversal(TreeNode root){
  LinkedList<Integer> reversedPostorder = new LinkedList<>();
  if(root == null) return reversedPostorder;

  Deque<TreeNode> traversalStack = new ArrayDeque<>();
  traversalStack.push(root);
  while(!traversalStack.isEmpty()){
    TreeNode currentNode = traversalStack.pop();
    reversedPostorder.addFirst(currentNode.val);   // prepend → final list is left-right-root
    if(currentNode.left  != null) traversalStack.push(currentNode.left);   // push LEFT first → right pops next
    if(currentNode.right != null) traversalStack.push(currentNode.right);
  }
  return reversedPostorder;
}

Brute For every node compute height(left) + height(right) separately. — O(n²) for skewed trees.
Convention pitfall ⚠ Two competing definitions in the wild:

  • Nodes convention (some Ankit examples / GFG variants): single node → 1; tree {1,2,3} → 3.

/**
 * Diameter (in EDGES) of a binary tree.
 * Trick: combine height-computation and diameter-tracking in one DFS. At each node the
 * longest path THROUGH it is exactly leftHeight + rightHeight.
 */
private int globalDiameterMax = 0;

int diameterOfBinaryTree(TreeNode root){
  globalDiameterMax = 0;
  measureHeight(root);
  return globalDiameterMax;
}

private int measureHeight(TreeNode node){
  if(node == null) return 0;
  int leftSubtreeHeight  = measureHeight(node.left);
  int rightSubtreeHeight = measureHeight(node.right);
  // best path through THIS node spans leftHeight + rightHeight edges
  globalDiameterMax = Math.max(globalDiameterMax, leftSubtreeHeight + rightSubtreeHeight);
  return 1 + Math.max(leftSubtreeHeight, rightSubtreeHeight);
}

Code (Java)
/**
 * Build binary tree from inorder + postorder.
 * Trick: postorder's LAST element of any segment is the root; inorder splits the
 * remaining elements into the left and right subtrees in O(1) via a HashMap.
 */
TreeNode buildTree(int[] inorder, int[] postorder){
  Map<Integer,Integer> inorderIndexOf = new HashMap<>();
  for(int i = 0; i < inorder.length; i++) inorderIndexOf.put(inorder[i], i);
  return constructSubtree(postorder, 0, postorder.length - 1,
                          inorder,   0, inorder.length  - 1,
                          inorderIndexOf);
}

private TreeNode constructSubtree(int[] postorder, int postStart, int postEnd,
                                  int[] inorder,   int inStart,   int inEnd,
                                  Map<Integer,Integer> inorderIndexOf){
  if(postStart > postEnd || inStart > inEnd) return null;

  int rootValue       = postorder[postEnd];                  // postorder's last = root
  int rootInorderIdx  = inorderIndexOf.get(rootValue);
  int leftSubtreeSize = rootInorderIdx - inStart;

  TreeNode root = new TreeNode(rootValue);
  // left subtree comes from the first leftSubtreeSize postorder slots
  root.left  = constructSubtree(postorder, postStart,                       postStart + leftSubtreeSize - 1,
                                inorder,   inStart,                         rootInorderIdx - 1,
                                inorderIndexOf);
  // right subtree comes from the next slots up to postEnd-1 (excluding root)
  root.right = constructSubtree(postorder, postStart + leftSubtreeSize,     postEnd - 1,
                                inorder,   rootInorderIdx + 1,              inEnd,
                                inorderIndexOf);
  return root;
}

 * Trick: inorder traversal of a BST is sorted; with a stack we can short-circuit
 * as soon as we've visited k nodes — no need to traverse the entire tree.
 */
int kthSmallest(TreeNode root, int k){

 * BFS traversal from vertex 0 using a queue and a visited array.

List<Integer> bfsOfGraph(int vertexCount, List<List<Integer>> adjacencyList){

Shortest paths from a source in a directed weighted graph that may have negative edge weights. Detect negative cycles.

Mental Model
Relax every edge V - 1 times — that suffices because any simple shortest path has at most V - 1 edges. One extra pass: if any distance still decreases, a negative cycle is reachable from the source.
Other Approaches
Dijkstra Faster but breaks on negative weights.
SPFA Queue-based optimisation; usually faster in practice, worst case still O(V·E).
Optimal Approach
  1. Initialize dist[] to ∞; dist[src] = 0.
  2. Repeat V - 1 times: for every edge (u, v, w), if dist[u] + w < dist[v] → relax.
  3. One more pass: if any edge can still relax → negative cycle exists.

  R --> E["for each edge (u, v, w)"]
  E --> X{"dist[u] + w < dist[v]?"}
  X -- yes --> A["relax: dist[v] = dist[u] + w"]

Code (Java)
/**
 * Bellman-Ford shortest paths from sourceVertex on a graph with possibly negative weights.
 * Trick: a simple shortest path has at most V-1 edges, so V-1 relaxation rounds are
 * enough. A V-th round that still finds an improvement proves a reachable negative cycle.
 */
int[] bellmanFord(int vertexCount, int[][] edges, int sourceVertex){
  int[] shortestDistance = new int[vertexCount];
  Arrays.fill(shortestDistance, Integer.MAX_VALUE);
  shortestDistance[sourceVertex] = 0;

  // ---- V-1 relaxation rounds ----
  for(int round = 0; round < vertexCount - 1; round++){
    boolean relaxedThisRound = false;
    for(int[] edge : edges){                     // edge = {u, v, w}
      int from = edge[0], to = edge[1], weight = edge[2];
      if(shortestDistance[from] != Integer.MAX_VALUE
         && shortestDistance[from] + weight < shortestDistance[to]){
        shortestDistance[to] = shortestDistance[from] + weight;
        relaxedThisRound = true;
      }
    }
    if(!relaxedThisRound) break;                 // early exit when stable
  }

  // ---- detect negative cycle via one extra pass ----
  for(int[] edge : edges){
    int from = edge[0], to = edge[1], weight = edge[2];
    if(shortestDistance[from] != Integer.MAX_VALUE
       && shortestDistance[from] + weight < shortestDistance[to]){
      return new int[]{-1};                      // sentinel: negative cycle reachable
    }
  }
  return shortestDistance;
}
Time O(V · E)
Space O(V)

25Longest Common Subsequence Medium

Section · Dynamic Programming
LeetCode Article
Problem

Given two strings a and b, return the length of their longest common subsequence.

Mental Model
dp[i][j] = LCS of a[0..i-1] and b[0..j-1]. Match → take diagonal + 1. Mismatch → take max of dropping one character from either side.
Other Approaches
Brute Recursion exploring all 2^(n+m) sub-pairs. — Exponential.
Better Memoized recursion. — O(n·m) time & space.
Optimal Approach (1D rolling DP)
  1. dp[0..m] all zero.
  2. For i = 1..n: keep prevDiag = 0; for j = 1..m: save tmp = dp[j]. If a[i-1] == b[j-1] → dp[j] = prevDiag + 1. Else dp[j] = max(dp[j], dp[j-1]). prevDiag = tmp.
  3. Return dp[m].
Visual
flowchart TD
  C{"a[i-1] == b[j-1]?"} -- yes --> M["dp[i][j] = dp[i-1][j-1] + 1"]
  C -- no --> N["dp[i][j] = max(dp[i-1][j], dp[i][j-1])"]
Code (Java)
/**
 * Longest Common Subsequence length with O(min(n,m)) space (1D rolling DP).
 * Trick: standard 2D DP only ever reads dp[i-1][j-1], dp[i-1][j], dp[i][j-1] — so a
 * single row plus one "previous diagonal" scalar is enough.
 */
int longestCommonSubsequence(String textA, String textB){
  int lenA = textA.length(), lenB = textB.length();
  int[] previousRowDp = new int[lenB + 1];

  for(int i = 1; i <= lenA; i++){
    int previousDiagonalValue = 0;                   // = dp[i-1][j-1] before being overwritten
    for(int j = 1; j <= lenB; j++){
      int aboveValue = previousRowDp[j];             // = dp[i-1][j] right now
      if(textA.charAt(i - 1) == textB.charAt(j - 1)){
        previousRowDp[j] = previousDiagonalValue + 1;
      } else {
        previousRowDp[j] = Math.max(previousRowDp[j], previousRowDp[j - 1]);
      }
      previousDiagonalValue = aboveValue;            // for next column iteration
    }
  }
  return previousRowDp[lenB];
}
Time O(n·m)
Space O(min(n,m))

260/1 Knapsack Medium

Section · DP Part-II
GFG Article
Problem

Given n items with weights and values and a knapsack capacity W, maximize total value carried (each item used at most once).

Mental Model
For every item, decide "take it" or "skip it" — that's the recursion. 1D DP iterated right-to-left over capacity ensures each item is counted at most once.
Other Approaches
Brute Try all 2ⁿ subsets. — Exponential.
Better 2D DP dp[i][w]. — O(n·W) time & space.
Optimal Approach
  1. dp[0..W] = 0.
  2. For each item i: for w = W down to weight[i]: dp[w] = max(dp[w], dp[w - weight[i]] + value[i]).
  3. Return dp[W].
Visual
flowchart TD
  I["dp[0..W] = 0"] --> L["for each item i"]
  L --> R["for w = W down to weight[i]"]
  R --> U["dp[w] = max(dp[w], dp[w - weight[i]] + value[i])"]
  U --> R
  R --> L
  L --> O["return dp[W]"]
Code (Java)
/**
 * 0/1 Knapsack with a 1D rolling DP.
 * Trick: iterating capacity from HIGH to LOW guarantees dp[w - weight[i]] still refers
 * to the "without item i" state — so item i can't be picked twice.
 */
int knapsack01(int[] itemWeights, int[] itemValues, int capacity){
  int[] bestValueAtCapacity = new int[capacity + 1];     // dp[w] after processing items

  for(int itemIdx = 0; itemIdx < itemWeights.length; itemIdx++){
    int weight = itemWeights[itemIdx];
    int value  = itemValues[itemIdx];
    // walk capacity from HIGH → LOW so each item is counted at most once
    for(int currentCapacity = capacity; currentCapacity >= weight; currentCapacity--){
      int valueIfTaken = bestValueAtCapacity[currentCapacity - weight] + value;
      bestValueAtCapacity[currentCapacity] = Math.max(bestValueAtCapacity[currentCapacity], valueIfTaken);
    }
  }
  return bestValueAtCapacity[capacity];
}
Time O(n·W)
Space O(W)

27Number of Distinct Substrings (Trie) Hard

Section · Trie
Coding Ninjas Article
Problem

Count the number of distinct non-empty substrings of s.

Mental Model
Insert every suffix into a Trie; count nodes created. Every distinct substring corresponds to a unique root-to-node path → one new node = one new distinct substring. Add 1 for the empty string if the problem counts it.
Other Approaches
Brute HashSet of all substrings. — O(n²) time, O(n²) space in chars.
Better Suffix Array + LCP. — O(n log n), conceptually heavier.
Optimal Approach (Trie)
  1. For each starting index i, walk s[i..n-1] through the trie.
  2. Whenever a child doesn't exist, create it and increment a counter.
  3. Final count = distinct non-empty substrings (subtract 0 since we don't count root).
Visual
flowchart TD
  R["root"] --> A["a (suffix abc, ab, a)"]
  A --> B["b"]
  B --> C["c"]
  R --> B2["b (suffix bc, b)"]
  B2 --> C2["c"]
  R --> C3["c (suffix c)"]
Code (Java)
/**
 * Counts distinct non-empty substrings using a trie of all suffixes.
 * Trick: every distinct substring corresponds to a unique root-to-node path in the
 * trie. Whenever we have to allocate a fresh child node we've discovered a NEW substring.
 */
int countDistinctSubstrings(String text){
  TrieNode root = new TrieNode();
  int distinctSubstringCount = 0;

  for(int startIdx = 0; startIdx < text.length(); startIdx++){
    TrieNode currentNode = root;
    for(int endIdx = startIdx; endIdx < text.length(); endIdx++){
      int childIdx = text.charAt(endIdx) - 'a';
      if(currentNode.children[childIdx] == null){
        currentNode.children[childIdx] = new TrieNode();
        distinctSubstringCount++;                  // new node = new distinct substring
      }
      currentNode = currentNode.children[childIdx];
    }
  }
  return distinctSubstringCount;
}

private static class TrieNode { TrieNode[] children = new TrieNode[26]; }
Time O(n²)
Space O(n²) trie
