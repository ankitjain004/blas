01Next Permutation Medium

Section · Arrays
LeetCode
Problem

Rearrange numbers into the lexicographically next greater permutation. If no such permutation, rearrange to the lowest (sorted ascending). Do it in place.

Example

Input A: [1,2,3] · Output: [1,3,2]

  1. Scan from right; first index where a[i] < a[i+1] is i=1 (2<3) — the pivot.
  2. From the right, find first j with a[j] > a[1]=2 → j=2; swap → [1,3,2].
  3. Reverse suffix after pivot (length 1, no-op) → [1,3,2].

Input B (edge — already largest): [3,2,1] · Output: [1,2,3]

  1. No pivot exists (strictly decreasing) → reverse the whole array → sorted ascending.
Mental Model
Find the rightmost "ascent", swap with the just-larger value to its right, then reverse the suffix. The suffix after the swap is in descending order — reversing it makes it the smallest possible tail → smallest jump up.
Other Approaches
Brute Generate all permutations, sort, find input, return next. — O(n! · n).
Better Use C++ STL next_permutation. — Hides the algorithm but is the same complexity.
Optimal Approach
  1. Scan from right, find first i where nums[i] < nums[i+1] ("pivot"). If none → reverse all and return.
  2. Scan from right, find first j where nums[j] > nums[pivot].
  3. Swap pivot with j.
  4. Reverse suffix starting at pivot+1.
Visual
flowchart TD
  S["scan right → pivot i (first i with nums[i] < nums[i+1])"] --> P{"pivot found?"}
  P -- no --> R["reverse whole array"]
  P -- yes --> J["scan right → j with nums[j] > nums[pivot]"]
  J --> X["swap(pivot, j)"]
  X --> V["reverse suffix from pivot+1"]
Code (Java)
/**
 * Next lexicographic permutation in place.
 * Trick: the rightmost ascending pair (pivot) is the position we need to nudge upward.
 * Swap pivot with the smallest value greater than it in the suffix, then reverse the
 * suffix so it becomes the smallest possible tail (yielding the *next* perm).
 */
void nextPermutation(int[] nums){
  int n = nums.length;

  // 1) find the rightmost index 'i' where nums[i] < nums[i+1]
  int i = n - 2;
  while(i >= 0 && nums[i] >= nums[i + 1]) i--;

  if(i >= 0){
    // 2) find the rightmost value strictly greater than nums[i]
    int j = n - 1;
    while(nums[j] <= nums[i]) j--;
    // 3) swap them — now the suffix is still descending
    swap(nums, i, j);
  }
  // 4) reverse the suffix to make it ascending = smallest possible tail
  reverseRange(nums, i + 1, n - 1);
}

private void swap(int[] a, int i, int j){ int t = a[i]; a[i] = a[j]; a[j] = t; }
private void reverseRange(int[] a, int left, int right){
  while(left < right){ swap(a, left++, right--); }
}
Time O(n)
Space O(1)

02Merge Two Sorted Arrays Without Extra Space Hard

Section · Arrays Part-II
LeetCode
Problem

Two sorted arrays A[n], B[m]. Modify them so that A contains the smaller n and B the larger m, each sorted.

Example

Input A: A=[1,4,7,8,10], B=[2,3,9] · Output: A=[1,2,3,4,7], B=[8,9,10]

  1. Use a Shell-sort gap sequence over the conceptual concatenation A+B; gap = ceil((n+m)/2), then halve.
  2. For each pair (i, i+gap): if left>right, swap. Cells beyond n live in B[index-n].
  3. After all gaps reach 0, the conceptual array is sorted globally → first n in A, last m in B.

Input B (edge — already split): A=[1,2,3], B=[4,5] · Output: unchanged

  1. Every gap-pair check finds left<right; no swaps; algorithm exits in O((n+m) log(n+m)) work but mutates nothing.
Mental Model
Gap method (Shell-sort-style). Compare elements spaced gap = ceil((n+m)/2) apart across the conceptual combined array; halve the gap each pass. No extra memory and faster than swap-and-bubble.
Other Approaches
Brute Copy both into a new array, sort, split back. — O((n+m) log(n+m)) time, O(n+m) space.
Better Two pointers from A[n-1] down and B[0] up: swap any A>B pair, then sort A and B independently. — O((n+m) log) due to per-pass sort.
Optimal Approach (Gap)
  1. Total n + m. Start gap = ceil((n+m)/2).
  2. For each pair (i, i+gap) across the conceptual joined array: if out of order, swap.
  3. Index into A or B based on whether index < n.
  4. Halve the gap (ceil) until gap == 0.
Visual
flowchart TD
  S["gap = ceil((n+m)/2)"] --> L{"gap > 0?"}
  L -- no --> D["done"]
  L -- yes --> P["for i = 0..n+m-1-gap"]
  P --> C{"value(i) > value(i+gap)?"}
  C -- yes --> X["swap across A/B boundary"]
  C -- no --> P
  X --> P
  P --> H["gap = ceil(gap/2); if was 1 → 0"]
  H --> L
Code (Java)
/**
 * Merges two sorted arrays in-place with O(1) extra space using the GAP method.
 * Trick: conceptually concatenate A+B. Compare entries 'gap' positions apart;
 * halving the gap each pass guarantees the combined array becomes sorted.
 * 'valueAt' / 'swapAt' route an index to whichever real array it falls in.
 */
void mergeWithoutExtraSpace(int[] a, int[] b){
  int n = a.length, m = b.length;
  int total = n + m;

  for(int gap = ceilDiv(total, 2); gap > 0; gap = (gap == 1 ? 0 : ceilDiv(gap, 2))){
    for(int i = 0; i + gap < total; i++){
      int j = i + gap;
      if(valueAt(i, a, b, n) > valueAt(j, a, b, n)){
        swapAt(i, j, a, b, n);
      }
    }
  }
}

private int  ceilDiv(int x, int y){ return (x + y - 1) / y; }
private int  valueAt(int idx, int[] a, int[] b, int n){
  return idx < n ? a[idx] : b[idx - n];
}
private void swapAt(int i, int j, int[] a, int[] b, int n){
  int vi = valueAt(i, a, b, n);
  int vj = valueAt(j, a, b, n);
  if(i < n) a[i] = vj; else b[i - n] = vj;
  if(j < n) a[j] = vi; else b[j - n] = vi;
}
Time O((n+m) log(n+m))
Space O(1)

03Majority Element (> N/2 times) Easy

Section · Arrays Part-III
LeetCode
Problem

Find the element appearing more than ⌊n/2⌋ times. Assume one always exists.

Example

Input A: [2,2,1,1,1,2,2] · Output: 2

  1. Boyer-Moore: candidate cand, count c=0.
  2. 2: c=0 → cand=2, c=1. 2: same → c=2. 1: diff → c=1. 1: diff → c=0. 1: c=0 → cand=1, c=1. 2: diff → c=0. 2: c=0 → cand=2, c=1.
  3. Final cand=2. (Optional verify pass confirms count > 7/2.)

Input B (edge — single element): [5] · Output: 5

  1. cand=5, c=1; loop ends → return 5.
Mental Model
Boyer-Moore voting. Pair each occurrence of the majority with one non-majority element — the majority still has surplus left. Track a candidate and a counter; reset when count hits zero.
Other Approaches
Brute Count every element with nested loops. — O(n²).
Better HashMap of frequencies. — O(n) time, O(n) space.
Optimal Approach (Boyer-Moore)
  1. candidate = null, count = 0.
  2. For each x: if count == 0 → candidate = x. Then count += (x == candidate ? 1 : -1).
  3. At the end, candidate is the majority (when one is guaranteed).
Visual
flowchart TD
  S["candidate=?, count=0"] --> L["for each x"]
  L --> Z{"count == 0?"}
  Z -- yes --> A["candidate = x"]
  Z -- no --> S2["keep candidate"]
  A --> U["count += (x==candidate?+1:-1)"]
  S2 --> U
  U --> L
  L --> R["return candidate"]
Code (Java)
/**
 * Boyer-Moore majority vote (works only when a >n/2 majority is guaranteed).
 * Trick: each non-majority element can "cancel" exactly one majority occurrence.
 * Since majority appears >n/2 times, it survives all cancellations.
 */
int majorityElement(int[] nums){
  int cand  = 0;
  int count = 0;

  for(int x : nums){
    if(count == 0){
      // no current candidate → adopt this value as our new candidate
      cand = x;
    }
    // matching value supports the candidate; differing value cancels one vote
    count += (x == cand) ? 1 : -1;
  }
  return cand;
}
Time O(n)
Space O(1)

04Longest Consecutive Sequence Medium

Section · Arrays Part-IV
LeetCode
Problem

Given an unsorted array of integers, return the length of the longest sequence of consecutive integers (e.g., {100,4,200,1,3,2} → 4 for 1,2,3,4).

Example

Input A: [100, 4, 200, 1, 3, 2] · Output: 4

  1. Insert all into a HashSet: {100, 4, 200, 1, 3, 2}.
  2. For each x in set, only start counting if x-1 is not in the set (true sequence start).
  3. Starts: 100 (no 99), 1 (no 0), 200 (no 199). Walk 1→2→3→4 → length 4. Others give 1.

Input B (edge — duplicates): [1, 2, 0, 1] · Output: 3

  1. Set deduplicates to {0,1,2}; only start at 0 → 0,1,2 → length 3.
Mental Model
HashSet + "start a streak only at the smallest element". If x-1 is in the set, skip — we'll count this streak when we reach the smallest member. This caps total work at O(n).
Other Approaches
Brute For each x, scan to count x+1, x+2, ... using contains-search. — O(n³) with arrays, O(n²) with set.
Better Sort, then walk counting consecutive runs. — O(n log n).
Optimal Approach
  1. Insert all numbers into a HashSet.
  2. For each x in the set: skip if x-1 in set.
  3. Else walk x, x+1, x+2, ... while in set and count the streak.
  4. Track the longest streak.
Visual
flowchart TD
  S["set = HashSet(nums)"] --> L["for each x in set"]
  L --> C{"contains x-1?"}
  C -- yes --> L
  C -- no --> W["count = 0; while contains x → count++, x++"]
  W --> U["best = max(best, count)"]
  U --> L
Code (Java)
/**
 * Longest run of consecutive integers — O(n) amortised.
 * Trick: only ATTEMPT to extend a streak from its smallest element. If x-1 is in the
 * set, x is not a start. This ensures each element is visited at most twice overall.
 */
int longestConsecutive(int[] nums){
  Set<Integer> set = new HashSet<>();
  for(int x : nums) set.add(x);

  int best = 0;
  for(int x : set){
    // only attempt to grow a streak from its smallest element
    if(set.contains(x - 1)) continue;

    int cur = x;
    int len = 0;
    while(set.contains(cur)){
      cur++;
      len++;
    }
    best = Math.max(best, len);
  }
  return best;
}
Time O(n)
Space O(n)

05Merge Two Sorted Linked Lists Easy

Section · Linked List
LeetCode
Problem

Merge two sorted singly linked lists into one sorted list. Reuse the existing nodes (no copies).

Example

Input A: L1: 1→2→4; L2: 1→3→4 · Output: 1→1→2→3→4→4

  1. Dummy head; tail pointer. Compare heads; splice the smaller and advance.
  2. Steps: pick L1.1 → L2.1 → L1.2 → L2.3 → L1.4 → L2.4 → done.
  3. Return dummy.next.

Input B (edge — one empty): L1: null; L2: 5→7 · Output: 5→7

  1. While loop never enters; remaining-tail attach picks up L2 directly.
Mental Model
Use a dummy head; weave the two lists with a tail pointer. Always extend the tail to the smaller of the two current heads, then advance that list. The dummy simplifies the "where does the merged list start?" edge case.
Other Approaches
Brute Push everything into an array, sort, rebuild. — O((n+m) log(n+m)), loses the in-place advantage.
Optimal Approach
  1. Create dummy sentinel. tail = dummy.
  2. While both lists non-empty: attach whichever head is smaller, advance that list, advance tail.
  3. Attach the remaining non-empty list (it's already sorted).
  4. Return dummy.next.
Visual
flowchart LR
  D["dummy"] --> T["tail"]
  A["a list head"] --> X["pick smaller"]
  B["b list head"] --> X
  X --> T2["tail.next = picked; advance picked list; tail = tail.next"]
  T2 --> A
  T2 --> B
Code (Java)
/**
 * Merges two sorted linked lists into one sorted list (re-using nodes, in place).
 * Trick: a 'dummy' sentinel removes the special-case for "is this the first node?"
 * and a single tail pointer keeps the splice cost O(1) per step.
 */
ListNode mergeTwoLists(ListNode a, ListNode b){
  ListNode dummy = new ListNode(0);        // sentinel so we always have a previous node
  ListNode tail  = dummy;

  while(a != null && b != null){
    if(a.val <= b.val){
      tail.next = a;                       // attach smaller, advance that list
      a = a.next;
    } else {
      tail.next = b;
      b = b.next;
    }
    tail = tail.next;
  }
  // exactly one of a/b is null; the other is already sorted → splice it on
  tail.next = (a != null) ? a : b;
  return dummy.next;
}
Time O(n+m)
Space O(1)

06Starting Point of Loop in Linked List Medium

Section · Linked List Part-II
LeetCode
Problem

If a linked list has a cycle, return the node where the cycle begins. Else return null.

Example

Input A: 3→2→0→-4 with -4.next → node 2 · Output: node 2

  1. Phase 1 — Floyd's: slow/fast collide inside the cycle (e.g., at node -4).
  2. Phase 2: reset slow to head; advance both 1 step at a time. Math says they meet exactly at the cycle entrance.
  3. Walk: head=3, slow=-4 → 2, slow=2,fast=2 → collide → return node 2.

Input B (edge — acyclic): 1→2→3 · Output: null

  1. Phase 1 fast pointer reaches null first → return null without entering phase 2.
Mental Model
Floyd's Algorithm — Phase 2. After slow & fast meet inside the cycle, restart one pointer at head and walk both at speed 1. They meet exactly at the cycle's entry. (Math: distance from head to cycle start equals distance from meeting point to cycle start when traversed around.)
Other Approaches
Brute HashSet of visited nodes; first repeat is the loop start. — O(n) time, O(n) space.
Optimal Approach
  1. Run Floyd's until slow == fast (or fast falls off → no cycle, return null).
  2. Reset slow = head.
  3. Move both 1 step at a time until they collide again — that node is the cycle start.
Visual
flowchart LR
  H[head] --> A[A] --> B[B] --> S["cycle start"] --> C[C] --> D[D] --> S
  P1["phase 1: slow/fast meet inside cycle"] --> P2["phase 2: slow=head; both step 1 → meet at S"]
Code (Java)
/**
 * Returns the node where the cycle begins, or null if no cycle.
 * Trick: once slow and fast collide inside the cycle, the distance from head to the
 * cycle entry equals the distance from the collision point to the entry — so two
 * walkers moving at speed 1 (from head and from collision) meet exactly at the entry.
 */
ListNode detectCycle(ListNode head){
  ListNode slow = head, fast = head;

  // ---- phase 1: detect any meeting inside the cycle ----
  while(fast != null && fast.next != null){
    slow = slow.next;
    fast = fast.next.next;
    if(slow == fast){
      // ---- phase 2: walk one from head, one from meeting; they meet at the entry ----
      ListNode p = head;
      while(p != slow){
        p    = p.next;
        slow = slow.next;
      }
      return p;
    }
  }
  return null;                              // fast fell off → no cycle
}
Time O(n)
Space O(1)

073 Sum Medium

Section · Linked List & Arrays
LeetCode
Problem

Return all unique triplets (a, b, c) in the array such that a + b + c = 0.

Example

Input A: nums=[-1,0,1,2,-1,-4] · Output: [[-1,-1,2], [-1,0,1]]

  1. Sort → [-4,-1,-1,0,1,2].
  2. Fix i=0 (-4); two-pointer never finds sum 4 in remainder.
  3. Fix i=1 (-1); l=2 (-1), r=5 (2) → sum=0 → record [-1,-1,2]; advance both, skip duplicates.
  4. l=3 (0), r=4 (1) → sum=0 → record [-1,0,1].
  5. Skip i=2 (duplicate -1); i=3 (0): l=4,r=5 → 0+1+2=3 → no.

Input B (edge — all zeros): [0,0,0,0] · Output: [[0,0,0]]

  1. Sorted same; i=0: l=1, r=3 → 0+0+0 → record. Skip duplicates at all levels → single triplet.
Mental Model
Sort, fix one, two-pointer the rest. A sorted array makes the "needed sum" monotonic — moving left/right pointers always trims toward the target. Skip duplicates at every level to avoid repeated triplets.
Other Approaches
Brute Three nested loops + HashSet for dedup. — O(n³).
Better Fix first two with loops, find third via HashSet. — O(n²) time, O(n) set.
Optimal Approach
  1. Sort.
  2. Fix index i (skip if duplicate of previous).
  3. Two pointers l = i+1, r = n-1; compute sum.
  4. If sum < 0 → l++; if > 0 → r--; if 0 → record, advance both past duplicates.
Visual
flowchart TD
  S["sort"] --> I["fix i"]
  I --> P["l = i+1, r = n-1"]
  P --> C{"sum vs 0"}
  C -- "<" --> A["l++"]
  C -- ">" --> B["r--"]
  C -- "=" --> R["record triple, advance past dupes"]
  A --> C
  B --> C
  R --> C
Code (Java)
/**
 * All unique triplets that sum to zero.
 * Trick: sort, fix the smallest element, then converge with two pointers from both ends.
 * Skipping duplicates at each of the three levels eliminates repeated triplets.
 */
List<List<Integer>> threeSum(int[] nums){
  Arrays.sort(nums);
  List<List<Integer>> res = new ArrayList<>();
  int n = nums.length;

  for(int i = 0; i < n - 2; i++){
    if(i > 0 && nums[i] == nums[i - 1]) continue;   // skip dup first
    int l = i + 1;
    int r = n - 1;
    while(l < r){
      int sum = nums[i] + nums[l] + nums[r];
      if(sum == 0){
        res.add(Arrays.asList(nums[i], nums[l], nums[r]));
        l++; r--;
        while(l < r && nums[l]  == nums[l - 1])  l++;
        while(l < r && nums[r] == nums[r + 1]) r--;
      } else if(sum < 0){
        l++;
      } else {
        r--;
      }
    }
  }
  return res;
}
Time O(n²)
Space O(1) aux

08Job Sequencing Problem Medium

Section · Greedy Algorithm
GFG
Problem

Each job has a deadline d and profit p. Schedule at most one job per time-slot to maximize total profit; every job takes 1 unit of time.

Example

Input A: jobs (id, d, p) = [(1,4,20), (2,1,10), (3,1,40), (4,1,30)] · Output: 2 jobs, profit=60

  1. Sort by profit desc: (3,1,40), (4,1,30), (1,4,20), (2,1,10).
  2. Slot array of size maxDeadline=4, all free.
  3. Job 3 d=1: try slot 1 → free → take, profit=40.
  4. Job 4 d=1: slot 1 taken → skip.
  5. Job 1 d=4: try 4→3→2→1 → slot 4 free → take, profit=60.
  6. Job 2 d=1: slot 1 taken → skip. Total 2 jobs, 60.

Input B (edge — all same deadline=1): [(A,1,5),(B,1,3)] · Output: 1 job, profit=5

  1. Only one slot exists; greedy picks the higher profit and skips the other.
Mental Model
Sort by profit DESC and greedily place each job into the latest free slot ≤ its deadline. Latest slot keeps earlier slots open for less-flexible (smaller-deadline) jobs we'll see next.
Other Approaches
Brute Try every permutation and pick the best schedule. — Factorial blow-up.
Better Use a Disjoint-Set "find next free slot" — O(N α(N)); more compact than a boolean scan.
Optimal Approach
  1. Sort jobs by profit descending.
  2. Let maxDeadline = max d; create boolean slotTaken[maxDeadline+1].
  3. For each job, scan slots from min(maxDeadline, d) down to 1; first free slot → place; record profit.
  4. If none free → drop the job.
Visual
flowchart TD
  S["sort jobs by profit DESC"] --> L["for each job (deadline d, profit p)"]
  L --> F["scan slot from min(maxDeadline,d) down to 1"]
  F --> C{"slot free?"}
  C -- yes --> P["take slot, profit += p"]
  C -- no --> F
  P --> L
Code (Java)
/**
 * Greedy job sequencing: maximise profit with unit-time jobs and per-job deadlines.
 * Trick: sort by profit DESC so higher-profit jobs get first pick; place each job in
 * the LATEST still-free slot ≤ its deadline, leaving earlier slots for tighter jobs.
 */
int[] jobSequencing(int[][] jobs){      // jobs[i] = {jobId, deadline, profit}
  // sort by profit descending
  Arrays.sort(jobs, (a, b) -> Integer.compare(b[2], a[2]));

  int maxDeadline = 0;
  for(int[] job : jobs) maxDeadline = Math.max(maxDeadline, job[1]);

  boolean[] taken = new boolean[maxDeadline + 1];   // taken[1..maxDeadline]
  int count = 0;
  int totalProfit = 0;

  for(int[] job : jobs){
    int d = job[1];
    int p = job[2];
    // scan slots from the latest possible one down to 1 — first free slot wins
    for(int s = Math.min(maxDeadline, d); s >= 1; s--){
      if(!taken[s]){
        taken[s] = true;
        count++;
        totalProfit += p;
        break;
      }
    }
  }
  return new int[]{count, totalProfit};
}
Time O(N · maxDeadline)
Space O(maxDeadline)

09Combination Sum I Medium

Section · Recursion
LeetCode
Problem

Given a set of distinct positive integers and a target, return every unique combination summing to target. The same number can be picked unlimited times.

Example

Input A: candidates=[2,3,6,7], target=7 · Output: [[2,2,3], [7]]

  1. Sort to enable early pruning. Backtrack with index i: at depth d iterate j ≥ i.
  2. Pick 2 → remaining 5; pick 2 again → remaining 3; pick 3 → remaining 0 → record [2,2,3].
  3. Backtrack; pick 3 (remaining 4) → dead-ends. Pick 7 (remaining 0) → record [7].
  4. Reusing the same index allows repeats; advancing prevents duplicate combinations.

Input B (edge — no solution): candidates=[2,4], target=3 · Output: []

  1. Every branch overshoots or stalls → base case never records anything.
Mental Model
Pick-or-move backtracking. At each index either "pick this number" (don't advance index — repeats allowed) or "skip to next index". Recursion's natural ordering gives unique combinations without dedup.
Other Approaches
Brute Try all 2ⁿ subsets and check sums. — Exponential, misses repeats.
Optimal Approach
  1. Recurse with (index, remaining, currentList).
  2. Base: if remaining == 0 → save copy of currentList.
  3. If index == n or remaining < 0 → return.
  4. If nums[index] ≤ remaining → pick: recurse with (index, remaining - nums[index]), then pop.
  5. Skip: recurse with (index + 1, remaining).
Visual
flowchart TD
  N["state (i, rem)"] --> Z{"rem == 0?"}
  Z -- yes --> S["save current"]
  Z -- no --> E{"i < n && rem >= nums[i]?"}
  E -- yes --> P["pick: recurse(i, rem-nums[i])"]
  P --> K["skip: recurse(i+1, rem)"]
  E -- no --> K
Code (Java)
/**
 * All combinations of 'candidates' (unlimited use) summing to 'targetSum'.
 * Trick: 'pick or move on' at each index, KEEPING the index when picking (allows reuse).
 * Moving on means we'll never revisit prior values → combinations stay unique.
 */
List<List<Integer>> combinationSum(int[] candidates, int target){
  List<List<Integer>> res = new ArrayList<>();
  exploreCombinations(0, target, candidates, new ArrayList<>(), res);
  return res;
}

private void exploreCombinations(int i, int rem,
                                 int[] candidates, List<Integer> cur,
                                 List<List<Integer>> res){
  if(rem == 0){
    res.add(new ArrayList<>(cur));     // snapshot the winning combo
    return;
  }
  if(i == candidates.length || rem < 0) return;

  // option A: pick candidates[i] (and stay at same index → allows reuse)
  if(candidates[i] <= rem){
    cur.add(candidates[i]);
    exploreCombinations(i, rem - candidates[i],
                        candidates, cur, res);
    cur.remove(cur.size() - 1);     // backtrack
  }
  // option B: skip this candidate entirely → advance index
  exploreCombinations(i + 1, rem,
                      candidates, cur, res);
}
Time O(2^t)
Space O(t) recursion

 * Solves a 9x9 Sudoku in place using backtracking.
 * Trick: O(1) validity check per (digit, cell) by scanning the row, column, and
 * 3x3 box — small enough that the recursion's branch factor stays manageable.
 */
boolean solveSudoku(char[][] board){
  for(int row = 0; row < 9; row++){
    for(int col = 0; col < 9; col++){
      if(board[row][col] != '.') continue;
      for(char d = '1'; d <= '9'; d++){
        if(!isPlacementValid(board, row, col, d)) continue;
        board[row][col] = d;                                  // try
        if(solveSudoku(board)) return true;                   // succeed → propagate up
        board[row][col] = '.';                                // backtrack
      }
      return false;                                           // no digit fits → backtrack
    }
  }
  return true;                                                // no empty cells → solved
}

private boolean isPlacementValid(char[][] board, int row, int col, char d){
  int boxRow = (row / 3) * 3;
  int boxCol = (col / 3) * 3;

    if(board[row][i] == d) return false;                      // row conflict
    if(board[i][col] == d) return false;                      // col conflict
    if(board[boxRow + i / 3][boxCol + i % 3] == d) return false;   // box conflict
  }
  return true;
}

 * Lone element in a sorted array where everything else appears twice.
 * Trick: before the lone element, pairs sit at (even, odd) indices; after it they
 * shift to (odd, even). Use a closed-interval binary search on this parity invariant.

int singleNonDuplicate(int[] nums){

  int lo = 0, hi = n - 1;

/**
 * Top k most frequent elements via a size-k min-heap on frequency.

/**
 * Stack-on-a-Queue using a single queue and an O(n) push.
 * Trick: after enqueuing x, cycle every prior element to the back so x sits at front.
 * Now pop()/top() are simple queue front operations — O(1).

Problem

Monotonic deque of indices, kept strictly decreasing by value. The front is always the current window's max. When a new element arrives, pop smaller-or-equal indices from the back (they can never be the max again).

Brute For each window scan k elements. — O(n·k).
Better Max-heap of (value, index); pop stale roots. — O(n log k).

  2. For each index i: while back's value ≤ nums[i] → pop back.
  3. Push i to back.
  4. If front index ≤ i - k → pop front (out of window).
  5. Once i ≥ k-1, record nums[front] as the window's max.

Code (Java)
/**
 * Sliding window maximum in O(n) using a monotonic deque of indices.
 * Trick: the deque is kept strictly decreasing by value. Smaller-or-equal trailing
 * indices can NEVER be the max while this larger element is still in the window,
 * so we discard them as soon as we see the new larger value.
 */
int[] maxSlidingWindow(int[] nums, int k){
  int n = nums.length;
  int[] res = new int[n - k + 1];
  Deque<Integer> dq = new ArrayDeque<>();   // stores INDICES, values strictly decreasing

  for(int i = 0; i < n; i++){
    // 1) evict smaller-or-equal trailing values — they can't be the max while current is around
    while(!dq.isEmpty()
          && nums[dq.peekLast()] <= nums[i]){
      dq.pollLast();
    }
    dq.addLast(i);

    // 2) evict front if it has slid out of the window
    if(dq.peekFirst() <= i - k){
      dq.pollFirst();
    }

    // 3) once we have a full window, front is always its max
    if(i >= k - 1){
      res[i - k + 1] = nums[dq.peekFirst()];
    }
  }
  return res;
}

private int best = 0;

  measureHeight(root);
  return best;
}

private int measureHeight(TreeNode node){
  if(node == null) return 0;
  int lh = measureHeight(node.left);
  int rh = measureHeight(node.right);

/**
 * Build binary tree from inorder + postorder.
 * Trick: postorder's LAST element of any segment is the root; inorder splits the
 * remaining elements into the left and right subtrees in O(1) via a HashMap.
 */
TreeNode buildTree(int[] inorder, int[] postorder){
  Map<Integer,Integer> pos = new HashMap<>();
  for(int i = 0; i < inorder.length; i++) pos.put(inorder[i], i);
  return constructSubtree(postorder, 0, postorder.length - 1,
                          inorder,   0, inorder.length  - 1,
                          pos);
}

private TreeNode constructSubtree(int[] postorder, int postStart, int postEnd,
                                  int[] inorder,   int inStart,   int inEnd,
                                  Map<Integer,Integer> pos){
  if(postStart > postEnd || inStart > inEnd) return null;

  int rootVal   = postorder[postEnd];                  // postorder's last = root
  int rootIdx   = pos.get(rootVal);
  int leftSize  = rootIdx - inStart;

  TreeNode root = new TreeNode(rootVal);
  // left subtree comes from the first leftSize postorder slots
  root.left  = constructSubtree(postorder, postStart,                 postStart + leftSize - 1,
                                inorder,   inStart,                   rootIdx - 1,
                                pos);
  // right subtree comes from the next slots up to postEnd-1 (excluding root)
  root.right = constructSubtree(postorder, postStart + leftSize,      postEnd - 1,
                                inorder,   rootIdx + 1,               inEnd,
                                pos);
  return root;
}

/**
 * k-th smallest in a BST via iterative inorder.
 * Trick: inorder traversal of a BST is sorted; with a stack we can short-circuit
 * as soon as we've visited k nodes — no need to traverse the entire tree.
 */
int kthSmallest(TreeNode root, int k){
  Deque<TreeNode> st = new ArrayDeque<>();
  TreeNode cur = root;
  int count = 0;

  while(cur != null || !st.isEmpty()){
    // descend leftmost, recording the ancestors we'll need to revisit
    while(cur != null){

      cur = cur.left;

    TreeNode node = st.pop();

/**
 * BFS traversal from vertex 0 using a queue and a visited array.
 * Mark on ENQUEUE (not dequeue) to avoid pushing the same vertex twice.
 */
List<Integer> bfsOfGraph(int n, List<List<Integer>> adj){
  boolean[] visited = new boolean[n];
  List<Integer> res = new ArrayList<>();

  Deque<Integer> q = new ArrayDeque<>();
  q.offer(0);
  visited[0] = true;

    int u = q.poll();
    res.add(u);
    for(int v : adj.get(u)){
      if(!visited[v]){
        visited[v] = true;        // mark on enqueue → no duplicates
        q.offer(v);
      }
    }
  }
  return res;

/**
 * Bellman-Ford shortest paths from src on a graph with possibly negative weights.
 * Trick: a simple shortest path has at most V-1 edges, so V-1 relaxation rounds are
 * enough. A V-th round that still finds an improvement proves a reachable negative cycle.
 */
int[] bellmanFord(int n, int[][] edges, int src){
  int[] dist = new int[n];
  Arrays.fill(dist, Integer.MAX_VALUE);
  dist[src] = 0;

  // ---- V-1 relaxation rounds ----
  for(int round = 0; round < n - 1; round++){
    boolean changed = false;
    for(int[] edge : edges){                     // edge = {u, v, w}
      int u = edge[0], v = edge[1], w = edge[2];
      if(dist[u] != Integer.MAX_VALUE
         && dist[u] + w < dist[v]){
        dist[v] = dist[u] + w;
        changed = true;
      }
    }
    if(!changed) break;                          // early exit when stable
  }

  // ---- detect negative cycle via one extra pass ----
  for(int[] edge : edges){
    int u = edge[0], v = edge[1], w = edge[2];
    if(dist[u] != Integer.MAX_VALUE
       && dist[u] + w < dist[v]){
      return new int[]{-1};                      // sentinel: negative cycle reachable
    }
  }
  return dist;
}

/**
 * 0/1 Knapsack — foundational 2D tabulation.
 * dp[i][c] = best value using the first i items within capacity c.
 */
int knapsack01(int[] wt, int[] val, int cap){

  int[][] dp = new int[n + 1][cap + 1];        // dp[0][*] = 0 (no items)

  for(int i = 1; i <= n; i++){
    for(int c = 0; c <= cap; c++){
      dp[i][c] = dp[i - 1][c];                  // skip item i
      if(wt[i - 1] <= c){                        // can it fit?
        int take = dp[i - 1][c - wt[i - 1]] + val[i - 1];
        dp[i][c] = Math.max(dp[i][c], take);    // best of skip / take
      }
    }
  }
  return dp[n][cap];
}

  4. Insert "ab": traverses existing "a→b" — 0 new edges → +0.
  5. Insert "b": existing → +0. Total = 4+3+0+0 = 7 distinct substrings: a, b, ab, ba, aba, bab, abab.

Input B (edge — all same chars): s="aaa" · Output: 3

Mental Model
Insert every suffix into a Trie; count nodes created. Every distinct substring corresponds to a unique root-to-node path → one new node = one new distinct substring. Add 1 for the empty string if the problem counts it.
Other Approaches
Brute HashSet of all substrings. — O(n²) time, O(n²) space in chars.
Better Suffix Array + LCP. — O(n log n), conceptually heavier.
Optimal Approach (Trie)
  1. For each starting index i, walk s[i..n-1] through the trie.
  2. Whenever a child doesn't exist, create it and increment a counter.
  3. Final count = distinct non-empty substrings (subtract 0 since we don't count root).

/**
 * Counts distinct non-empty substrings using a trie of all suffixes.
 * Trick: every distinct substring corresponds to a unique root-to-node path in the
 * trie. Whenever we have to allocate a fresh child node we've discovered a NEW substring.
 */
int countDistinctSubstrings(String s){
  TrieNode root = new TrieNode();
  int count = 0;

  for(int i = 0; i < s.length(); i++){
    TrieNode node = root;
    for(int j = i; j < s.length(); j++){
      int c = s.charAt(j) - 'a';
      if(node.children[c] == null){
        node.children[c] = new TrieNode();
        count++;                  // new node = new distinct substring
      }
      node = node.children[c];
    }
  }
  return count;
}

private static class TrieNode { TrieNode[] children = new TrieNode[26]; }
