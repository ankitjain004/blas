01Kadane's Algorithm Medium

Section · Arrays
LeetCode Article
Problem

Find the contiguous subarray with the largest sum and return that sum.

Mental Model
At every index ask: "is the running sum still helping me, or should I start fresh from here?" If the running sum goes negative it can only drag future sums down — reset.
Other Approaches
Brute Try all O(n²) subarrays, sum each → O(n²).
Better Divide & conquer → O(n log n).
Optimal Approach
  1. Walk left → right. Track runningSum (reset to x if it goes negative) and bestSum.
  2. Update bestSum = max(bestSum, runningSum) each step.
Visual
flowchart LR
  S["start: best = -∞, run = 0"] --> L["for each x"]
  L --> A["run = max(x, run + x)"]
  A --> B["best = max(best, run)"]
  B --> L
  L --> O["return best"]
Code (Java)
/**
 * Kadane's algorithm: maximum contiguous subarray sum in O(n).
 * Key insight: a running prefix only helps if it's positive — the moment it dips below
 * zero, restarting from the current element strictly improves any future window.
 */
int maxSubArray(int[] numbers){
  int bestSumSoFar = numbers[0];
  int currentWindowSum = numbers[0];

  for(int idx = 1; idx < numbers.length; idx++){
    int currentValue = numbers[idx];
    // either extend the window or start fresh — whichever is larger
    currentWindowSum = Math.max(currentValue, currentWindowSum + currentValue);
    bestSumSoFar    = Math.max(bestSumSoFar, currentWindowSum);
  }
  return bestSumSoFar;
}
Time O(n)
Space O(1)

02Find the Duplicate Number Medium

Section · Arrays Part-II
LeetCode Article
Problem

Array of n + 1 integers, each in [1, n]. Exactly one number repeats. Find it without modifying the array, in O(1) extra space.

Mental Model
Treat indices as next-pointers — i → nums[i] — and the array becomes a linked list whose cycle entrance is the duplicate. Floyd's tortoise & hare finds that entrance in O(n) time / O(1) space.
Other Approaches
Brute Sort → adjacent equal → O(n log n), mutates array.
Better HashSet → O(n) time, O(n) space.
Optimal Approach (Floyd)
  1. Phase 1: slow = nums[slow]; fast = nums[nums[fast]] — until they meet inside the cycle.
  2. Phase 2: reset slow = 0; move both one step at a time — they meet at the cycle entrance = duplicate.
Visual
flowchart TD
  S["slow = nums[0]; fast = nums[0]"] --> P1["loop: slow=nums[slow], fast=nums[nums[fast]] until equal"]
  P1 --> R["slow = 0"]
  R --> P2["loop: slow=nums[slow], fast=nums[fast] until equal"]
  P2 --> O["meeting point = duplicate"]
Code (Java)
/**
 * Find the unique duplicate among n+1 values in [1, n] using Floyd's cycle detection.
 * Trick: interpreting index → nums[index] turns the array into a linked list with a
 * guaranteed cycle; the cycle's entry node IS the duplicated value.
 */
int findDuplicate(int[] numbers){
  // ---- phase 1: find a meeting point inside the cycle ----
  int slowPointer = numbers[0];
  int fastPointer = numbers[0];
  do {
    slowPointer = numbers[slowPointer];
    fastPointer = numbers[numbers[fastPointer]];
  } while(slowPointer != fastPointer);

  // ---- phase 2: walk one pointer from start; both meet at cycle entrance ----
  slowPointer = numbers[0];
  while(slowPointer != fastPointer){
    slowPointer = numbers[slowPointer];
    fastPointer = numbers[fastPointer];
  }
  return slowPointer;
}
Time O(n)
Space O(1)

03Majority Element (>N/3) Medium

Section · Arrays Part-III
LeetCode Article
Problem

Return all elements that appear strictly more than ⌊n/3⌋ times.

Mental Model
At most 2 such elements can exist. Boyer-Moore generalised: maintain 2 candidates and 2 counters; resolve clashes by decrementing both. Verify the survivors in a second pass.
Other Approaches
Brute Count every element → O(n²).
Better HashMap of frequencies → O(n) time, O(n) space.
Optimal Approach (Boyer-Moore ×2)
  1. Maintain (cand1, count1) and (cand2, count2).
  2. If x matches a cand → increment its count. Else if a counter is 0 → claim that slot. Else → decrement both counts.
  3. Second pass: count actual occurrences of cand1 / cand2; include those with count > n/3.
Visual
flowchart TD
  S["cand1=∅,c1=0; cand2=∅,c2=0"] --> L["for each x"]
  L --> A{"x == cand1?"}
  A -- yes --> A1["c1++"]
  A -- no --> B{"x == cand2?"}
  B -- yes --> B1["c2++"]
  B -- no --> C{"c1 == 0?"}
  C -- yes --> C1["cand1=x, c1=1"]
  C -- no --> D{"c2 == 0?"}
  D -- yes --> D1["cand2=x, c2=1"]
  D -- no --> D2["c1--; c2--"]
Code (Java)
/**
 * Boyer-Moore voting extended to find all elements appearing > n/3 times (at most 2).
 * Two slots are enough: any third "candidate" would constantly cancel out the others
 * given the n/3 threshold.
 */
List<Integer> majorityElementOverNBy3(int[] numbers){
  int firstCandidate = Integer.MIN_VALUE, firstCount = 0;
  int secondCandidate = Integer.MIN_VALUE, secondCount = 0;

  // ---- pass 1: pick up to 2 candidates ----
  for(int value : numbers){
    if(value == firstCandidate)       firstCount++;
    else if(value == secondCandidate) secondCount++;
    else if(firstCount == 0)          { firstCandidate = value;  firstCount = 1; }
    else if(secondCount == 0)         { secondCandidate = value; secondCount = 1; }
    else                              { firstCount--; secondCount--; }   // cancel
  }

  // ---- pass 2: verify counts (>n/3) ----
  firstCount = secondCount = 0;
  for(int value : numbers){
    if(value == firstCandidate)       firstCount++;
    else if(value == secondCandidate) secondCount++;
  }
  int threshold = numbers.length / 3;
  List<Integer> majorityElements = new ArrayList<>();
  if(firstCount  > threshold) majorityElements.add(firstCandidate);
  if(secondCount > threshold) majorityElements.add(secondCandidate);
  return majorityElements;
}
Time O(n)
Space O(1)

04Longest Subarray with Sum K Medium

Section · Arrays Part-IV
GFG Article
Problem

Given an array (may contain negatives) and target k, return length of the longest contiguous subarray whose sum equals k.

Mental Model
If prefix sum at index j is S, then a subarray ending at j has sum k iff some earlier prefix sum equalled S - k. Store the earliest index each prefix sum was seen → longest subarray.
Other Approaches
Brute Try all subarrays → O(n²) or O(n³).
Better (only positives) Sliding window → O(n), but breaks with negatives.
Optimal Approach
  1. HashMap firstSeenIndex of prefixSum → earliest index.
  2. Walk i = 0..n-1: prefix += a[i]. If prefix == k → answer = i+1. Else if map has prefix - k → answer = max(answer, i - map.get(prefix - k)).
  3. Store prefix in map only if absent (keep earliest index).
Visual
flowchart TD
  S["prefix = 0; map = {}; best = 0"] --> L["for i = 0..n-1"]
  L --> A["prefix += a[i]"]
  A --> B{"prefix == k?"}
  B -- yes --> C["best = max(best, i+1)"]
  B -- no --> D{"map has prefix - k?"}
  D -- yes --> E["best = max(best, i - map[prefix-k])"]
  D -- no --> F["skip"]
  C --> G["if !map has prefix → map[prefix] = i"]
  E --> G
  F --> G
Code (Java)
/**
 * Longest subarray with sum K — works for negative numbers via prefix-sum hashing.
 * Trick: if prefix[j] - prefix[i] == k then sum(a[i+1..j]) == k. To maximise length we
 * record only the FIRST occurrence of each prefix sum.
 */
int longestSubarrayWithSumK(int[] numbers, int targetSum){
  Map<Long, Integer> firstSeenIndexByPrefix = new HashMap<>();
  long runningPrefixSum = 0L;
  int longestLengthSoFar = 0;

  for(int idx = 0; idx < numbers.length; idx++){
    runningPrefixSum += numbers[idx];

    if(runningPrefixSum == targetSum){
      longestLengthSoFar = idx + 1;                                  // whole [0..idx]
    }
    Long lookupKey = runningPrefixSum - targetSum;
    if(firstSeenIndexByPrefix.containsKey(lookupKey)){
      longestLengthSoFar = Math.max(longestLengthSoFar, idx - firstSeenIndexByPrefix.get(lookupKey));
    }
    // store only the FIRST time we see this prefix → longest possible window later
    firstSeenIndexByPrefix.putIfAbsent(runningPrefixSum, idx);
  }
  return longestLengthSoFar;
}
Time O(n)
Space O(n)

05Remove Nth Node from End of LL Medium

Section · Linked List
LeetCode Article
Problem

Given the head of a singly linked list, remove the n-th node from the end and return the head.

Mental Model
Two pointers with a head-start of n. When the leader hits null, the trailer sits just before the node to remove → single pass, O(1) space.
Other Approaches
Brute Count length L → walk again to L-n → remove. — Two passes.
Optimal Approach
  1. Dummy node before head (handles head removal).
  2. Advance leader n steps forward.
  3. Walk leader and trailer together until leader.next == null.
  4. trailer.next = trailer.next.next; return dummy.next.
Visual
flowchart LR
  D["dummy"] --> A["1"] --> B["2"] --> C["3"] --> E["4"] --> F["5"] --> N["null"]
  T["trailer (dummy)"] -.-> D
  L["leader → advance n=2 steps"] -.-> B
Code (Java)
/**
 * Remove the n-th node from the end in a single pass using the gap technique.
 * Trick: a leader pointer starts n steps ahead. When the leader reaches the end, the
 * trailer is exactly at the node BEFORE the one to delete — no length count needed.
 */
ListNode removeNthFromEnd(ListNode head, int nFromEnd){
  ListNode dummyBeforeHead = new ListNode(0);
  dummyBeforeHead.next = head;

  ListNode leaderPointer  = dummyBeforeHead;
  ListNode trailerPointer = dummyBeforeHead;

  // give the leader an n-step head start
  for(int step = 0; step < nFromEnd; step++) leaderPointer = leaderPointer.next;

  // walk both until leader is at the LAST node → trailer is just BEFORE the target
  while(leaderPointer.next != null){
    leaderPointer  = leaderPointer.next;
    trailerPointer = trailerPointer.next;
  }
  trailerPointer.next = trailerPointer.next.next;     // splice out the target
  return dummyBeforeHead.next;
}
Time O(L)
Space O(1)

06Palindrome Linked List Medium

Section · Linked List Part-II
LeetCode Article
Problem

Return true if the linked list reads the same forward and backward. Do it in O(n) time, O(1) extra space.

Mental Model
Find the middle, reverse the second half, walk two halves in parallel. If every paired value matches, it's a palindrome.
Other Approaches
Brute Copy values into an array, two-pointer check → O(n) time, O(n) space.
Stack Push first half, pop while walking second → same complexity.
Optimal Approach
  1. Slow/fast to find middle (slow lands at midpoint for odd, just before for even).
  2. Reverse the list starting at slow.next.
  3. Compare from head and from reversed tail in lock-step.
  4. (Optional) restore the second half.
Visual
flowchart LR
  A["1"] --> B["2"] --> C["3"] --> D["2"] --> E["1"]
  M["mid = C; reverse 2→1 after mid"] --> Cmp["compare head & reversed tail"]
Code (Java)
/**
 * O(1)-space palindrome check on a singly linked list.
 * Trick: locate middle (slow/fast), reverse the second half in place, then walk both
 * halves together. If every paired value matches → palindrome.
 */
boolean isPalindrome(ListNode head){
  if(head == null || head.next == null) return true;

  // 1) find middle using slow/fast
  ListNode slowPointer = head, fastPointer = head;
  while(fastPointer.next != null && fastPointer.next.next != null){
    slowPointer = slowPointer.next;
    fastPointer = fastPointer.next.next;
  }

  // 2) reverse second half (starts at slow.next)
  ListNode reversedSecondHalfHead = reverseList(slowPointer.next);

  // 3) compare in lock-step
  ListNode firstHalfPtr  = head;
  ListNode secondHalfPtr = reversedSecondHalfHead;
  while(secondHalfPtr != null){
    if(firstHalfPtr.val != secondHalfPtr.val) return false;
    firstHalfPtr  = firstHalfPtr.next;
    secondHalfPtr = secondHalfPtr.next;
  }
  return true;
}

private ListNode reverseList(ListNode node){
  ListNode previousNode = null;
  while(node != null){
    ListNode nextNode = node.next;
    node.next = previousNode;
    previousNode = node;
    node = nextNode;
  }
  return previousNode;
}
Time O(n)
Space O(1)

07Trapping Rain Water Hard

Section · LL & Arrays
LeetCode Article
Problem

Given n non-negative heights representing an elevation map, compute the amount of water that can be trapped after raining.

Mental Model
Water above index i = min(maxLeft[i], maxRight[i]) − height[i]. Two pointers compute these lazily: the smaller side is always limited by its own running max, so its water is decidable now.
Other Approaches
Brute For each bar scan both sides for max → O(n²).
Better Precompute maxLeft[] and maxRight[] arrays → O(n) time, O(n) space.
Optimal Approach (Two Pointers)
  1. Two pointers left, right at ends; track maxLeft, maxRight.
  2. Move the smaller side inward. If height[left] < height[right]: if height[left] >= maxLeft update maxLeft, else water += maxLeft - height[left]; left++. Symmetric on right.
Visual
flowchart TD
  S["left=0, right=n-1, maxL=0, maxR=0, water=0"] --> L{"left < right?"}
  L -- no --> O["return water"]
  L -- yes --> C{"height[left] < height[right]?"}
  C -- yes --> A{"height[left] >= maxL?"}
  A -- yes --> A1["maxL = height[left]"]
  A -- no --> A2["water += maxL - height[left]"]
  A1 --> M1["left++"] --> L
  A2 --> M1
  C -- no --> B{"height[right] >= maxR?"}
  B -- yes --> B1["maxR = height[right]"]
  B -- no --> B2["water += maxR - height[right]"]
  B1 --> M2["right--"] --> L
  B2 --> M2
Code (Java)
/**
 * Trapping Rain Water in O(n) time, O(1) space via two pointers.
 * Trick: the side with the SMALLER current height is guaranteed to be bounded by its
 * own running max — so we can settle its water contribution immediately and advance.
 */
int trap(int[] heights){
  int leftPointer  = 0;
  int rightPointer = heights.length - 1;
  int maxHeightFromLeft  = 0;
  int maxHeightFromRight = 0;
  int totalTrappedWater  = 0;

  while(leftPointer < rightPointer){
    if(heights[leftPointer] < heights[rightPointer]){
      // left side is the limiting wall
      if(heights[leftPointer] >= maxHeightFromLeft){
        maxHeightFromLeft = heights[leftPointer];           // new tallest left wall
      } else {
        totalTrappedWater += maxHeightFromLeft - heights[leftPointer];
      }
      leftPointer++;
    } else {
      // right side is the limiting wall
      if(heights[rightPointer] >= maxHeightFromRight){
        maxHeightFromRight = heights[rightPointer];
      } else {
        totalTrappedWater += maxHeightFromRight - heights[rightPointer];
      }
      rightPointer--;
    }
  }
  return totalTrappedWater;
}
Time O(n)
Space O(1)

08Fractional Knapsack Medium

Section · Greedy
GFG Article
Problem

Maximise value carried in a knapsack of capacity W when you may take fractions of items.

Mental Model
Always grab the highest "value-per-unit-weight" first. Because fractions are allowed, the local greedy choice is globally optimal — unlike 0/1 knapsack.
Other Approaches
Brute Try all subsets and splits — exponential.
DP Doesn't apply: continuous capacity space.
Optimal Approach
  1. Sort items by value / weight descending.
  2. Take whole items while they fit. Fill remaining capacity with a fraction of the next.
Visual
flowchart TD
  S["sort items by value/weight desc"] --> L["for each item"]
  L --> F{"weight fits in capacity?"}
  F -- yes --> T["take whole; capacity -= weight"]
  F -- no --> P["take fraction = capacity / weight; STOP"]
  T --> L
Code (Java)
/**
 * Fractional Knapsack — greedy on value-per-unit-weight.
 * Trick: because we can split items, the locally best ratio is also globally best.
 * This breaks for 0/1 Knapsack where indivisibility forces DP.
 */
double fractionalKnapsack(int knapsackCapacity, int[] itemWeights, int[] itemValues){
  int itemCount = itemWeights.length;
  Integer[] itemOrderByRatio = new Integer[itemCount];
  for(int i = 0; i < itemCount; i++) itemOrderByRatio[i] = i;

  // sort indices by value/weight DESC
  Arrays.sort(itemOrderByRatio, (a, b) ->
      Double.compare((double) itemValues[b] / itemWeights[b],
                     (double) itemValues[a] / itemWeights[a]));

  double remainingCapacity = knapsackCapacity;
  double totalValueCollected = 0.0;
  for(int idx : itemOrderByRatio){
    if(itemWeights[idx] <= remainingCapacity){
      totalValueCollected += itemValues[idx];
      remainingCapacity   -= itemWeights[idx];
    } else {
      totalValueCollected += itemValues[idx] * (remainingCapacity / itemWeights[idx]);
      break;                                    // bag is now full
    }
  }
  return totalValueCollected;
}
Time O(n log n)
Space O(n)

09Combination Sum II Medium

Section · Recursion
LeetCode Article
Problem

Given candidates (may contain duplicates) and a target, return all unique combinations that sum to target. Each number may be used at most once.

Mental Model
Sort, then at each recursion level skip equal siblings. Sorting groups duplicates, so we explore each distinct value once per level — preventing duplicate combinations without a global set.
Other Approaches
Brute Generate all subsets, dedupe with a HashSet of sorted tuples → exponential + extra memory.
Optimal Approach
  1. Sort candidates ascending.
  2. Backtrack(start): for i from start: if i > start && cand[i] == cand[i-1] skip. If cand[i] > remaining break.
  3. Push, recurse(i+1, remaining - cand[i]), pop.
Visual
flowchart TD
  S["sort candidates"] --> R["recurse(start, remaining)"]
  R --> L["for i = start..n-1"]
  L --> D{"i > start && cand[i] == cand[i-1]?"}
  D -- yes --> L
  D -- no --> P{"cand[i] > remaining?"}
  P -- yes --> O["break (sorted, rest too big)"]
  P -- no --> T["pick → recurse(i+1) → unpick"]
  T --> L
Code (Java)
/**
 * Combination Sum II — every number used at most once, all combinations must be unique.
 * Trick: sort + "skip duplicate siblings at the SAME recursion level". We allow a
 * duplicate value to appear in a combination only via different array positions reached
 * through nested calls, never as a sibling choice at the same depth.
 */
List<List<Integer>> combinationSum2(int[] candidates, int target){
  Arrays.sort(candidates);
  List<List<Integer>> allCombinations = new ArrayList<>();
  backtrack(candidates, 0, target, new ArrayDeque<>(), allCombinations);
  return allCombinations;
}

private void backtrack(int[] candidates, int startIdx, int remainingTarget,
                       Deque<Integer> currentCombination,
                       List<List<Integer>> allCombinations){
  if(remainingTarget == 0){
    allCombinations.add(new ArrayList<>(currentCombination));
    return;
  }
  for(int i = startIdx; i < candidates.length; i++){
    if(i > startIdx && candidates[i] == candidates[i - 1]) continue;   // dup sibling
    if(candidates[i] > remainingTarget) break;                          // sorted → prune

    currentCombination.push(candidates[i]);
    backtrack(candidates, i + 1, remainingTarget - candidates[i], currentCombination, allCombinations);
    currentCombination.pop();
  }
}
Time O(2ⁿ) worst
Space O(target) depth

10M-Coloring Problem Medium

Section · Recursion & Backtracking
GFG Article
Problem

Given an undirected graph and m colors, decide whether you can color every vertex such that no two adjacent vertices share a color.

Mental Model
Backtracking: try each color 1..m on each vertex; before committing, check all already-colored neighbours. Constraint propagation by neighbour scan prunes invalid branches early.
Other Approaches
Brute Enumerate all mⁿ colorings — same theoretical worst case but no pruning.
Optimal Approach
  1. solve(v): if v == n → return true.
  2. For color c in 1..m: if no neighbour has color c → assign c, recurse(v+1); if true → true; else unassign.
  3. Return false.
Visual
flowchart TD
  S["solve(v)"] --> E{"v == n?"}
  E -- yes --> Y["return true"]
  E -- no --> L["for c = 1..m"]
  L --> C{"safe(v, c)?"}
  C -- yes --> A["color[v] = c → solve(v+1)"]
  A --> R{"recursion true?"}
  R -- yes --> Y
  R -- no --> U["color[v] = 0 → next c"]
  C -- no --> L
  L --> F["return false"]
Code (Java)
/**
 * Decides if the graph is m-colorable via backtracking.
 * Trick: try color c at vertex v ONLY if no already-colored neighbour uses c.
 * Pruning at the constraint boundary is what makes this tractable in practice.
 */
boolean graphColoring(boolean[][] adjacencyMatrix, int colorCount){
  int vertexCount = adjacencyMatrix.length;
  int[] assignedColor = new int[vertexCount];               // 0 = uncolored
  return tryColor(0, adjacencyMatrix, assignedColor, colorCount);
}

private boolean tryColor(int vertex, boolean[][] adjacencyMatrix, int[] assignedColor, int colorCount){
  if(vertex == adjacencyMatrix.length) return true;          // all vertices colored ok

  for(int color = 1; color <= colorCount; color++){
    if(isSafe(vertex, color, adjacencyMatrix, assignedColor)){
      assignedColor[vertex] = color;
      if(tryColor(vertex + 1, adjacencyMatrix, assignedColor, colorCount)) return true;
      assignedColor[vertex] = 0;                             // undo + try next color
    }
  }
  return false;
}

private boolean isSafe(int vertex, int color, boolean[][] adjacencyMatrix, int[] assignedColor){
  for(int neighbour = 0; neighbour < adjacencyMatrix.length; neighbour++){
    if(adjacencyMatrix[vertex][neighbour] && assignedColor[neighbour] == color) return false;
  }
  return true;
}
Time O(mⁿ) worst
Space O(n)

11Search in Rotated Sorted Array Medium

Section · Binary Search
LeetCode Article
Problem

Sorted unique array rotated at some pivot. Find target index in O(log n) or return -1.

Mental Model
At every step, one half is guaranteed sorted. Identify which half (compare nums[lo] and nums[mid]), check if target lies in that sorted range, then move there.
Other Approaches
Brute Linear scan → O(n).
Two-step BS Find pivot first, then BS in the right half → also O(log n) but two passes.
Optimal Approach
  1. Standard BS loop. Compute mid.
  2. If nums[lo] <= nums[mid] → left half sorted. If target in [nums[lo], nums[mid]) → hi = mid-1 else lo = mid+1.
  3. Else right half sorted. If target in (nums[mid], nums[hi]] → lo = mid+1 else hi = mid-1.
Visual
flowchart TD
  S["lo, hi"] --> M["mid = (lo+hi)/2"]
  M --> E{"nums[mid] == target?"}
  E -- yes --> O["return mid"]
  E -- no --> H{"nums[lo] <= nums[mid]? (left sorted)"}
  H -- yes --> L{"target in [nums[lo], nums[mid])?"}
  L -- yes --> A["hi = mid-1"]
  L -- no --> B["lo = mid+1"]
  H -- no --> R{"target in (nums[mid], nums[hi]]?"}
  R -- yes --> C["lo = mid+1"]
  R -- no --> D["hi = mid-1"]
Code (Java)
/**
 * Search in a rotated sorted array of distinct values in O(log n).
 * Trick: a rotated sorted array, when split at mid, ALWAYS has one sorted half. Pick
 * the half whose sorted range contains the target.
 */
int searchInRotatedSortedArray(int[] numbers, int target){
  int lo = 0, hi = numbers.length - 1;
  while(lo <= hi){
    int mid = lo + (hi - lo) / 2;
    if(numbers[mid] == target) return mid;

    if(numbers[lo] <= numbers[mid]){
      // left half [lo..mid] is sorted
      if(numbers[lo] <= target && target < numbers[mid]) hi = mid - 1;
      else                                                lo = mid + 1;
    } else {
      // right half [mid..hi] is sorted
      if(numbers[mid] < target && target <= numbers[hi]) lo = mid + 1;
      else                                                hi = mid - 1;
    }
  }
  return -1;
}
Time O(log n)
Space O(1)

12Merge K Sorted Arrays Medium

Section · Heaps
GFG LeetCode (variant)
Problem

Given k sorted arrays, merge them into one sorted array.

Mental Model
Min-heap of (value, sourceArray, indexInArray) entries — pop the smallest, push its successor. The heap stores at most k elements at once → O(N log k).
Other Approaches
Brute Concatenate and sort → O(N log N).
Divide & Conquer Pair-wise merges → also O(N log k).
Optimal Approach
  1. Push the first element of each array into a min-heap.
  2. Pop the smallest; append to output; push the next element from the same source if any.
  3. Repeat until heap empty.
Visual
flowchart TD
  S["seed heap with first of every array"] --> P["pop min (val, arr, idx)"]

/**
 * Merge k sorted arrays into one sorted array using a size-k min-heap.
 * Trick: the global next-smallest must be among the K "frontier" values — one per
 * source. A k-sized heap pops in O(log k); we do N pops total → O(N log k).
 */
int[] mergeKSortedArrays(int[][] sortedArrays){
  PriorityQueue<int[]> frontierMinHeap =
      new PriorityQueue<>((a, b) -> Integer.compare(a[0], b[0]));   // {value, srcIdx, posInSrc}

  int totalLength = 0;
  for(int srcIdx = 0; srcIdx < sortedArrays.length; srcIdx++){
    if(sortedArrays[srcIdx].length > 0){
      frontierMinHeap.offer(new int[]{sortedArrays[srcIdx][0], srcIdx, 0});
    }
    totalLength += sortedArrays[srcIdx].length;
  }

  int[] mergedOutput = new int[totalLength];
  int writeIdx = 0;
  while(!frontierMinHeap.isEmpty()){
    int[] smallestEntry = frontierMinHeap.poll();
    int value = smallestEntry[0], srcIdx = smallestEntry[1], posInSrc = smallestEntry[2];
    mergedOutput[writeIdx++] = value;

    if(posInSrc + 1 < sortedArrays[srcIdx].length){
      frontierMinHeap.offer(new int[]{sortedArrays[srcIdx][posInSrc + 1], srcIdx, posInSrc + 1});
    }
  }
  return mergedOutput;
}

/**
 * FIFO queue built from two LIFO stacks with amortised O(1) operations.
 * Trick: pushes always go to 'input'. Pops/peeks use 'output'. Whenever output is
 * empty we drain input into output — reversing the order so the OLDEST element ends up
 * on top of output, ready to be popped.
 */
class QueueUsingTwoStacks {
  private final Deque<Integer> inputStack  = new ArrayDeque<>();
  private final Deque<Integer> outputStack = new ArrayDeque<>();

  void push(int value){ inputStack.push(value); }

  int pop(){
    moveInputToOutputIfNeeded();
    return outputStack.pop();
  }

  int peek(){
    moveInputToOutputIfNeeded();
    return outputStack.peek();
  }

  boolean empty(){ return inputStack.isEmpty() && outputStack.isEmpty(); }

  private void moveInputToOutputIfNeeded(){
    if(outputStack.isEmpty()){
      while(!inputStack.isEmpty()) outputStack.push(inputStack.pop());   // reverse order
    }
  }
}

14LFU Cache Hard

Section · Stack & Queue Part-II
LeetCode Article
Problem

Mental Model

Code (Java) — skeleton
/**
 * O(1) LFU cache.
 * Layout: keyToNode (HashMap), freqToBucket (HashMap of DoublyLinkedList), running
 * minFreq. Each access detaches a node from its freq bucket, increments freq, appends
 * to the (freq+1) bucket. Eviction targets the OLDEST node in the lowest freq bucket.
 */
class LFUCache {
  private final int capacity;
  private int currentSize = 0;
  private int minimumFrequency = 0;
  private final Map<Integer, Node> keyToNode = new HashMap<>();
  private final Map<Integer, DoublyLinkedList> frequencyToBucket = new HashMap<>();

  LFUCache(int capacity){ this.capacity = capacity; }

  int get(int key){
    Node node = keyToNode.get(key);
    if(node == null) return -1;
    promoteFrequency(node);
    return node.value;
  }

  void put(int key, int value){
    if(capacity == 0) return;
    Node existing = keyToNode.get(key);
    if(existing != null){
      existing.value = value;
      promoteFrequency(existing);
      return;
    }
    if(currentSize == capacity){
      DoublyLinkedList lowestBucket = frequencyToBucket.get(minimumFrequency);
      Node victim = lowestBucket.removeLast();           // LRU among LFU
      keyToNode.remove(victim.key);
      currentSize--;
    }
    Node fresh = new Node(key, value, 1);
    keyToNode.put(key, fresh);
    frequencyToBucket.computeIfAbsent(1, f -> new DoublyLinkedList()).addFirst(fresh);
    minimumFrequency = 1;
    currentSize++;
  }

  private void promoteFrequency(Node node){
    int oldFreq = node.freq;
    DoublyLinkedList oldBucket = frequencyToBucket.get(oldFreq);
    oldBucket.remove(node);
    if(oldBucket.isEmpty() && oldFreq == minimumFrequency) minimumFrequency++;
    node.freq++;
    frequencyToBucket.computeIfAbsent(node.freq, f -> new DoublyLinkedList()).addFirst(node);
  }

  private static class Node {
    int key, value, freq; Node prev, next;
    Node(int k, int v, int f){ key = k; value = v; freq = f; }
  }
  // DoublyLinkedList: addFirst(Node), remove(Node), removeLast(), isEmpty() — all O(1)
  private static class DoublyLinkedList { /* head/tail dummies + O(1) ops */ }
}
Time O(1) per op
Space O(capacity)

17Morris Inorder Traversal Medium

Section · Binary Tree
LeetCode Article
Problem

Inorder traversal in O(1) extra space (no stack, no recursion).

Mental Model
Use the tree's null right pointers as temporary "threads" back to the inorder successor. Each edge is traversed at most twice → O(n) time, O(1) space.
Other Approaches
Recursive O(h) call-stack space.
Iterative w/ Stack O(h) explicit stack.
Optimal Approach (Morris)
  1. While cur != null: if no left → visit cur, go right.
  2. Else find inorder predecessor (cur.left's rightmost). If pred.right == null → thread it to cur, go left.
  3. If pred.right == cur → remove thread, visit cur, go right.
Visual
flowchart TD
  C["cur = root"] --> L{"cur.left == null?"}
  L -- yes --> V["visit cur; cur = cur.right"]
  L -- no --> P["pred = rightmost of cur.left"]
  P --> T{"pred.right == null?"}
  T -- yes --> A["pred.right = cur (thread); cur = cur.left"]
  T -- no --> B["pred.right = null; visit cur; cur = cur.right"]
Code (Java)
/**
 * Morris Inorder Traversal — O(n) time, O(1) space.
 * Trick: temporarily "thread" the rightmost node of a left subtree to point back to the
 * current node, so we can find our way home without a stack. The thread is removed on
 * the second visit, restoring the tree.
 */
List<Integer> morrisInorder(TreeNode root){
  List<Integer> inorderValues = new ArrayList<>();
  TreeNode currentNode = root;

  while(currentNode != null){
    if(currentNode.left == null){
      inorderValues.add(currentNode.val);                  // no left subtree → visit now
      currentNode = currentNode.right;
    } else {
      TreeNode predecessor = currentNode.left;
      while(predecessor.right != null && predecessor.right != currentNode){
        predecessor = predecessor.right;                   // walk to rightmost of left
      }
      if(predecessor.right == null){
        predecessor.right = currentNode;                   // create thread back
        currentNode = currentNode.left;
      } else {
        predecessor.right = null;                          // remove thread → restore tree
        inorderValues.add(currentNode.val);                // left subtree fully visited
        currentNode = currentNode.right;
      }
    }
  }
  return inorderValues;
}
Variant — Morris Preorder
Same algorithm, one line moved. Visit while laying the thread (before going left) instead of when removing it. Morris preorder eats on the way down; Morris inorder eats on the way back up.
/**
 * Morris Preorder Traversal — O(n) time, O(1) space.
 * Only difference from Morris inorder: visit happens BEFORE diving into the left subtree
 * (when we create the thread), not AFTER returning (when we remove it).
 */
List<Integer> morrisPreorder(TreeNode root){
  List<Integer> preorderValues = new ArrayList<>();
  TreeNode currentNode = root;

  while(currentNode != null){
    if(currentNode.left == null){
      preorderValues.add(currentNode.val);                 // no left → visit and go right
      currentNode = currentNode.right;
    } else {
      TreeNode predecessor = currentNode.left;
      while(predecessor.right != null && predecessor.right != currentNode){
        predecessor = predecessor.right;
      }
      if(predecessor.right == null){
        preorderValues.add(currentNode.val);               // ← PREORDER: visit on the way DOWN
        predecessor.right = currentNode;                   // lay thread
        currentNode = currentNode.left;
      } else {
        predecessor.right = null;                          // just clean up; do NOT re-visit
        currentNode = currentNode.right;
      }
    }
  }
  return preorderValues;
}

 * Trick: instead of computing heights twice per node (O(n log n) worst), one DFS
 * returns either the real height OR a sentinel -1 meaning "already unbalanced below".

boolean isBalanced(TreeNode root){

private int heightOrMinusOne(TreeNode node){

  if(rightHeight == -1) return -1;
  if(Math.abs(leftHeight - rightHeight) > 1) return -1;

}

/**
 * Two-Sum in a BST using two stack-based iterators walking from both ends.
 * Trick: BSTIterator(asc) yields values in increasing order, BSTIterator(desc) in
 * decreasing order — together they emulate the classic two-pointer two-sum on a
 * sorted array, in O(h) extra space instead of O(n).
 */
boolean findTargetTwoSumInBST(TreeNode root, int targetSum){
  BSTIterator ascendingIterator  = new BSTIterator(root, /*ascending*/ true);
  BSTIterator descendingIterator = new BSTIterator(root, /*ascending*/ false);

  int smallerCurrent = ascendingIterator.next();
  int largerCurrent  = descendingIterator.next();
  while(smallerCurrent < largerCurrent){
    int currentSum = smallerCurrent + largerCurrent;
    if(currentSum == targetSum)        return true;
    else if(currentSum < targetSum)    smallerCurrent = ascendingIterator.next();
    else                               largerCurrent  = descendingIterator.next();
  }
  return false;
}

/** BST iterator that yields values in ascending OR descending order using a stack of O(h). */
class BSTIterator {
  private final Deque<TreeNode> pendingStack = new ArrayDeque<>();
  private final boolean ascendingOrder;
  BSTIterator(TreeNode root, boolean ascendingOrder){
    this.ascendingOrder = ascendingOrder;
    pushBranch(root);
  }
  int next(){
    TreeNode topNode = pendingStack.pop();
    pushBranch(ascendingOrder ? topNode.right : topNode.left);
    return topNode.val;
  }
  private void pushBranch(TreeNode node){
    while(node != null){
      pendingStack.push(node);
      node = ascendingOrder ? node.left : node.right;
    }
  }
}

Code (Java)
/**
 * Detect a cycle in an undirected graph via BFS tracking each node's parent.
 * Trick: in BFS, every cross-edge between two ALREADY-visited nodes (neither being the
 * other's parent) closes a cycle. Use a parent array to recognize "this neighbour is
 * not just where I came from".
 */
boolean hasCycleUndirected(int vertexCount, List<List<Integer>> adjacencyList){
  boolean[] visitedVertices = new boolean[vertexCount];

  for(int startVertex = 0; startVertex < vertexCount; startVertex++){
    if(visitedVertices[startVertex]) continue;
    if(bfsFindsCycle(startVertex, adjacencyList, visitedVertices)) return true;
  }
  return false;
}

private boolean bfsFindsCycle(int startVertex, List<List<Integer>> adjacencyList, boolean[] visitedVertices){
  Deque<int[]> bfsQueue = new ArrayDeque<>();      // each entry: {currentNode, parentNode}
  bfsQueue.offer(new int[]{startVertex, -1});
  visitedVertices[startVertex] = true;

  while(!bfsQueue.isEmpty()){
    int[] frame = bfsQueue.poll();
    int currentNode = frame[0], parentNode = frame[1];

    for(int neighbour : adjacencyList.get(currentNode)){
      if(!visitedVertices[neighbour]){
        visitedVertices[neighbour] = true;
        bfsQueue.offer(new int[]{neighbour, currentNode});
      } else if(neighbour != parentNode){
        return true;                                    // back edge → cycle
      }
    }
  }
  return false;
}
Time O(V + E)
Space O(V)

24Floyd-Warshall (All-Pairs Shortest Paths) Medium

Section · Graph Part-II
GFG Article
Problem

Compute the shortest path between every pair of vertices in a weighted directed graph (no negative cycles).

Mental Model
Dynamic programming over "intermediate vertex sets". After loop iteration k, dist[i][j] holds the shortest path using only vertices {0..k} as intermediates. Final answer at k = V - 1.
Other Approaches
Dijkstra from each node Faster on sparse non-negative graphs — O(V·(V+E) log V).
Bellman-Ford from each node Handles negatives — O(V²·E).
Optimal Approach
  1. Init dist[i][j] from edge weights; dist[i][i] = 0.
  2. For k = 0..V-1: for i, j: if dist[i][k] + dist[k][j] < dist[i][j] → relax.
  3. Negative cycle iff any dist[i][i] < 0 after the loop.
Visual
flowchart TD
  I["dist[][] from edges"] --> K["for k = 0..V-1"]
  K --> R["for i, j: dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])"]
  R --> N["after V rounds → dist holds all-pairs SP"]
Code (Java)
/**
 * Floyd-Warshall all-pairs shortest paths.
 * Trick: the k-th outer iteration relaxes shortest paths allowed to use vertices
 * {0..k} as INTERMEDIATES. After V outer iterations every vertex has been considered.
 */
void floydWarshall(int[][] distanceMatrix){
  int vertexCount = distanceMatrix.length;
  final int INF = Integer.MAX_VALUE;

  for(int intermediate = 0; intermediate < vertexCount; intermediate++){
    for(int source = 0; source < vertexCount; source++){
      for(int target = 0; target < vertexCount; target++){
        int viaIntermediate = (distanceMatrix[source][intermediate] == INF
                            || distanceMatrix[intermediate][target] == INF)
            ? INF
            : distanceMatrix[source][intermediate] + distanceMatrix[intermediate][target];
        if(viaIntermediate < distanceMatrix[source][target]){
          distanceMatrix[source][target] = viaIntermediate;
        }
      }
    }
  }
  // (optional) negative cycle detection: any distanceMatrix[v][v] < 0
}
Time O(V³)
Space O(V²)

25Edit Distance Hard

Section · Dynamic Programming
LeetCode Article
Problem

Minimum operations (insert, delete, replace — each cost 1) to convert word1 into word2.

Mental Model
dp[i][j] = min ops to transform word1[0..i-1] → word2[0..j-1]. If last chars match → no op (take diagonal). Else 1 + min(replace, delete, insert) = 1 + min(diag, up, left).
Other Approaches
Brute Recursion over 3 choices → exponential.
Better Memoized recursion → O(n·m) time & space.
Optimal Approach
  1. Base: dp[0][j] = j (insert j chars); dp[i][0] = i (delete i chars).
  2. If chars match → dp[i][j] = dp[i-1][j-1].
  3. Else dp[i][j] = 1 + min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1]).
Visual
flowchart TD
  C{"w1[i-1] == w2[j-1]?"} -- yes --> S["dp[i][j] = dp[i-1][j-1]"]
  C -- no --> O["dp[i][j] = 1 + min(diag, up, left)"]
Code (Java)
/**
 * Edit distance (Levenshtein) in O(n·m) time, O(m) space via 1D rolling DP.
 * Trick: each cell needs only diag (dp[i-1][j-1]), up (dp[i-1][j]), and left (dp[i][j-1]).
 * Keep one row and a saved "previousDiagonal" scalar to substitute for the full 2D grid.
 */
int minEditDistance(String word1, String word2){
  int lenA = word1.length(), lenB = word2.length();
  int[] currentRow = new int[lenB + 1];

  for(int j = 0; j <= lenB; j++) currentRow[j] = j;     // base row: dp[0][j] = j inserts

  for(int i = 1; i <= lenA; i++){
    int previousDiagonal = currentRow[0];               // = dp[i-1][0]
    currentRow[0] = i;                                  // dp[i][0] = i deletes
    for(int j = 1; j <= lenB; j++){
      int storedAbove = currentRow[j];                  // = dp[i-1][j] before overwrite
      if(word1.charAt(i - 1) == word2.charAt(j - 1)){
        currentRow[j] = previousDiagonal;               // free move
      } else {
        currentRow[j] = 1 + Math.min(previousDiagonal,                       // replace
                          Math.min(storedAbove,                               // delete
                                   currentRow[j - 1]));                       // insert
      }
      previousDiagonal = storedAbove;
    }
  }
  return currentRow[lenB];
}
Time O(n·m)
Space O(min(n,m))

26Rod Cutting Medium

Section · DP Part-II
GFG Article
Problem

Given prices for rod lengths 1..n, maximise total revenue by cutting a rod of length n into pieces (any number of cuts, any sizes).

Mental Model
Unbounded knapsack in disguise — each "length" is an item available infinitely. 1D DP traversed left-to-right over capacity (the opposite of 0/1 knapsack) lets each length be reused.
Other Approaches
Brute Recursion over first-cut length → exponential.
Ankit pick / not-pick 2D Treat each rod-length as an item with infinite supply. State f(i, remaining) = best revenue using lengths 0..i in a rod of size remaining. NOT-pick → f(i-1, remaining). PICK (stays on same index because unbounded) → price[i] + f(i, remaining - (i+1)). 
int helper(int idx, int remainingLength, int[] price, int[][] memo){
  if(idx == 0) return remainingLength * price[0];          // unlimited length-1 pieces
  if(memo[idx][remainingLength] != -1) return memo[idx][remainingLength];

  int notPick = helper(idx - 1, remainingLength, price, memo);
  int pick    = Integer.MIN_VALUE;
  int rodLen  = idx + 1;                                    // length represented by index idx
  if(remainingLength >= rodLen)
    pick = price[idx] + helper(idx, remainingLength - rodLen, price, memo);

  return memo[idx][remainingLength] = Math.max(notPick, pick);
}
Better Tabulated 2D DP dp[i][len] → O(n²) time & space, then space-optimised to the 1D form shown below.
Optimal Approach
  1. dp[0..n] = 0.
  2. For each piece length p = 1..n: for len = p..n: dp[len] = max(dp[len], dp[len - p] + price[p-1]).
  3. Return dp[n].
Visual
flowchart TD
  I["dp[0..n] = 0"] --> L["for p = 1..n"]
  L --> R["for len = p..n (LEFT → RIGHT)"]
  R --> U["dp[len] = max(dp[len], dp[len - p] + price[p-1])"]
  U --> R
  R --> L
  L --> O["return dp[n]"]
Code (Java)
/**
 * Rod cutting maximum revenue — unbounded knapsack pattern.
 * Trick: walk the capacity loop LEFT-TO-RIGHT so dp[len - piece] already reflects the
 * "with current piece allowed" state — letting each piece length be reused freely.
 */
int rodCuttingMaxRevenue(int[] priceForLength){
  int rodLength = priceForLength.length;
  int[] bestRevenueAtLength = new int[rodLength + 1];

  for(int pieceLength = 1; pieceLength <= rodLength; pieceLength++){
    int piecePrice = priceForLength[pieceLength - 1];
    for(int currentLength = pieceLength; currentLength <= rodLength; currentLength++){
      int revenueIfCutPiece = bestRevenueAtLength[currentLength - pieceLength] + piecePrice;
      bestRevenueAtLength[currentLength] = Math.max(bestRevenueAtLength[currentLength], revenueIfCutPiece);
    }
  }
  return bestRevenueAtLength[rodLength];
}
Time O(n²)
Space O(n)

27Longest String With All Prefixes (Trie) Medium

Section · Trie
Coding Ninjas Article
Problem

Given a list of strings, return the longest string such that every prefix of it is also present in the list. Tie-break: lexicographically smallest. Return "" if none.

Mental Model
Insert all words into a trie. Then a word is "complete" iff every node on its path is marked end-of-word. Scan words, keep the longest (lexicographically smallest on ties) that passes this check.
Other Approaches
HashSet Store all words; for each word check every prefix in the set → O(Σ|w|²) char ops.
Optimal Approach
  1. Insert every word into a trie marking isEnd = true at terminal nodes.
  2. For each word, walk it from root — every intermediate node must have isEnd. If yes, candidate.
  3. Keep the longest such word; on tie, pick the lexicographically smaller.
Visual
flowchart TD
  R["root"] --> N1["n (end)"] --> N2["i (end)"] --> N3["n (end)"] --> N4["j (end)"] --> N5["a (end)"]
  N5 --> O["niinja → every prefix ends → complete"]
Code (Java)
/**
 * Longest "complete" word — every prefix must also be in the dictionary.
 * Trick: build a trie marking is-end on each inserted word; a word is complete iff
 * walking it from root encounters is-end on EVERY node along the way.
 */
String longestStringWithAllPrefixes(String[] words){
  TrieNode root = new TrieNode();
  for(String word : words) insert(root, word);

  String bestComplete = "";
  for(String word : words){
    if(!everyPrefixEndsHere(root, word)) continue;
    // longer wins; on tie pick lexicographically smaller
    if(word.length() > bestComplete.length()
       || (word.length() == bestComplete.length() && word.compareTo(bestComplete) < 0)){
      bestComplete = word;
    }
  }
  return bestComplete;
}

private boolean everyPrefixEndsHere(TrieNode root, String word){
  TrieNode current = root;
  for(int i = 0; i < word.length(); i++){
    current = current.children[word.charAt(i) - 'a'];
    if(current == null || !current.isEndOfWord) return false;
  }
  return true;
}

private void insert(TrieNode root, String word){
  TrieNode current = root;
  for(int i = 0; i < word.length(); i++){
    int childIdx = word.charAt(i) - 'a';
    if(current.children[childIdx] == null) current.children[childIdx] = new TrieNode();
    current = current.children[childIdx];
  }
  current.isEndOfWord = true;
}

private static class TrieNode { TrieNode[] children = new TrieNode[26]; boolean isEndOfWord; }
Time O(Σ|w|)
Space O(Σ|w|)
