01Pascal's Triangle Medium

Section · Arrays
LeetCode Article
Problem

Three flavors: (a) value at (r,c), (b) print the entire r-th row, (c) print the first n rows.

Mental Model
Pascal's value is a binomial coefficient: C(r-1, c-1). A whole row can be generated in O(r) by sliding multiplication — no factorials, no DP table.
Other Approaches
Brute Build the full triangle row by row using T[i][j] = T[i-1][j-1] + T[i-1][j]. — O(n²) time & space.
Better For a single value (r,c) compute C(r,c) with factorials. — O(r) per query but risks overflow.
Optimal Approach (full row in O(r))
  1. Row r starts with 1.
  2. For i = 1..r-1: next value = prev × (r - i) / i.
  3. Repeating gives the whole row without computing factorials separately.
  4. To print first n rows, just call this generator n times.
Visual
flowchart TD
  A["Start row r with 1"] --> B["i = 1"]
  B --> C{"i < r?"}
  C -- no --> D["Row complete"]
  C -- yes --> E["next = prev * (r - i) / i"]
  E --> F["Append next, prev = next"]
  F --> G["i = i + 1"]
  G --> C
Code (Java)
/**
 * Returns the r-th row of Pascal's triangle (1-indexed) in O(r) time and O(r) space.
 * Insight: each entry is C(r-1, k), and consecutive entries differ by the factor (r-1-k)/(k+1),
 * so we slide multiplicatively instead of recomputing binomials.
 */
List<Integer> generateRow(int rowNumber){
  List<Integer> pascalRow = new ArrayList<>();
  long currentValue = 1;                           // C(r-1, 0) = 1
  pascalRow.add(1);
  for(int column = 1; column < rowNumber; column++){
    // slide from C(r-1, column-1) to C(r-1, column)
    currentValue = currentValue * (rowNumber - column) / column;
    pascalRow.add((int) currentValue);
  }
  return pascalRow;
}

/**
 * Generates the first numRows rows of Pascal's triangle by reusing generateRow.
 */
List<List<Integer>> generate(int numRows){
  List<List<Integer>> triangle = new ArrayList<>();
  for(int rowIndex = 1; rowIndex <= numRows; rowIndex++){
    triangle.add(generateRow(rowIndex));
  }
  return triangle;
}
Time O(n²)
Space O(n²) (output)

02Merge Overlapping Intervals Medium

Section · Arrays Part-II
LeetCode Article
Problem

Given an array of intervals [start, end], merge all overlapping intervals and return the disjoint result.

Mental Model
Sort by start, then walk left-to-right merging on the fly. If the next interval starts at or before the current end, extend the end; otherwise close the current and open a new one.
Other Approaches
Brute Sort, then for each interval scan ahead merging while overlap exists, marking merged ones used. — O(n²).
Optimal Approach
  1. Sort intervals by start.
  2. Maintain a "current" interval; iterate the rest.
  3. If next.start ≤ current.end → current.end = max(current.end, next.end).
  4. Else push current to output, set current = next.
  5. Push the last current.
Visual
flowchart TD
  S["Sort by start"] --> C["current = first"]
  C --> L{"more intervals?"}
  L -- no --> O["Push current to output"]
  L -- yes --> N["next = take one"]
  N --> Q{"next.start <= current.end?"}
  Q -- yes --> M["current.end = max(current.end, next.end)"]
  Q -- no --> P["Push current; current = next"]
  M --> L
  P --> L
Code (Java)
/**
 * Merges overlapping intervals.
 * Trick: after sorting by start, a single linear sweep is enough — once we've
 * passed an interval's end, nothing later can extend it backward.
 */
int[][] merge(int[][] intervals){
  if(intervals.length == 0) return intervals;

  // sort by start time so overlaps are always adjacent
  Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

  List<int[]> mergedIntervals = new ArrayList<>();
  int currentStart = intervals[0][0];
  int currentEnd   = intervals[0][1];

  for(int i = 1; i < intervals.length; i++){
    int nextStart = intervals[i][0];
    int nextEnd   = intervals[i][1];

    if(nextStart <= currentEnd){
      // overlap → stretch the current interval to cover the next one
      currentEnd = Math.max(currentEnd, nextEnd);
    } else {
      // gap → close out the current interval and start a fresh one
      mergedIntervals.add(new int[]{currentStart, currentEnd});
      currentStart = nextStart;
      currentEnd   = nextEnd;
    }
  }
  mergedIntervals.add(new int[]{currentStart, currentEnd});
  return mergedIntervals.toArray(new int[0][]);
}
Time O(n log n)
Space O(n) (output)

03Pow(x, n) Medium

Section · Arrays Part-III
LeetCode Article
Problem

Compute x^n for real x and (possibly negative) integer n.

Mental Model
Binary exponentiation: peel off bits of n. Square x at every step and multiply into the answer whenever the current bit is 1. Negative exponent → invert at the end.
Other Approaches
Brute Multiply x by itself n times. — O(n); too slow for large n.
Optimal Approach
  1. Take |n| into a long (because INT_MIN can't be negated as int).
  2. Initialize answer = 1, base = x.
  3. While exponent > 0: if odd, answer *= base; base *= base; exponent >>= 1.
  4. If original n < 0 → return 1 / answer.
Visual
flowchart TD
  S["answer=1, base=x, e=|n|"] --> L{"e > 0?"}
  L -- no --> R{"n < 0?"}
  R -- yes --> A["return 1/answer"]
  R -- no --> B["return answer"]
  L -- yes --> O{"e odd?"}
  O -- yes --> M["answer *= base"]
  O -- no --> K["skip"]
  M --> Q["base *= base; e >>= 1"]
  K --> Q
  Q --> L
Code (Java)
/**
 * Fast exponentiation in O(log |n|).
 * Trick: write n in binary; the answer is the product of x^(2^k) for every bit k set in n.
 * We accumulate x^(2^k) by repeatedly squaring 'base'.
 */
double myPow(double x, int n){
  long absoluteExponent = Math.abs((long) n);   // long because INT_MIN's abs overflows int
  double accumulatedResult = 1.0;
  double repeatedlySquaredBase = x;

  while(absoluteExponent > 0){
    if((absoluteExponent & 1) == 1){
      // current bit is 1 → multiply this power of x into the result
      accumulatedResult *= repeatedlySquaredBase;
    }
    repeatedlySquaredBase *= repeatedlySquaredBase;   // promote to next power of two
    absoluteExponent >>= 1;
  }
  // negative exponent → reciprocal
  return n < 0 ? 1.0 / accumulatedResult : accumulatedResult;
}
Time O(log n)
Space O(1)

044 Sum Medium

Section · Arrays Part-IV
LeetCode Article
Problem

Find all unique quadruplets (a, b, c, d) in the array such that a + b + c + d = target.

Mental Model
Sort, then two nested loops fix the first two elements; collapse the remaining two with two-pointers. Skip duplicates at each of the four levels to avoid repeating the same quadruplet.
Other Approaches
Brute Four nested loops + HashSet for dedup. — O(n⁴).
Better Three loops + HashSet to find the fourth value. — O(n³) time, O(n) set space; still messy dedup.
Optimal Approach
  1. Sort the array.
  2. Two outer loops fix indices i and j (skip if value equals previous to avoid dupes).
  3. Two pointers l = j+1, r = n-1.
  4. If sum < target → l++; if > target → r--; if equal → record and skip duplicate l & r.
Visual
flowchart TD
  S["Sort array"] --> I["fix i"]
  I --> J["fix j (j > i)"]
  J --> P["l = j+1, r = n-1"]
  P --> C{"sum vs target"}
  C -- "<" --> LR["l++"]
  C -- ">" --> RR["r--"]
  C -- "=" --> X["record quad, l++, r--, skip dupes"]
  LR --> C
  RR --> C
  X --> C
Code (Java)
/**
 * Returns all unique quadruplets that sum to 'target'.
 * Trick: sort + 2 fixed pointers + 2-pointer shrink. Use 'long' inside the sum
 * because (numbers[firstIdx] + numbers[secondIdx]) can already overflow int.
 */
List<List<Integer>> fourSum(int[] numbers, int target){
  Arrays.sort(numbers);
  List<List<Integer>> quadruplets = new ArrayList<>();
  int length = numbers.length;

  for(int firstIdx = 0; firstIdx < length - 3; firstIdx++){
    if(firstIdx > 0 && numbers[firstIdx] == numbers[firstIdx - 1]) continue;   // skip dup first
    for(int secondIdx = firstIdx + 1; secondIdx < length - 2; secondIdx++){
      if(secondIdx > firstIdx + 1 && numbers[secondIdx] == numbers[secondIdx - 1]) continue;

      int leftPointer  = secondIdx + 1;
      int rightPointer = length - 1;
      while(leftPointer < rightPointer){
        long quadSum = (long) numbers[firstIdx] + numbers[secondIdx]
                     + numbers[leftPointer] + numbers[rightPointer];
        if(quadSum == target){
          quadruplets.add(Arrays.asList(
              numbers[firstIdx], numbers[secondIdx],
              numbers[leftPointer], numbers[rightPointer]));
          leftPointer++; rightPointer--;
          // skip duplicates of the two moving pointers
          while(leftPointer < rightPointer && numbers[leftPointer]  == numbers[leftPointer - 1])  leftPointer++;
          while(leftPointer < rightPointer && numbers[rightPointer] == numbers[rightPointer + 1]) rightPointer--;
        } else if(quadSum < target){
          leftPointer++;
        } else {
          rightPointer--;
        }
      }
    }
  }
  return quadruplets;
}
Time O(n³)
Space O(1) aux

05Find Middle of Linked List Easy

Section · Linked List
LeetCode Article
Problem

Return the middle node of a singly linked list. For even length, return the second middle.

Mental Model
Tortoise & hare. Move a slow pointer 1 step and a fast pointer 2 steps. When fast reaches the end, slow sits on the middle — one traversal, no length count, no extra memory.
Other Approaches
Brute Walk once to count n, then walk again to node n/2. — Two passes, O(n).
Optimal Approach
  1. Initialize slow = head, fast = head.
  2. While fast != null AND fast.next != null: advance slow by 1, fast by 2.
  3. Return slow.
Visual
flowchart LR
  H[head] --> A[1]
  A --> B[2]
  B --> C[3]
  C --> D[4]
  D --> E[5]
  E --> X[null]
  subgraph Step
    direction TB
    S["slow +1 each tick"]
    F["fast +2 each tick"]
  end
Code (Java)
/**
 * Returns the middle node (the 2nd middle for even length lists).
 * Two-pointer trick: when 'fastWalker' has covered 2n steps, 'slowWalker' is at n.
 */
ListNode middleNode(ListNode head){
  ListNode slowWalker = head;
  ListNode fastWalker = head;
  while(fastWalker != null && fastWalker.next != null){
    slowWalker = slowWalker.next;           // +1 step
    fastWalker = fastWalker.next.next;      // +2 steps
  }
  return slowWalker;
}
Time O(n)
Space O(1)

06Detect a Loop in Linked List Medium

Section · Linked List Part-II
LeetCode Article
Problem

Return true if the linked list has a cycle, else false.

Mental Model
Floyd's cycle finding. If two runners on a circular track move at different speeds, the faster must lap the slower. On a non-cyclic list, the faster simply falls off the end.
Other Approaches
Brute HashSet of visited nodes; cycle exists if a node repeats. — O(n) time, O(n) space.
Optimal Approach
  1. slow = fast = head.
  2. Advance slow by 1, fast by 2 each iteration.
  3. If fast hits null → no cycle. If slow == fast → cycle found.
Visual
flowchart LR
  H[head] --> A[A] --> B[B] --> C[C] --> D[D] --> E[E] --> B
Code (Java)
/**
 * Floyd's tortoise & hare: returns true iff the list contains a cycle.
 * If the list is finite, the fast pointer falls off the end.
 * If it's cyclic, the fast pointer will eventually lap and collide with the slow one.
 */
boolean hasCycle(ListNode head){
  ListNode slowWalker = head;
  ListNode fastWalker = head;
  while(fastWalker != null && fastWalker.next != null){
    slowWalker = slowWalker.next;          // +1
    fastWalker = fastWalker.next.next;     // +2
    if(slowWalker == fastWalker) return true;   // collision == cycle
  }
  return false;                            // fast fell off → no cycle
}
Time O(n)
Space O(1)

07Clone a LL with Random Pointer Hard

Section · Linked List & Arrays
LeetCode Article
Problem

Each node has next AND a random pointer (to any node or null). Return a deep copy.

Mental Model
Interleave clones into the original list to encode the mapping in pointer geometry itself. Then random pointers can be copied as orig.random.next with zero hashing.
Other Approaches
Brute HashMap<origNode, cloneNode> — pass 1 creates clones, pass 2 wires next/random via the map. — O(n) time, O(n) space.
Optimal Approach (O(1) extra)
  1. Pass 1 — Weave: between each orig and orig.next, insert a clone, so list becomes O₁→C₁→O₂→C₂→…
  2. Pass 2 — Random: for each original O, set O.next.random = O.random == null ? null : O.random.next.
  3. Pass 3 — Unweave: detach clones from originals, restoring both lists.
Visual
flowchart LR
  O1["O1"] --> C1["C1"] --> O2["O2"] --> C2["C2"] --> O3["O3"] --> C3["C3"]
  O1 -. random .-> O3
  C1 -. random .-> C3
Code (Java)
/**
 * Deep copies a linked list whose nodes have an extra 'random' pointer.
 * Trick: temporarily INTERLEAVE clones with originals, so each clone sits at
 * original.next. Then a clone's random is just original.random.next — no map needed.
 */
Node copyRandomList(Node head){
  if(head == null) return null;

  // ---- Pass 1: weave clones between originals: O1 -> C1 -> O2 -> C2 ... ----
  Node originalCursor = head;
  while(originalCursor != null){
    Node clonedNode = new Node(originalCursor.val);
    clonedNode.next = originalCursor.next;
    originalCursor.next = clonedNode;
    originalCursor = clonedNode.next;
  }

  // ---- Pass 2: assign random pointers on the clones ----
  originalCursor = head;
  while(originalCursor != null){
    // clone of X is X.next; clone of X.random is X.random.next
    originalCursor.next.random = (originalCursor.random == null)
        ? null
        : originalCursor.random.next;
    originalCursor = originalCursor.next.next;
  }

  // ---- Pass 3: unweave: restore originals, extract clones into their own list ----
  originalCursor = head;
  Node clonedListHead = head.next;
  Node clonedCursor   = clonedListHead;
  while(originalCursor != null){
    originalCursor.next = originalCursor.next.next;                    // skip clone
    clonedCursor.next   = (clonedCursor.next != null) ? clonedCursor.next.next : null;
    originalCursor = originalCursor.next;
    clonedCursor   = clonedCursor.next;
  }
  return clonedListHead;
}
Time O(n)
Space O(1) aux

08Minimum Number of Platforms Medium

Section · Greedy Algorithm
GFG Article
Problem

Given arrival and departure times of trains, find the minimum number of platforms required so no train waits.

Mental Model
Sort arrivals and departures independently, then sweep like a two-finger merge. Every arrival before the next departure needs an extra platform; max simultaneous count is the answer.
Other Approaches
Brute For each train, count overlaps with every other train. — O(n²).
Better Min-heap of current departure times; pop when an arrival happens after the earliest departure. — O(n log n), easier to reason about online.
Optimal Approach
  1. Sort arr[] and dep[] separately.
  2. Two indices i, j; platforms = 0, peak = 0.
  3. If arr[i] ≤ dep[j] → train arrived before earliest departure: platforms++, i++.
  4. Else a platform freed up: platforms--, j++.
  5. Track peak = max(peak, platforms).
Visual
flowchart TD
  S["Sort arrivals & departures"] --> I["i=0, j=0, plat=0, peak=0"]
  I --> L{"i < n?"}
  L -- no --> R["return peak"]
  L -- yes --> C{"arr[i] <= dep[j]?"}
  C -- yes --> P["plat++, i++, peak=max(peak,plat)"]
  C -- no --> F["plat--, j++"]
  P --> L
  F --> L
Code (Java)
/**
 * Minimum platforms needed so no train ever waits.
 * Key insight: ordering the arrivals and departures separately lets us simulate
 * the "current trains on station" count as we sweep time forward.
 */
int findPlatform(int[] arrivalTimes, int[] departureTimes){
  Arrays.sort(arrivalTimes);
  Arrays.sort(departureTimes);

  int totalTrains       = arrivalTimes.length;
  int arrivalIdx        = 0;
  int departureIdx      = 0;
  int currentlyParked   = 0;
  int peakPlatformsUsed = 0;

  while(arrivalIdx < totalTrains){
    if(arrivalTimes[arrivalIdx] <= departureTimes[departureIdx]){
      // a train arrived before/at the earliest pending departure → need another platform
      currentlyParked++;
      arrivalIdx++;
      peakPlatformsUsed = Math.max(peakPlatformsUsed, currentlyParked);
    } else {
      // earliest pending train departed before the next arrival → free a platform
      currentlyParked--;
      departureIdx++;
    }
  }
  return peakPlatformsUsed;
}
Time O(n log n)
Space O(1)

09Subsets II (with duplicates) Medium

Section · Recursion
LeetCode Article
Problem

Return all unique subsets of an array that may contain duplicates.

Mental Model
Sort first, then at every recursion level skip duplicates by index. The rule: "the first duplicate at a level may be picked, but the second copy at the same level must be skipped" prevents generating the same subset twice.
Other Approaches
Brute Generate all 2ⁿ subsets via bitmask, store in a HashSet to dedup. — O(2ⁿ · n) time, O(2ⁿ · n) space.
Optimal Approach
  1. Sort the array so duplicates are adjacent.
  2. Recurse with a start index; at every call, push the current subset.
  3. Loop i = start..n-1: skip if i > start && nums[i] == nums[i-1].
  4. Add nums[i], recurse with i+1, then pop and continue.
Visual
flowchart TD
  R["recurse(start, path)"] --> A["add path to answer"]
  A --> L["for i = start..n-1"]
  L --> C{"i > start AND nums[i] == nums[i-1]?"}
  C -- yes --> S["skip"]
  C -- no --> P["pick nums[i] → recurse(i+1) → unpick"]
  S --> L
  P --> L
Code (Java)
/**
 * All unique subsets when the input may contain duplicates.
 * Trick: after sorting, the rule "skip nums[i] if it equals nums[i-1] AND it's not the
 * first pick at this recursion level" guarantees each subset is emitted exactly once.
 */
List<List<Integer>> subsetsWithDup(int[] numbers){
  Arrays.sort(numbers);                                    // bring duplicates together
  List<List<Integer>> allSubsets = new ArrayList<>();
  generateSubsets(0, numbers, new ArrayList<>(), allSubsets);
  return allSubsets;
}

private void generateSubsets(int startIdx, int[] numbers,
                             List<Integer> currentSubset,
                             List<List<Integer>> allSubsets){
  allSubsets.add(new ArrayList<>(currentSubset));          // snapshot the current subset
  for(int pickIdx = startIdx; pickIdx < numbers.length; pickIdx++){
    // dedup: at this level the first occurrence of a value can be picked,
    // but the second/third occurrences would re-create an already-emitted subset
    if(pickIdx > startIdx && numbers[pickIdx] == numbers[pickIdx - 1]) continue;
    currentSubset.add(numbers[pickIdx]);
    generateSubsets(pickIdx + 1, numbers, currentSubset, allSubsets);
    currentSubset.remove(currentSubset.size() - 1);        // backtrack
  }
}
Time O(2ⁿ · n)
Space O(n) recursion

10N-Queens Hard

Section · Recursion & Backtracking
LeetCode Article
Problem

Place n queens on an n × n board so no two attack each other. Return all distinct boards.

Mental Model
Backtrack column-by-column with three "attack" boolean arrays. A queen at (row, col) blocks one column, one "↙↗" diagonal (row + col) and one "↘↖" anti-diagonal (row - col + n - 1). O(1) check per placement.
Other Approaches
Brute For every column placement, re-scan the partial board to check safety. — O(N! · N).
Optimal Approach
  1. Place one queen per column; recurse on the next column.
  2. Three boolean arrays: rowUsed[], diag1[row+col], diag2[row-col+n-1].
  3. If a position is safe, mark, recurse, then unmark (backtrack).
  4. When column reaches n, materialize the board as strings.
Visual
flowchart TD
  S["place(col=0)"] --> L["for row in 0..n-1"]
  L --> C{"row, diag1, diag2 all free?"}
  C -- no --> L
  C -- yes --> M["mark all three; board[row][col]='Q'"]
  M --> R["place(col+1)"]
  R --> U["unmark (backtrack)"]
  U --> L
  R --> D{"col == n?"}
  D -- yes --> O["save board"]
Code (Java)
/**
 * Generates all N-queen placements.
 * Trick: cells on the same "/" diagonal share (row+col); on the same "\" diagonal share
 * (row-col+n-1). Tracking these as boolean arrays makes safety checks O(1).
 */
List<List<String>> solveNQueens(int boardSize){
  List<List<String>> allBoards = new ArrayList<>();
  char[][] board = new char[boardSize][boardSize];
  for(char[] row : board) Arrays.fill(row, '.');

  boolean[] rowAttacked       = new boolean[boardSize];
  boolean[] slashDiagAttacked = new boolean[2 * boardSize - 1];  // row + col
  boolean[] backDiagAttacked  = new boolean[2 * boardSize - 1];  // row - col + n - 1

  placeQueens(0, board, allBoards, rowAttacked, slashDiagAttacked, backDiagAttacked);
  return allBoards;
}

private void placeQueens(int currentColumn, char[][] board, List<List<String>> allBoards,
                         boolean[] rowAttacked, boolean[] slashDiagAttacked, boolean[] backDiagAttacked){
  int boardSize = board.length;
  if(currentColumn == boardSize){
    // built a complete safe board → record it as a list of row-strings
    List<String> snapshot = new ArrayList<>();
    for(char[] row : board) snapshot.add(new String(row));
    allBoards.add(snapshot);
    return;
  }
  for(int candidateRow = 0; candidateRow < boardSize; candidateRow++){
    int slashIdx = candidateRow + currentColumn;
    int backIdx  = candidateRow - currentColumn + boardSize - 1;
    if(rowAttacked[candidateRow] || slashDiagAttacked[slashIdx] || backDiagAttacked[backIdx]) continue;

    // place
    board[candidateRow][currentColumn] = 'Q';
    rowAttacked[candidateRow] = slashDiagAttacked[slashIdx] = backDiagAttacked[backIdx] = true;

    placeQueens(currentColumn + 1, board, allBoards, rowAttacked, slashDiagAttacked, backDiagAttacked);

    // un-place (backtrack)
    board[candidateRow][currentColumn] = '.';
    rowAttacked[candidateRow] = slashDiagAttacked[slashIdx] = backDiagAttacked[backIdx] = false;
  }
}
Time O(N!)
Space O(N²) board

11Matrix Median Hard

Section · Binary Search
InterviewBit Article
Problem

A matrix with row-sorted rows and an odd total count R·C. Find the median value.

Mental Model
Binary-search on the answer value, not on indices. For a candidate x, count how many elements are ≤ x (using upper-bound on each row). If the count exceeds half, the median is at most x.
Other Approaches
Brute Flatten all R·C elements, sort, return middle. — O(R·C log(R·C)) time, O(R·C) space.
Optimal Approach
  1. Range of values: lo = min(first column), hi = max(last column).
  2. Binary search on value mid.
  3. Count elements ≤ mid across rows via upper-bound (binary search per row).
  4. If count ≤ (R·C)/2 → answer is > mid: lo = mid + 1; else hi = mid - 1.
  5. Return lo.
Visual
flowchart TD
  S["lo=min, hi=max"] --> L{"lo <= hi?"}
  L -- no --> R["return lo"]
  L -- yes --> M["mid = (lo+hi)/2"]
  M --> C["count = sum upperBound(row, mid) over rows"]
  C --> Q{"count <= half?"}
  Q -- yes --> A["lo = mid + 1"]
  Q -- no --> B["hi = mid - 1"]
  A --> L
  B --> L
Code (Java)
/**
 * Median of a row-sorted matrix without flattening.
 * Trick: binary-search the VALUE space (between matrix min and max), and use the
 * monotonic relation "count(elements <= v) is non-decreasing in v" to converge.
 */
int matrixMedian(int[][] rowSortedMatrix){
  int rowCount    = rowSortedMatrix.length;
  int colCount    = rowSortedMatrix[0].length;
  int desiredRank = (rowCount * colCount) / 2;            // 0-indexed median position

  int lowValue  = Integer.MAX_VALUE;
  int highValue = Integer.MIN_VALUE;
  for(int row = 0; row < rowCount; row++){
    lowValue  = Math.min(lowValue,  rowSortedMatrix[row][0]);
    highValue = Math.max(highValue, rowSortedMatrix[row][colCount - 1]);
  }

  while(lowValue <= highValue){
    int midValue = lowValue + (highValue - lowValue) / 2;

    // count how many matrix entries are <= midValue
    int countLessOrEqual = 0;
    for(int row = 0; row < rowCount; row++){
      countLessOrEqual += upperBound(rowSortedMatrix[row], midValue);
    }

    if(countLessOrEqual <= desiredRank){
      // not enough elements are <= midValue → median must be larger
      lowValue = midValue + 1;
    } else {
      highValue = midValue - 1;
    }
  }
  return lowValue;
}

// number of elements in 'row' that are <= target
private int upperBound(int[] row, int target){
  int left = 0, right = row.length;
  while(left < right){
    int mid = (left + right) / 2;
    if(row[mid] <= target) left = mid + 1;
    else                   right = mid;
  }
  return left;
}
Time O(R · log C · log(maxVal))
Space O(1)

 * k-th largest element using a bounded min-heap.

13Implement Queue using Arrays Easy

Problem

Implement enqueue, dequeue, peek, size using a fixed-size array.

Circular buffer. Two indices — front and rear — chase each other modulo capacity. Wrapping around reuses freed slots so the array never "drifts" rightward.

Brute Plain array: enqueue at end, dequeue by shifting all remaining elements left. — Dequeue O(n).

  2. enqueue: if count == capacity throw; else buffer[rear] = x; rear = (rear+1) % cap; count++.
  3. dequeue: if empty throw; else val = buffer[front]; front = (front+1) % cap; count--; return val.
  4. peek: return buffer[front].
Visual
flowchart LR
  F["front"] --> S0["slot 0"]

/**
 * Fixed-capacity queue backed by a circular buffer.
 * Trick: 'frontIndex' and 'rearIndex' advance modulo capacity; an explicit
 * 'elementCount' avoids the classic "is empty vs full" ambiguity when front == rear.
 */
class ArrayQueue {
  private final int[] buffer;
  private final int   capacity;
  private int frontIndex   = 0;
  private int rearIndex    = 0;
  private int elementCount = 0;

  ArrayQueue(int capacity){
    this.capacity = capacity;
    this.buffer   = new int[capacity];
  }

  void enqueue(int value){
    if(elementCount == capacity) throw new RuntimeException("Queue full");
    buffer[rearIndex] = value;
    rearIndex = (rearIndex + 1) % capacity;     // wrap around end of array
    elementCount++;
  }

  int dequeue(){
    if(elementCount == 0) throw new RuntimeException("Queue empty");
    int dequeuedValue = buffer[frontIndex];
    frontIndex = (frontIndex + 1) % capacity;   // wrap front too
    elementCount--;
    return dequeuedValue;
  }

  int peek(){
    if(elementCount == 0) throw new RuntimeException("Queue empty");
    return buffer[frontIndex];
  }

  int size(){ return elementCount; }
}

14LRU Cache Medium

Section · Stack & Queue Part-II
LeetCode Article
Problem

Design an O(1) get(key) and put(key, value) cache that evicts the least recently used key when over capacity.

Mental Model
Doubly Linked List + HashMap. The list orders entries by recency (head = most recent, tail = least recent). The map gives O(1) lookup to the list node so you can re-link without scanning.
Other Approaches
Brute Single array; on every access scan to find/move entry. — O(n) per op.
Better Use Java's built-in LinkedHashMap with access-order = true and override removeEldestEntry. — Concise, but hides the mechanism.
Optimal Approach
  1. Node has key, value, prev, next. Sentinel head and tail.
  2. get: if key in map → move node right after head → return value. Else -1.
  3. put: if key exists → update + move to head. Else create node, insert after head; if over capacity, remove node before tail and erase its key from the map.
Visual
flowchart LR
  H["head sentinel"] --> N1["MRU"] --> N2["..."] --> N3["LRU"] --> T["tail sentinel"]
  M["HashMap key → node"] -. O(1) lookup .-> N2
Code (Java)
/**
 * O(1) LRU cache: HashMap for lookup + doubly linked list for ordering by recency.
 * Head sentinel side = MOST recently used, Tail sentinel side = LEAST recently used.
 */
class LRUCache {
  private static class CacheNode {
    int key, value;
    CacheNode prev, next;
    CacheNode(int key, int value){ this.key = key; this.value = value; }
  }

  private final int capacity;
  private final Map<Integer, CacheNode> keyToNode = new HashMap<>();
  private final CacheNode headSentinel = new CacheNode(0, 0);
  private final CacheNode tailSentinel = new CacheNode(0, 0);

  LRUCache(int capacity){
    this.capacity = capacity;
    headSentinel.next = tailSentinel;
    tailSentinel.prev = headSentinel;
  }

  int get(int key){
    if(!keyToNode.containsKey(key)) return -1;
    CacheNode node = keyToNode.get(key);
    moveToFront(node);                              // refresh recency
    return node.value;
  }

  void put(int key, int value){
    if(keyToNode.containsKey(key)){
      CacheNode existing = keyToNode.get(key);
      existing.value = value;
      moveToFront(existing);
      return;
    }
    if(keyToNode.size() == capacity){
      // evict the least recently used (node just before tail sentinel)
      CacheNode leastRecentlyUsed = tailSentinel.prev;
      unlink(leastRecentlyUsed);
      keyToNode.remove(leastRecentlyUsed.key);
    }
    CacheNode freshNode = new CacheNode(key, value);
    insertAfterHead(freshNode);
    keyToNode.put(key, freshNode);
  }

  private void moveToFront(CacheNode node){ unlink(node); insertAfterHead(node); }
  private void unlink(CacheNode node){
    node.prev.next = node.next;
    node.next.prev = node.prev;
  }
  private void insertAfterHead(CacheNode node){
    node.prev = headSentinel;
    node.next = headSentinel.next;
    headSentinel.next.prev = node;
    headSentinel.next = node;
  }
}
Time O(1) per op
Space O(capacity)

15Longest Palindromic Substring Medium

Section · String
LeetCode Article
Problem

Given a string s, return the longest substring of s that reads the same forward and backward.

Mental Model
Expand from each centre. A palindrome has either 1 character or 2 characters at its centre. Try every index as both kinds of centre and grow outward while characters match — best length wins.
Other Approaches
Brute Check every substring for palindromicity. — O(n³).
Better DP table isPal[i][j] built by length. — O(n²) time, O(n²) space.
Optimal Approach (Expand-around-center)
  1. For each i in 0..n-1 try odd centre (i, i) and even centre (i, i+1).
  2. While s[left] == s[right] and both inside bounds: expand outward.
  3. After expansion the palindrome is s[left+1 .. right-1].
  4. Track the longest span seen and return it.
Visual
flowchart TD
  S["for each i"] --> A["expand(i, i) for odd"]
  S --> B["expand(i, i+1) for even"]
  A --> C["update best span"]
  B --> C

/**
 * Longest palindromic substring via centre expansion.
 * Trick: every palindrome has a centre — either a single char (odd length) or a
 * gap between two equal chars (even length). Trying both for each position covers all cases.
 */
String longestPalindrome(String text){
  if(text == null || text.isEmpty()) return "";

  int bestStart  = 0;
  int bestLength = 1;

  for(int centerIdx = 0; centerIdx < text.length(); centerIdx++){
    int oddLength  = expandAroundCentre(text, centerIdx, centerIdx);     // single-char centre
    int evenLength = expandAroundCentre(text, centerIdx, centerIdx + 1); // gap centre
    int longest    = Math.max(oddLength, evenLength);

    if(longest > bestLength){
      // recover start of palindrome: subtracting (longest-1)/2 from centre gives left edge
      bestStart  = centerIdx - (longest - 1) / 2;
      bestLength = longest;
    }
  }
  return text.substring(bestStart, bestStart + bestLength);
}

private int expandAroundCentre(String text, int leftIdx, int rightIdx){
  while(leftIdx >= 0 && rightIdx < text.length()
        && text.charAt(leftIdx) == text.charAt(rightIdx)){
    leftIdx--;  rightIdx++;
  }
  // when loop exits, the palindrome was text[leftIdx+1 .. rightIdx-1] → length = rightIdx - leftIdx - 1
  return rightIdx - leftIdx - 1;
}

16KMP — Longest Proper Prefix-Suffix (LPS) Hard

Section · String Part-II
LeetCode Article
Problem

For a pattern p, compute the LPS array — at each index i, the length of the longest proper prefix of p[0..i] that is also its suffix. Use it to find the first occurrence of p in a text t in O(n + m).

Mental Model
The LPS array tells the matcher how far to jump back on a mismatch without rescanning the text. If we just matched p[0..i-1] and fail at p[i], we already know the next lps[i-1] characters match — skip past them.
Other Approaches
Brute Naive substring search: for every shift, compare up to m characters. — O(n·m).
Optimal Approach (LPS build + match)
  1. Build LPS: pointers length = 0, i = 1. If p[i] == p[length] → length++; lps[i++] = length. Else if length > 0 → length = lps[length-1] (fallback), else lps[i++] = 0.
  2. Match: pointers i on text, j on pattern. On match advance both; when j == m report hit at i - m, then j = lps[j-1].
  3. On mismatch: if j > 0 → j = lps[j-1]; else i++.
Visual
flowchart TD
  B["Build LPS over pattern"] --> M["i=0 on text, j=0 on pattern"]
  M --> C{"t[i] == p[j]?"}
  C -- yes --> A["i++, j++"]
  A --> H{"j == m?"}
  H -- yes --> R["found at i - m; j = lps[j-1]"]
  H -- no --> C
  C -- no --> F{"j > 0?"}
  F -- yes --> J["j = lps[j-1]"]
  F -- no --> N["i++"]
  J --> C
  N --> C
Code (Java)
/**
 * Builds the LPS (Longest Proper Prefix that is also Suffix) array for 'pattern'.
 * Trick: extending the previous LPS by one char succeeds when pattern[i] equals
 * pattern[currentPrefixLen]; on failure we recursively fall back to the next-best
 * prefix length, which is lps[currentPrefixLen - 1].
 */
int[] buildLpsArray(String pattern){
  int patternLen = pattern.length();
  int[] lps = new int[patternLen];
  int currentPrefixLen = 0;        // length of the previous longest prefix-suffix
  int i = 1;
  while(i < patternLen){
    if(pattern.charAt(i) == pattern.charAt(currentPrefixLen)){
      currentPrefixLen++;
      lps[i++] = currentPrefixLen;
    } else if(currentPrefixLen > 0){
      // fall back to the next-best prefix that could still extend
      currentPrefixLen = lps[currentPrefixLen - 1];
    } else {
      lps[i++] = 0;
    }
  }
  return lps;
}

/**
 * Returns the index of the first occurrence of 'pattern' in 'text', or -1.
 * Trick: after a mismatch at j we don't rewind the text pointer — we slide the
 * pattern by exactly j - lps[j-1] positions and resume.
 */
int strStr(String text, String pattern){
  if(pattern.isEmpty()) return 0;
  int[] lps = buildLpsArray(pattern);
  int textIdx = 0, patternIdx = 0;
  while(textIdx < text.length()){
    if(text.charAt(textIdx) == pattern.charAt(patternIdx)){
      textIdx++;  patternIdx++;
      if(patternIdx == pattern.length()) return textIdx - patternIdx;
    } else if(patternIdx > 0){
      patternIdx = lps[patternIdx - 1];     // skip the already-matched prefix
    } else {
      textIdx++;
    }
  }
  return -1;
}

/**
 * Iterative preorder traversal using an explicit stack.
 * Trick: a stack is LIFO, so pushing right child before left makes the left subtree
 * pop next — exactly the order preorder requires.
 */
List<Integer> preorderTraversal(TreeNode root){
  List<Integer> preorderOutput = new ArrayList<>();
  if(root == null) return preorderOutput;

  Deque<TreeNode> traversalStack = new ArrayDeque<>();
  traversalStack.push(root);
  while(!traversalStack.isEmpty()){
    TreeNode currentNode = traversalStack.pop();
    preorderOutput.add(currentNode.val);

    // push RIGHT first so LEFT is processed next (preorder = root, left, right)
    if(currentNode.right != null) traversalStack.push(currentNode.right);
    if(currentNode.left  != null) traversalStack.push(currentNode.left);
  }
  return preorderOutput;
}

/**
 * Builds a binary tree from preorder + inorder arrays.
 * Trick: preorder gives ROOT at its head; inorder shows EXACTLY which elements are in
 * the left vs right subtree (everything left of root in inorder is the left subtree).
 * A HashMap of value→inorderIndex makes each split O(1) → overall O(n).
 */
TreeNode buildTree(int[] preorder, int[] inorder){
  Map<Integer, Integer> inorderIndexOf = new HashMap<>();
  for(int i = 0; i < inorder.length; i++) inorderIndexOf.put(inorder[i], i);
  return constructSubtree(preorder, 0, preorder.length - 1,
                          inorder,  0, inorder.length  - 1,
                          inorderIndexOf);

private TreeNode constructSubtree(int[] preorder, int preStart, int preEnd,
                                  int[] inorder,  int inStart,  int inEnd,
                                  Map<Integer,Integer> inorderIndexOf){
  if(preStart > preEnd || inStart > inEnd) return null;

  int rootValue        = preorder[preStart];               // preorder[0] of this segment = root
  int rootInorderIdx   = inorderIndexOf.get(rootValue);
  int leftSubtreeSize  = rootInorderIdx - inStart;

  TreeNode root = new TreeNode(rootValue);
  // left subtree comes from the next leftSubtreeSize preorder slots and inorder slots before root
  root.left  = constructSubtree(preorder, preStart + 1,                 preStart + leftSubtreeSize,
                                inorder,  inStart,                      rootInorderIdx - 1,
                                inorderIndexOf);
  root.right = constructSubtree(preorder, preStart + leftSubtreeSize + 1, preEnd,
                                inorder,  rootInorderIdx + 1,            inEnd,
                                inorderIndexOf);
  return root;
}

 * Trick: every time we move LEFT, the current node is a *valid* upper bound,

int findCeil(TreeNode root, int targetValue){

22Find Median from Data Stream Hard

Section · Binary Tree (Misc)
LeetCode Article
Problem

Design a class supporting addNum(x) and findMedian() over a stream.

Mental Model
Two heaps split the stream at the median. A MAX-heap holds the lower half, a MIN-heap holds the upper half. The median is either the larger of the two roots or their average — depending on parity.
Other Approaches
Brute Sort the list every findMedian call. — O(n log n) per query.
Better Insert into a sorted list using binary search. — O(n) insert (shift cost).
Optimal Approach
  1. Maintain lowerMaxHeap (lower half) and upperMinHeap (upper half).
  2. addNum: push to lowerMaxHeap, then move its top to upperMinHeap (so sides are sorted).
  3. Rebalance: if upperMinHeap.size() > lowerMaxHeap.size() → move top of upper back.
  4. findMedian: if sizes equal → average of two tops; else top of lowerMaxHeap.
Visual
flowchart LR
  L["lowerMaxHeap (smaller half)
root = largest of lower"] --- M["MEDIAN line"]
  M --- U["upperMinHeap (larger half)
root = smallest of upper"]
Code (Java)
/**
 * Running median via two heaps.
 * Invariant: lowerMaxHeap.size() == upperMinHeap.size() OR exactly one more.
 * Every element in lowerMaxHeap is <= every element in upperMinHeap → median is at the boundary.
 */
class MedianFinder {
  private final PriorityQueue<Integer> lowerMaxHeap = new PriorityQueue<>(Collections.reverseOrder());
  private final PriorityQueue<Integer> upperMinHeap = new PriorityQueue<>();

  void addNum(int value){
    // 1) provisionally place in lower half
    lowerMaxHeap.offer(value);
    // 2) move the largest of lower to upper, ensuring ordering between halves
    upperMinHeap.offer(lowerMaxHeap.poll());
    // 3) rebalance: lower must be the same size as or one larger than upper
    if(upperMinHeap.size() > lowerMaxHeap.size()){
      lowerMaxHeap.offer(upperMinHeap.poll());
    }
  }

  double findMedian(){
    if(lowerMaxHeap.size() > upperMinHeap.size()){
      return lowerMaxHeap.peek();                                  // odd total → bigger half's root
    }
    return (lowerMaxHeap.peek() + upperMinHeap.peek()) / 2.0;      // even total → average of roots
  }
}

 */
List<Integer> dfsOfGraph(int vertexCount, List<List<Integer>> adjacencyList){
  boolean[] visitedVertices = new boolean[vertexCount];
  List<Integer> traversalOrder = new ArrayList<>();
  dfsExplore(0, adjacencyList, visitedVertices, traversalOrder);
  return traversalOrder;
}

private void dfsExplore(int currentVertex, List<List<Integer>> adjacencyList,
                        boolean[] visitedVertices, List<Integer> traversalOrder){
  visitedVertices[currentVertex] = true;
  traversalOrder.add(currentVertex);
  for(int neighbourVertex : adjacencyList.get(currentVertex)){
    if(!visitedVertices[neighbourVertex]){
      dfsExplore(neighbourVertex, adjacencyList, visitedVertices, traversalOrder);
    }
  }
}
Time O(V + E)
Space O(V)

24Dijkstra's Algorithm Hard

Section · Graph Part-II
GFG Article
Problem

Given a weighted graph with non-negative edge weights and a source, compute the shortest distance from source to every vertex.

Mental Model
BFS where the "next" vertex is the closest unfinished one — pulled in O(log V) from a min-heap. Non-negative weights ensure that the first time we pop a vertex its distance is final.
Other Approaches
Brute Linear scan to find the closest vertex each round. — O(V²); competitive only for dense graphs.
Optimal Approach
  1. Distance array initialized to ∞; dist[src] = 0.
  2. Push (0, src) to a min-heap of (distance, vertex).
  3. Pop closest; for each neighbour, if dist[u] + w < dist[v] → relax and push.
  4. Stale heap entries (where popped distance > current dist) are skipped.
Visual
flowchart TD
  S["push (0, src)"] --> L{"heap empty?"}
  L -- yes --> D["done"]
  L -- no --> P["pop (d, u)"]
  P --> X{"d > dist[u]?"}
  X -- yes --> L
  X -- no --> N["for each neighbour v with weight w"]
  N --> R{"dist[u]+w < dist[v]?"}
  R -- yes --> A["relax + push (dist[u]+w, v)"]
  R -- no --> N
  A --> L
Code (Java)
/**
 * Dijkstra's shortest path from source on a non-negative-weighted graph.
 * Trick: a min-heap of (distance, vertex) always exposes the closest unprocessed vertex
 * in O(log V). The first time we pop a vertex its shortest distance is finalised.
 */
int[] dijkstra(int vertexCount, List<List<int[]>> adjacencyList, int sourceVertex){
  int[] shortestDistance = new int[vertexCount];
  Arrays.fill(shortestDistance, Integer.MAX_VALUE);
  shortestDistance[sourceVertex] = 0;

  // each entry: {currentDistance, vertex}
  PriorityQueue<int[]> explorationHeap = new PriorityQueue<>((a, b) -> Integer.compare(a[0], b[0]));
  explorationHeap.offer(new int[]{0, sourceVertex});

  while(!explorationHeap.isEmpty()){
    int[] currentEntry  = explorationHeap.poll();
    int   currentDist   = currentEntry[0];
    int   currentVertex = currentEntry[1];

    // skip stale entries that were superseded by a shorter relaxation
    if(currentDist > shortestDistance[currentVertex]) continue;

    for(int[] neighbourEdge : adjacencyList.get(currentVertex)){
      int neighbourVertex = neighbourEdge[0];
      int edgeWeight      = neighbourEdge[1];
      int candidateDist   = currentDist + edgeWeight;

      if(candidateDist < shortestDistance[neighbourVertex]){
        // found a strictly better path to neighbour → record & schedule it
        shortestDistance[neighbourVertex] = candidateDist;
        explorationHeap.offer(new int[]{candidateDist, neighbourVertex});
      }
    }
  }
  return shortestDistance;
}
Time O((V+E) log V)
Space O(V)

25Longest Increasing Subsequence Medium

Section · Dynamic Programming
LeetCode Article
Problem

Return the length of the longest strictly increasing subsequence of an array.

Mental Model
Maintain a "tails" array — the smallest possible tail value for an LIS of each length. For each new element, binary-search the first tail ≥ it and overwrite it (or append). The array's length IS the LIS length.
Other Approaches
Brute Generate all subsequences, check increasing. — O(2ⁿ · n).
Better Classic DP: dp[i] = 1 + max(dp[j]) for j < i and nums[j] < nums[i]. — O(n²).
Optimal Approach (Patience-sort / Binary Search)
  1. Maintain tails[] (initially empty).
  2. For each x: binary-search lower_bound(tails, x).
  3. If position equals length → append x (LIS extended).
  4. Else overwrite tails[pos] = x (a better, smaller tail at that length).
  5. LIS length = tails.size().
Visual
flowchart TD
  L["for each x in nums"] --> F["lower_bound(tails, x)"]
  F --> C{"pos == tails.length?"}
  C -- yes --> A["append x"]
  C -- no --> O["tails[pos] = x"]
  A --> L
  O --> L
  L --> R["return tails.length"]
Code (Java)
/**
 * Length of the Longest Increasing Subsequence in O(n log n).
 * Trick: 'minimumTailForLength[len]' stores the smallest possible tail value of any
 * increasing subsequence of length len+1 seen so far. Smaller tails leave more room
 * for future extensions, so we always overwrite with the smallest.
 */
int lengthOfLIS(int[] numbers){
  int[] minimumTailForLength = new int[numbers.length];
  int currentLisLength = 0;

  for(int currentValue : numbers){
    // find leftmost tail that is >= currentValue (i.e. lower bound)
    int left = 0, right = currentLisLength;
    while(left < right){
      int mid = (left + right) / 2;
      if(minimumTailForLength[mid] < currentValue) left = mid + 1;
      else                                          right = mid;
    }
    minimumTailForLength[left] = currentValue;            // either extend or improve a tail
    if(left == currentLisLength) currentLisLength++;
  }
  return currentLisLength;
}
Time O(n log n)
Space O(n)

26Coin Change II (number of ways) Hard

Section · DP Part-II
LeetCode Article
Problem

Count the number of distinct combinations of coins (unlimited supply) that sum to amount.

Mental Model
1D unbounded knapsack with the OUTER loop on coins. Iterating coins outermost guarantees each combination is counted in a canonical order (coin1's count, then coin2's count, …) — never as two different orderings of the same coins.
Other Approaches
Brute Recursion: at each step pick "take this coin" or "skip to next coin". — Exponential.
Better 2D DP dp[i][s] = ways using first i coin types for sum s. — O(n · amount) time, O(n · amount) space.
Optimal Approach (1D rolling DP)
  1. dp[0] = 1 (one way to make 0: pick nothing).
  2. For each coin c (outer): for each sum = c..amount → dp[sum] += dp[sum - c].
  3. Coin loop OUTER is critical — swapping gives permutations, not combinations.
  4. Return dp[amount].
Visual
flowchart TD
  I["dp[0] = 1"] --> O["for each coin c"]
  O --> S["for sum = c..amount"]
  S --> U["dp[sum] += dp[sum - c]"]
  U --> S
  S --> O
  O --> R["return dp[amount]"]
Code (Java)
/**
 * Counts distinct combinations of coins (unlimited supply) summing to targetAmount.
 * Trick: iterating coins in the OUTER loop fixes a canonical order (use coin0 some
 * number of times, then coin1, …). This prevents double-counting permutations.
 */
int change(int targetAmount, int[] coinDenominations){
  int[] waysToMakeSum = new int[targetAmount + 1];
  waysToMakeSum[0] = 1;                                    // exactly one way to make 0: empty set

  for(int coinValue : coinDenominations){                  // outer: lock the coin
    for(int currentSum = coinValue; currentSum <= targetAmount; currentSum++){
      // every way to make (currentSum - coinValue) becomes a new way to make currentSum
      waysToMakeSum[currentSum] += waysToMakeSum[currentSum - coinValue];
    }
  }
  return waysToMakeSum[targetAmount];
}
Time O(coins · amount)
Space O(amount)

27Trie — Advanced (Prefix & Word Counts) Hard

Section · Trie
Coding Ninjas Article
Problem

Support insert(word), countWordsEqualTo(word), countWordsStartingWith(prefix), erase(word) — each in O(L).

Mental Model
Augment each Trie node with two counters: endCount (words ending here) and passCount (words passing through here). Insert bumps both, erase decrements both, queries read them directly — no need to recount the subtree.
Other Approaches
Brute HashMap<String, Integer>. Prefix count needs scanning all keys. — O(N·L) per prefix query.
Better Plain trie + DFS over subtree to count words/prefix. — O(subtree-size).
Optimal Approach
  1. Each node has children[26], endCount, passCount.
  2. insert: walk/create children, passCount++ at every visited node; at the last node endCount++.
  3. countWordsEqualTo: walk; if any child missing → 0; else return last node's endCount.
  4. countWordsStartingWith: walk; if any child missing → 0; else return last node's passCount.
  5. erase: walk; passCount-- at every visited node; at last node endCount--.
Visual
flowchart TD
  R["root passCount = 3"] --> A["a pass=2"]
  A --> P["p pass=2"]
  P --> P2["p pass=2"]
  P2 --> L["l pass=1 end=1 (app+le)"]
  P2 --> X["pass=1 end=1 (app)"]
  R --> B["b pass=1"]
  B --> A2["a pass=1"]
  A2 --> T["t pass=1 end=1 (bat)"]
Code (Java)
/**
 * Trie with two per-node counters:
 *   endCount  = how many inserted words END exactly at this node
 *   passCount = how many inserted words PASS through this node (incl. ending here)
 * These let us answer "how many words equal X" and "how many start with prefix P"
 * in O(L) without traversing the subtree.
 */
class Trie {
  private static class Node {
    Node[] children = new Node[26];
    int    endCount  = 0;
    int    passCount = 0;
  }
  private final Node root = new Node();

  void insert(String word){
    Node currentNode = root;
    for(char letter : word.toCharArray()){
      int childIdx = letter - 'a';
      if(currentNode.children[childIdx] == null) currentNode.children[childIdx] = new Node();
      currentNode = currentNode.children[childIdx];
      currentNode.passCount++;                           // one more word passes through here
    }
    currentNode.endCount++;                              // word ends at this node
  }

  int countWordsEqualTo(String word){
    Node node = walkAlong(word);
    return node == null ? 0 : node.endCount;
  }

  int countWordsStartingWith(String prefix){
    Node node = walkAlong(prefix);
    return node == null ? 0 : node.passCount;
  }

  void erase(String word){
    // assumes the word was previously inserted
    Node currentNode = root;
    for(char letter : word.toCharArray()){
      int childIdx = letter - 'a';
      currentNode = currentNode.children[childIdx];
      currentNode.passCount--;
    }
    currentNode.endCount--;
  }

  private Node walkAlong(String text){
    Node currentNode = root;
    for(char letter : text.toCharArray()){
      int childIdx = letter - 'a';
      if(currentNode.children[childIdx] == null) return null;
      currentNode = currentNode.children[childIdx];
    }
    return currentNode;
  }
}
Time O(L) per op
Space O(Σ L)
