01Sort Colors (Dutch National Flag) Medium

Section · Arrays
LeetCode
Problem

Sort an array of 0, 1, 2 in place in a single pass with O(1) extra space.

Mental Model
Three pointers low, mid, high maintain four invariant zones: [0..low-1]=0, [low..mid-1]=1, [mid..high]=unknown, [high+1..n-1]=2. Each step shrinks the unknown.
Other Approaches
Brute Any O(n log n) sort.
Better Count 0s/1s/2s then overwrite — two passes.
Optimal Approach
  1. low = mid = 0; high = n-1.
  2. While mid <= high: if a[mid] == 0 swap(low,mid), low++, mid++. If 1 → mid++. If 2 → swap(mid,high), high-- (don't advance mid).
Visual
flowchart TD
  S["low=mid=0, high=n-1"] --> L{"mid <= high?"}
  L -- no --> O["done"]
  L -- yes --> A{"a[mid]?"}
  A -- "0" --> Z["swap(low,mid); low++; mid++"]
  A -- "1" --> M["mid++"]
  A -- "2" --> H["swap(mid,high); high--"]
  Z --> L
  M --> L
  H --> L
Code (Java)
/**
 * Dutch National Flag — sort {0,1,2} in one pass, O(1) extra space.
 * Trick: three pointers carve the array into 4 regions [0s | 1s | unknown | 2s].
 * Each comparison shrinks the "unknown" middle by one slot.
 */
void sortColors(int[] numbers){
  int lowBoundary = 0;                      // next slot expecting a 0
  int midScanner  = 0;                      // current unknown
  int highBoundary = numbers.length - 1;    // next slot expecting a 2

  while(midScanner <= highBoundary){
    switch(numbers[midScanner]){
      case 0:
        swap(numbers, lowBoundary, midScanner);
        lowBoundary++; midScanner++;        // settled 0 region grows
        break;
      case 1:
        midScanner++;                       // already in correct middle band
        break;
      default: // case 2
        swap(numbers, midScanner, highBoundary);
        highBoundary--;                     // do NOT advance mid: swapped-in value unknown
    }
  }
}
Time O(n)
Space O(1)

02Find Missing and Repeating Numbers Hard

Section · Arrays Part-II
GFG
Problem

Array of n numbers in [1, n] with one repeated and one missing. Find both in O(n) / O(1).

Mental Model
XOR all array values with 1..n. The XOR result equals missing XOR repeated. Use a set bit in this XOR to partition both arrays into two groups → one group contains missing, the other contains repeated.
Other Approaches
Brute Count each value with a HashMap → O(n)/O(n).
Math Use sum and sum-of-squares to derive two equations → O(n)/O(1) but risk overflow.
Optimal Approach (XOR)
  1. xorAll = XOR(array) XOR XOR(1..n). This equals repeated ^ missing.
  2. Pick any set bit b of xorAll.
  3. Split numbers into "bit set" and "bit unset" groups (do same for 1..n). XOR within each group gives the two values.
  4. Decide which is missing vs repeated by scanning the array once.
Visual
flowchart TD
  X["xorAll = repeated XOR missing"] --> B["pick a set bit b"]
  B --> P["partition into bit-set / bit-unset groups"]
  P --> R["each group XORs to one of {repeated, missing}"]
  R --> S["scan once → label which is which"]
Code (Java)
/**
 * Returns {repeated, missing} in O(n) time, O(1) space using XOR partitioning.
 * Trick: x XOR x = 0 cancels every value present in both array and 1..n. What remains
 * is (repeated XOR missing). A single set bit of that XOR distinguishes them.
 */
int[] findRepeatedAndMissing(int[] numbers){
  int xorOfRepeatedAndMissing = 0;
  for(int value : numbers) xorOfRepeatedAndMissing ^= value;
  for(int i = 1; i <= numbers.length; i++) xorOfRepeatedAndMissing ^= i;

  int distinguishingBit = xorOfRepeatedAndMissing & -xorOfRepeatedAndMissing;  // lowest set bit
  int bucketA = 0, bucketB = 0;
  for(int value : numbers){
    if((value & distinguishingBit) == 0) bucketA ^= value;
    else                                  bucketB ^= value;
  }
  for(int i = 1; i <= numbers.length; i++){
    if((i & distinguishingBit) == 0) bucketA ^= i;
    else                              bucketB ^= i;
  }
  // figure out which is repeated vs missing
  for(int value : numbers){
    if(value == bucketA) return new int[]{bucketA, bucketB};   // bucketA appears → repeated
  }
  return new int[]{bucketB, bucketA};
}
Time O(n)
Space O(1)

03Unique Paths Medium

Section · Arrays Part-III
LeetCode
Problem

On an m × n grid, count paths from top-left to bottom-right moving only right or down.

Mental Model
Pure combinatorics: every path takes exactly (m-1) downs and (n-1) rights — total m+n-2 moves. Choose where the downs go: C(m+n-2, m-1).
Other Approaches
Brute DFS counting paths → exponential.
DP dp[i][j] = dp[i-1][j] + dp[i][j-1] — O(m·n) time & space.
Optimal Approach
  1. Compute C(m+n-2, min(m-1, n-1)) incrementally.
  2. Multiplicative form: ans = (N · (N-1) · … (N-r+1)) / r! computed step-by-step.
Visual
flowchart LR
  P["total moves = m+n-2"] --> C["choose downs = m-1"]
  C --> R["answer = C(m+n-2, m-1)"]
Code (Java)
/**
 * Unique Paths in an m×n grid — pure combinatorics, O(min(m,n)) time, O(1) space.
 * Trick: every path has the same length m+n-2 with exactly m-1 downs. Counting the
 * orderings is C(m+n-2, m-1).
 */
int uniquePaths(int rowCount, int columnCount){
  int totalMoves = rowCount + columnCount - 2;
  int smallerHalf = Math.min(rowCount - 1, columnCount - 1);   // pick smaller for fewer multiplications

  long result = 1L;
  // result = totalMoves * (totalMoves-1) * ... / 1 * 2 * ... * smallerHalf
  for(int step = 1; step <= smallerHalf; step++){
    result = result * (totalMoves - smallerHalf + step) / step;
  }
  return (int) result;
}
Time O(min(m,n))
Space O(1)

04Count Subarrays with XOR K Medium

Section · Arrays Part-IV
GFG
Problem

Count subarrays whose XOR equals a target k.

Mental Model
Prefix XOR + HashMap. A subarray a[i+1..j] XORs to k iff prefixXor[j] XOR prefixXor[i] == k, i.e. prefixXor[i] == prefixXor[j] ^ k. Count earlier prefix XORs equal to currentXor ^ k.
Other Approaches
Brute All subarrays, XOR each → O(n²).
Optimal Approach
  1. HashMap count[prefixXor] = freq; seed with {0:1}.
  2. Walk array maintaining currentXor. Add count.get(currentXor ^ k) to answer.
  3. Increment count[currentXor].
Visual
flowchart TD
  S["map = {0:1}; xor = 0; ans = 0"] --> L["for each a[i]"]
  L --> X["xor ^= a[i]"]
  X --> A["ans += map.getOrDefault(xor ^ k, 0)"]
  A --> M["map[xor]++"]
  M --> L
Code (Java)
/**
 * Count subarrays whose XOR equals targetXor — prefix-XOR + HashMap.
 * Trick: if prefixXor up to j is P and we want subarray ending at j to XOR to k, we
 * need an earlier prefix-XOR equal to P XOR k. Counting frequencies makes this O(n).
 */
int countSubarraysWithXorEqualsK(int[] numbers, int targetXor){
  Map<Integer, Integer> prefixXorFrequency = new HashMap<>();
  prefixXorFrequency.put(0, 1);                       // empty prefix

  int runningXor = 0;
  int totalSubarrays = 0;
  for(int value : numbers){
    runningXor ^= value;
    int desiredEarlierPrefix = runningXor ^ targetXor;
    totalSubarrays += prefixXorFrequency.getOrDefault(desiredEarlierPrefix, 0);
    prefixXorFrequency.merge(runningXor, 1, Integer::sum);
  }
  return totalSubarrays;
}
Time O(n)
Space O(n)

05Add Two Numbers (Linked List) Medium

Section · Linked List
LeetCode
Problem

Two non-empty LLs represent integers in reverse order (one digit per node). Return their sum as a LL in the same form.

Mental Model
Classic grade-school addition: walk both lists, sum digit + digit + carry, store result mod 10, carry = sum / 10. Continue while any list or carry remains.
Other Approaches
Convert Build integers from LL, sum, build LL back — fails for big numbers.
Optimal Approach
  1. Dummy head; tail = dummy; carry = 0.
  2. While a != null || b != null || carry != 0: sum = (a?val:0) + (b?val:0) + carry; append sum % 10; carry = sum/10; advance.
  3. Return dummy.next.
Visual
flowchart LR
  A["a: 2 → 4 → 3"] --> S["sum each digit + carry"]
  B["b: 5 → 6 → 4"] --> S
  S --> R["result: 7 → 0 → 8  (342 + 465 = 807)"]
Code (Java)
/**
 * Add two numbers stored as reverse-order linked lists.
 * Trick: simulate column addition with a carry. The loop runs as long as ANY of the
 * three sources (l1, l2, carry) still has something to contribute.
 */
ListNode addTwoNumbers(ListNode l1, ListNode l2){
  ListNode dummyHead = new ListNode(0);
  ListNode tail = dummyHead;
  int carry = 0;

  while(l1 != null || l2 != null || carry != 0){
    int digitA = (l1 != null) ? l1.val : 0;
    int digitB = (l2 != null) ? l2.val : 0;
    int columnSum = digitA + digitB + carry;

    carry = columnSum / 10;
    tail.next = new ListNode(columnSum % 10);
    tail = tail.next;

    if(l1 != null) l1 = l1.next;
    if(l2 != null) l2 = l2.next;
  }
  return dummyHead.next;
}
Time O(max(n,m))
Space O(max(n,m))

06Flatten a Multi-Level Linked List Hard

Section · Linked List Part-II
GFG
Problem

Each node has next and bottom pointers; sub-lists via bottom are sorted. Flatten into one sorted bottom-linked list.

Mental Model
Recurse to the right first, then merge two sorted bottom-lists. Reduces to repeated 2-way merges of sorted lists.
Other Approaches
Brute Collect all node values, sort, rebuild → O(N log N).
Min-heap Merge K sorted bottom-lists with a heap.
Optimal Approach
  1. flatten(node): if node or node.next null → return node.
  2. node.next = flatten(node.next).
  3. Return mergeTwoSortedBottom(node, node.next).
Visual
flowchart TD
  F["flatten(node)"] --> R["flatten(node.next) first"]
  R --> M["merge two sorted bottom-lists"]
  M --> O["return head"]
Code (Java)
/**
 * Flatten a multi-level sorted linked list into one sorted bottom-list.
 * Trick: recurse to flatten the right tail FIRST, then merge the current bottom-list
 * with the already-flattened tail. Each node visited once during its merge.
 */
Node flatten(Node head){
  if(head == null || head.next == null) return head;

  Node flattenedRight = flatten(head.next);
  return mergeTwoSortedBottomLists(head, flattenedRight);
}

private Node mergeTwoSortedBottomLists(Node a, Node b){
  Node dummyHead = new Node(0);
  Node tail = dummyHead;

  while(a != null && b != null){
    if(a.val <= b.val){ tail.bottom = a; a = a.bottom; }
    else               { tail.bottom = b; b = b.bottom; }
    tail = tail.bottom;
  }
  tail.bottom = (a != null) ? a : b;
  dummyHead.next = null;                       // flattened result uses bottom-pointers only
  return dummyHead.bottom;
}
Time O(N) total nodes
Space O(depth) recursion

07Remove Duplicates from Sorted Array Easy

Section · LL & Arrays
LeetCode
Problem

Given a sorted array, remove duplicates in place. Return the new length; the first k elements should hold the unique values.

Mental Model
Two pointers: write marks the slot for the next unique value; read scans. Whenever a[read] != a[write-1] → copy and bump write.
Other Approaches
HashSet O(n) time but O(n) space and loses sortedness.
Optimal Approach
  1. Edge: empty → 0. Else write = 1.
  2. For read = 1..n-1: if a[read] != a[write-1] → a[write++] = a[read].
  3. Return write.
Visual
flowchart LR
  S["write=1, read=1"] --> L["for read = 1..n-1"]
  L --> C{"a[read] != a[write-1]?"}
  C -- yes --> W["a[write++] = a[read]"]
  C -- no --> N["skip"]
  W --> L
  N --> L
  L --> O["return write"]
Code (Java)
/**
 * In-place remove duplicates from a sorted array; returns the new length.
 * Trick: maintain a write pointer such that a[0..write-1] holds the unique prefix; the
 * read pointer scans the rest, copying only values different from the last unique one.
 */
int removeDuplicatesFromSortedArray(int[] numbers){
  if(numbers.length == 0) return 0;

  int writeIdx = 1;                                      // first slot is always unique
  for(int readIdx = 1; readIdx < numbers.length; readIdx++){
    if(numbers[readIdx] != numbers[writeIdx - 1]){
      numbers[writeIdx++] = numbers[readIdx];            // commit new unique value
    }
  }
  return writeIdx;
}
Time O(n)
Space O(1)

08Minimum Coins (Canonical Denominations) Medium

Section · Greedy
GFG
Problem

Given Indian-currency denominations (canonical), return the minimum count of coins/notes that sum to amount.

Mental Model
Always take the largest denomination ≤ remaining. Works because the canonical Indian / US denomination set is "greedy-safe" — every coin is ≥ 2× the next smaller (or otherwise non-canceling).
Other Approaches
DP Coin Change (general): dp[v] = 1 + min(dp[v - c]) — needed if denominations aren't greedy-safe.
Optimal Approach
  1. Scan denominations descending.
  2. Repeatedly take the largest that fits; amount -= denom; count++.
Visual
flowchart LR
  A["amount = 70"] --> D1["take 50 → amount 20, count 1"]
  D1 --> D2["take 20 → amount 0, count 2"]
  D2 --> O["answer = 2"]
Code (Java)
/**
 * Greedy minimum coin/note count for a canonical denomination set.
 * Trick: with greedy-safe denominations (e.g. {1,2,5,10,20,50,100,200,500,1000}),
 * always taking the largest that fits is globally optimal — no need for DP.
 */
int minimumCoinsCanonical(int amount){
  int[] denominations = {1000, 500, 200, 100, 50, 20, 10, 5, 2, 1};

  int remainingAmount = amount;
  int coinsUsed = 0;
  for(int denomination : denominations){
    while(remainingAmount >= denomination){
      remainingAmount -= denomination;
      coinsUsed++;
    }
  }
  return coinsUsed;
}
Time O(amount) worst
Space O(1)

09Palindrome Partitioning Medium

Section · Recursion
LeetCode
Problem

Partition string s such that every substring of the partition is a palindrome. Return all possible partitionings.

Mental Model
Backtrack on the cut position; only accept a substring if it is a palindrome. The recursion tree explores every valid first-cut, then recurses on the remaining suffix.
Other Approaches
Palindrome-table precompute Each isPalindromeRange call is O(n); precompute a 2D boolean table in O(n²) so every palindrome check becomes O(1). Build bottom-up by length: pal[i][j] = (s[i]==s[j]) && (j - i < 2 || pal[i+1][j-1]). 
boolean[][] pal = new boolean[n][n];
for(int len = 1; len <= n; len++){
  for(int i = 0; i + len - 1 < n; i++){
    int j = i + len - 1;
    pal[i][j] = (s.charAt(i) == s.charAt(j))
             && (len < 3 || pal[i + 1][j - 1]);
  }
}
// then inside backtrack: if(!pal[start][end]) continue;
Iterative DP variant dp[i] = list of all valid partitions of s[0..i-1]. Build left to right; for each i append every palindromic suffix s[j..i-1] to each partition in dp[j]. Useful when only the count or boolean (LC 139) is needed instead of all sentences.
Optimal Approach
  1. partition(start): if start == n → record current; return.
  2. For end = start..n-1: if s[start..end] palindrome → push, recurse(end+1), pop.
Visual
flowchart TD
  R["recurse(start)"] --> E{"start == n?"}
  E -- yes --> S["save current partition"]
  E -- no --> L["for end = start..n-1"]
  L --> P{"s[start..end] palindrome?"}
  P -- yes --> A["push → recurse(end+1) → pop"]
  P -- no --> L
Code (Java)
/**
 * All palindromic partitions of s via backtracking on the first cut.
 * Trick: at each level we try every possible prefix as the next piece; only palindromic
 * prefixes are taken, pruning entire subtrees of bad cuts.
 */
List<List<String>> palindromePartitioning(String text){
  List<List<String>> allPartitions = new ArrayList<>();
  backtrack(text, 0, new ArrayDeque<>(), allPartitions);
  return allPartitions;
}

private void backtrack(String text, int startIdx, Deque<String> currentPartition,
                       List<List<String>> allPartitions){
  if(startIdx == text.length()){
    allPartitions.add(new ArrayList<>(currentPartition));
    return;
  }
  for(int endIdx = startIdx; endIdx < text.length(); endIdx++){
    if(!isPalindromeRange(text, startIdx, endIdx)) continue;       // prune non-palindrome

    currentPartition.addLast(text.substring(startIdx, endIdx + 1));
    backtrack(text, endIdx + 1, currentPartition, allPartitions);
    currentPartition.removeLast();
  }
}

private boolean isPalindromeRange(String text, int lo, int hi){
  while(lo < hi) if(text.charAt(lo++) != text.charAt(hi--)) return false;
  return true;
}
Time O(2ⁿ·n)
Space O(n) depth

10Rat in a Maze Medium

Section · Recursion & Backtracking
GFG
Problem

An n × n grid (0=blocked, 1=open). Starting at (0,0), reach (n-1,n-1) moving D, L, R, U. Return all unique paths in lexicographic move order.

Mental Model
Backtracking DFS with a visited matrix. Try moves in lex order (D, L, R, U); record letter; on dead-end, undo. Lex order of moves yields lex-sorted output for free.
Other Approaches
BFS Shortest path but doesn't enumerate all.
Optimal Approach
  1. If (r,c) is destination → save current path string.
  2. Mark (r,c) visited. For each direction in {D, L, R, U}: if in-bounds & open & unvisited → append letter, recurse, pop letter.
  3. Unmark (r,c).
Visual
flowchart TD
  D["dfs(r,c)"] --> E{"(r,c) == goal?"}
  E -- yes --> S["save path"]
  E -- no --> M["mark visited"]
  M --> L["try D, L, R, U in order"]
  L --> R["recurse if valid"]
  R --> U["unmark on return"]
Code (Java)
/**
 * Enumerate all paths from (0,0) to (n-1,n-1) in lexicographic move order.
 * Trick: visiting directions in alphabetical order (D, L, R, U) makes the output list
 * naturally sorted without a final sort step.
 */
List<String> ratInAMaze(int[][] mazeGrid, int gridSize){
  List<String> allPaths = new ArrayList<>();
  if(mazeGrid[0][0] == 0) return allPaths;
  boolean[][] visited = new boolean[gridSize][gridSize];
  dfsExplore(0, 0, mazeGrid, gridSize, visited, new StringBuilder(), allPaths);
  return allPaths;
}

private static final int[] ROW_DELTA = { 1, 0, 0, -1};      // D, L, R, U
private static final int[] COL_DELTA = { 0,-1, 1,  0};
private static final char[] MOVE_LET = {'D','L','R','U'};

private void dfsExplore(int row, int col, int[][] mazeGrid, int gridSize,
                        boolean[][] visited, StringBuilder pathSoFar, List<String> allPaths){
  if(row == gridSize - 1 && col == gridSize - 1){
    allPaths.add(pathSoFar.toString());
    return;
  }
  visited[row][col] = true;
  for(int dir = 0; dir < 4; dir++){
    int nextRow = row + ROW_DELTA[dir], nextCol = col + COL_DELTA[dir];
    if(nextRow >= 0 && nextRow < gridSize && nextCol >= 0 && nextCol < gridSize
       && !visited[nextRow][nextCol] && mazeGrid[nextRow][nextCol] == 1){
      pathSoFar.append(MOVE_LET[dir]);
      dfsExplore(nextRow, nextCol, mazeGrid, gridSize, visited, pathSoFar, allPaths);
      pathSoFar.deleteCharAt(pathSoFar.length() - 1);
    }
  }
  visited[row][col] = false;
}
Time O(4^(n²)) worst
Space O(n²)

11Median of Two Sorted Arrays Hard

Section · Binary Search
LeetCode
Problem

Given two sorted arrays of sizes m and n, return the median of the combined sorted array in O(log(min(m,n))).

Mental Model
Binary-search the cut position in the smaller array. A valid partition has equal halves with max(leftA, leftB) ≤ min(rightA, rightB) — the median is then a function of those four values.
Other Approaches
Merge Merge sort-style walk → O(m+n).
Kth element BS Generalised version — also O(log(m+n)).
Optimal Approach
  1. Ensure A is the smaller array. BS cutA ∈ [0, m]; cutB = (m+n+1)/2 - cutA.
  2. Compute leftA, rightA, leftB, rightB with ±∞ sentinels.
  3. If leftA <= rightB && leftB <= rightA → valid partition. Else shrink window.
  4. Median = (combined length even) avg(max(left), min(right)) ; else max(left).
Visual
flowchart TD
  S["BS cutA in [0, m]"] --> P["cutB = (m+n+1)/2 - cutA"]
  P --> V{"leftA <= rightB AND leftB <= rightA?"}
  V -- yes --> M["median from {leftA,rightA,leftB,rightB}"]
  V -- "leftA > rightB" --> L["shrink cutA right"]
  V -- "leftB > rightA" --> R["grow cutA right"]
Code (Java)
/**
 * Median of two sorted arrays in O(log(min(m,n))).
 * Trick: binary-search a "cut" in the SMALLER array. A valid cut splits the combined
 * arrays into a left-half and right-half where every value on the left is ≤ every value
 * on the right. The median is then determined by four border values.
 */
double findMedianSortedArrays(int[] arrayA, int[] arrayB){
  if(arrayA.length > arrayB.length) return findMedianSortedArrays(arrayB, arrayA);

  int sizeA = arrayA.length, sizeB = arrayB.length;
  int half = (sizeA + sizeB + 1) / 2;       // target size of combined left side
  int lo = 0, hi = sizeA;

  while(lo <= hi){
    int cutA = (lo + hi) / 2;
    int cutB = half - cutA;

    int leftA  = (cutA == 0)      ? Integer.MIN_VALUE : arrayA[cutA - 1];
    int rightA = (cutA == sizeA)  ? Integer.MAX_VALUE : arrayA[cutA];
    int leftB  = (cutB == 0)      ? Integer.MIN_VALUE : arrayB[cutB - 1];
    int rightB = (cutB == sizeB)  ? Integer.MAX_VALUE : arrayB[cutB];

    if(leftA <= rightB && leftB <= rightA){
      if(((sizeA + sizeB) & 1) == 1) return Math.max(leftA, leftB);
      return (Math.max(leftA, leftB) + Math.min(rightA, rightB)) / 2.0;
    } else if(leftA > rightB){
      hi = cutA - 1;                         // need fewer from A
    } else {
      lo = cutA + 1;                         // need more from A
    }
  }
  throw new IllegalArgumentException("inputs not sorted");
}
Time O(log(min(m,n)))
Space O(1)

12K Most Frequent Elements Medium

Section · Heaps
LeetCode
Problem

Return the k most frequent values in an array.

Mental Model
HashMap of counts → bucket sort by frequency. Frequencies are bounded by n, so we can store lists per frequency and walk from high → low until we've collected k.
Other Approaches
Heap Min-heap of size k by frequency → O(n log k).
Sort Count, then sort entries by freq → O(n log n).
Optimal Approach (Bucket Sort)
  1. Count frequencies in a HashMap.
  2. Build buckets[freq] (length n+1), each a list of values with that frequency.
  3. Walk freq from n down to 1, collecting values until size == k.
Visual
flowchart TD
  C["count frequencies"] --> B["buckets[freq] = list of values"]
  B --> W["walk freq high → low, collect until k"]
Code (Java)
/**
 * Top-k frequent elements in O(n) average via bucket sort by frequency.
 * Trick: frequencies are bounded by n, so a length-(n+1) array of buckets indexed by
 * frequency lets us "sort by frequency" in O(n) — no heap, no real sort.
 */
int[] topKFrequent(int[] numbers, int k){
  Map<Integer, Integer> frequencyByValue = new HashMap<>();
  for(int value : numbers) frequencyByValue.merge(value, 1, Integer::sum);

  List<Integer>[] valuesAtFrequency = new List[numbers.length + 1];
  for(Map.Entry<Integer, Integer> entry : frequencyByValue.entrySet()){
    int freq = entry.getValue();
    if(valuesAtFrequency[freq] == null) valuesAtFrequency[freq] = new ArrayList<>();
    valuesAtFrequency[freq].add(entry.getKey());
  }

  int[] topK = new int[k];
  int writeIdx = 0;
  for(int freq = valuesAtFrequency.length - 1; freq >= 1 && writeIdx < k; freq--){
    if(valuesAtFrequency[freq] == null) continue;
    for(int value : valuesAtFrequency[freq]){
      topK[writeIdx++] = value;
      if(writeIdx == k) break;
    }
  }
  return topK;
}

Problem

Brute For every (i,j) compute min·width → O(n²).
Two passes Build NSL[] and NSR[] separately, then area pass — also O(n) but 3 passes.
Optimal Approach (single pass)
  1. Monotonic-increasing stack of indices. Walk bars (append sentinel 0).
  2. While top bar > current bar → pop; area = poppedHeight · (current − newTop − 1); update max.
  3. Push current index.

flowchart TD

/**
 * Largest rectangle in a histogram in O(n) using a single monotonic-increasing stack.
 * Trick: when we pop a bar of height h, the bars on either side that are STILL on the
 * stack (or beyond the right index i) are by definition shorter than h — they bound
 * the maximum rectangle that uses h as its limiting height.
 */
int largestRectangleArea(int[] barHeights){

  Deque<Integer> increasingIdxStack = new ArrayDeque<>();
  int bestArea = 0;

  for(int i = 0; i <= barCount; i++){

      int leftBoundIdx  = increasingIdxStack.isEmpty() ? -1 : increasingIdxStack.peek();
      int widthForPopped = i - leftBoundIdx - 1;
      bestArea = Math.max(bestArea, poppedHeight * widthForPopped);
    }
    increasingIdxStack.push(i);
  }

  2. Read optional + / -.

  4. Apply sign; return result.

Code (Java)
/**
 * Robust atoi with explicit overflow detection.
 * Trick: before doing accumulator = accumulator * 10 + digit, verify accumulator is
 * <= (INT_MAX - digit) / 10. If not, clamp immediately — no 64-bit arithmetic needed.
 */
int myAtoi(String text){
  int length = text.length();
  int cursor = 0;

  while(cursor < length && text.charAt(cursor) == ' ') cursor++;          // skip whitespace
  if(cursor == length) return 0;

  int signMultiplier = 1;
  if(text.charAt(cursor) == '+' || text.charAt(cursor) == '-'){
    signMultiplier = (text.charAt(cursor) == '-') ? -1 : 1;
    cursor++;
  }

  int accumulator = 0;
  while(cursor < length && Character.isDigit(text.charAt(cursor))){
    int digit = text.charAt(cursor) - '0';

    // overflow guard BEFORE multiplying
    if(accumulator > (Integer.MAX_VALUE - digit) / 10){
      return (signMultiplier == 1) ? Integer.MAX_VALUE : Integer.MIN_VALUE;
    }
    accumulator = accumulator * 10 + digit;
    cursor++;
  }
  return accumulator * signMultiplier;
}

  1. BFS by level. For each level: build a deque; if going left→right addLast(val), else addFirst(val).

flowchart TD
  L1["level 0: L→R"]
  L2["level 1: R→L"]
  L3["level 2: L→R"]

Code (Java)
/**
 * Zig-zag level order traversal via BFS + a per-level Deque.
 * Trick: a Deque lets us write to head or tail in O(1) per node — toggling the
 * direction flag between levels avoids an explicit reverse step.
 */
List<List<Integer>> zigzagLevelOrder(TreeNode root){
  List<List<Integer>> allLevels = new ArrayList<>();
  if(root == null) return allLevels;

  Deque<TreeNode> bfsQueue = new ArrayDeque<>();
  bfsQueue.offer(root);
  boolean leftToRight = true;

  while(!bfsQueue.isEmpty()){
    int levelSize = bfsQueue.size();
    Deque<Integer> levelDeque = new ArrayDeque<>();

    for(int i = 0; i < levelSize; i++){
      TreeNode current = bfsQueue.poll();
      if(leftToRight) levelDeque.addLast(current.val);
      else            levelDeque.addFirst(current.val);

      if(current.left  != null) bfsQueue.offer(current.left);
      if(current.right != null) bfsQueue.offer(current.right);
    }
    allLevels.add(new ArrayList<>(levelDeque));
    leftToRight = !leftToRight;
  }
  return allLevels;
}

/**

 * Trick: push each vertex onto the stack ONLY AFTER all its reachable children have
 * been processed. The reverse of post-order finish times is a valid topological order.
 */
int[] topologicalSortDFS(int vertexCount, List<List<Integer>> adjacencyList){

/**
 * Unbounded knapsack max value with 1D DP.
 * Trick: walking capacity LEFT → RIGHT lets dp[w - weight] already include this item,
 * so the same item can be picked multiple times.
 */
int unboundedKnapsack(int[] itemWeights, int[] itemValues, int capacity){
  int[] bestValueAtCapacity = new int[capacity + 1];

  for(int itemIdx = 0; itemIdx < itemWeights.length; itemIdx++){
    int weight = itemWeights[itemIdx];
    int value  = itemValues[itemIdx];
    for(int currentCapacity = weight; currentCapacity <= capacity; currentCapacity++){
      int valueIfPick = bestValueAtCapacity[currentCapacity - weight] + value;
      bestValueAtCapacity[currentCapacity] = Math.max(bestValueAtCapacity[currentCapacity], valueIfPick);
    }
  }
  return bestValueAtCapacity[capacity];
}

Space O(W)

27Maximum XOR of Two Numbers (Trie) Hard

Section · Trie
LeetCode
Problem

Return the maximum XOR value of any two numbers in an array.

Mental Model
Bit-trie of all 32-bit values. For each number, descend greedily preferring the OPPOSITE bit at each level — that's the bit choice maximising XOR. If the opposite branch exists, take it; otherwise fall back to the same bit.
Other Approaches
Brute All pairs → O(n²).
Optimal Approach
  1. Insert every number into a binary trie keyed by bits from MSB (bit 30 or 31) to LSB.
  2. For each number, walk the trie greedily preferring the opposite bit; OR a 1 into the answer when you can flip.
  3. Track the overall maximum.

  R --> I["bit 30 = 1"]

  Q --> M["if exists → max bit possible; else fall back"]
Code (Java)
/**
 * Maximum XOR of two numbers using a bit-trie in O(n · 32) time.
 * Trick: XOR is maximised bit-by-bit greedily. For each query bit b, the OPPOSITE bit
 * (1 - b) in some prior number produces a 1 in the XOR result — descend that branch
 * whenever it exists.
 */
int findMaximumXOR(int[] numbers){
  BitTrieNode root = new BitTrieNode();
  for(int value : numbers) insertBits(root, value);

  int bestXOR = 0;
  for(int value : numbers){
    bestXOR = Math.max(bestXOR, maxXorFor(root, value));
  }
  return bestXOR;
}

private static final int HIGHEST_BIT = 30;             // values fit in 31 bits (non-negative)

private void insertBits(BitTrieNode root, int value){
  BitTrieNode current = root;
  for(int bit = HIGHEST_BIT; bit >= 0; bit--){
    int bitValue = (value >> bit) & 1;
    if(current.children[bitValue] == null) current.children[bitValue] = new BitTrieNode();
    current = current.children[bitValue];
  }
}

private int maxXorFor(BitTrieNode root, int value){
  BitTrieNode current = root;
  int xorAccumulator = 0;
  for(int bit = HIGHEST_BIT; bit >= 0; bit--){
    int bitValue = (value >> bit) & 1;
    int oppositeBit = 1 - bitValue;
    if(current.children[oppositeBit] != null){
      xorAccumulator |= (1 << bit);                    // we can flip this bit → contributes 1
      current = current.children[oppositeBit];
    } else {
      current = current.children[bitValue];
    }
  }
  return xorAccumulator;
}

private static class BitTrieNode { BitTrieNode[] children = new BitTrieNode[2]; }

Space O(n · 32)
