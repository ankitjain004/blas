01Sort Colors (Dutch National Flag) Medium

Section · Arrays
LeetCode
Problem

Sort an array of 0, 1, 2 in place in a single pass with O(1) extra space.

Example

Input A: [2,0,2,1,1,0] · Output: [0,0,1,1,2,2]

  1. Dutch-flag: pointers lo=0, mid=0, hi=n-1. Walk mid:
  2. nums[mid]==0 → swap(lo, mid); lo++; mid++. ==1 → mid++. ==2 → swap(mid, hi); hi--.
  3. Trace: 2→20210 swap(2,0)→0..210; 0 ok; 2→2→0→1→swap; 1 ok; 1 ok; final [0,0,1,1,2,2].

Input B (edge — already sorted): [0,1,2] · Output: [0,1,2]

  1. 0 → lo,mid both advance; 1 → mid advances; 2 → swap with hi (itself), hi--; loop ends → unchanged.
Mental Model
Three pointers low, mid, high maintain four invariant zones: [0..low-1]=0, [low..mid-1]=1, [mid..high]=unknown, [high+1..n-1]=2. Each step shrinks the unknown.
Other Approaches
Brute Any O(n log n) sort.
Better Count 0s/1s/2s then overwrite — two passes.
Optimal Approach
  1. low = mid = 0; high = n-1.
  2. While mid <= high: if a[mid] == 0 swap(low,mid), low++, mid++. If 1 → mid++. If 2 → swap(mid,high), high-- (don't advance mid).

Code (Java)
/**
 * Dutch National Flag — sort {0,1,2} in one pass, O(1) extra space.
 * Trick: three pointers carve the array into 4 regions [0s | 1s | unknown | 2s].
 * Each comparison shrinks the "unknown" middle by one slot.
 */
void sortColors(int[] nums){
  int lo = 0;                      // next slot expecting a 0
  int mid = 0;                     // current unknown
  int hi = nums.length - 1;        // next slot expecting a 2

  while(mid <= hi){
    switch(nums[mid]){
      case 0:
        swap(nums, lo, mid);
        lo++; mid++;               // settled 0 region grows
        break;
      case 1:
        mid++;                     // already in correct middle band
        break;
      default: // case 2
        swap(nums, mid, hi);
        hi--;                      // do NOT advance mid: swapped-in value unknown
    }
  }
}
Time O(n)
Space O(1)

02Find Missing and Repeating Numbers Hard

Section · Arrays Part-II
GFG
Problem

Array of n numbers in [1, n] with one repeated and one missing. Find both in O(n) / O(1).

Example

Input A: [4,3,6,2,1,1] (n=6) · Output: repeating=1, missing=5

  1. XOR all nums with 1..n: every value cancels except x ⊕ y where x=missing, y=repeating.
  2. Pick any set bit in xor (lowest via x & -x); partition both lists by that bit.
  3. XOR each partition independently → yields x and y. Verify by counting one of them in nums to label which is missing vs repeating.
  4. Result → missing=5, repeating=1.

Input B (edge — small input): [2,2] (n=2) · Output: repeating=2, missing=1

  1. XOR(2,2,1,2) = 1⊕2 = 3 (binary 11). Lowest set bit = 1; partitions: bit-set group {1}, bit-clear group {2,2,2}; XOR → 1 and 2 → verify count(2)=2 → repeating=2.
Mental Model
XOR all array values with 1..n. The XOR result equals missing XOR repeated. Use a set bit in this XOR to partition both arrays into two groups → one group contains missing, the other contains repeated.
Other Approaches
Brute Count each value with a HashMap → O(n)/O(n).
Math Use sum and sum-of-squares to derive two equations → O(n)/O(1) but risk overflow.
Optimal Approach (XOR)
  1. xorAll = XOR(array) XOR XOR(1..n). This equals repeated ^ missing.
  2. Pick any set bit b of xorAll.
  3. Split numbers into "bit set" and "bit unset" groups (do same for 1..n). XOR within each group gives the two values.
  4. Decide which is missing vs repeated by scanning the array once.

  B --> P["partition into bit-set / bit-unset groups"]
  P --> R["each group XORs to one of {repeated, missing}"]

Code (Java)
/**
 * Returns {repeated, missing} in O(n) time, O(1) space using XOR partitioning.
 * Trick: x XOR x = 0 cancels every value present in both array and 1..n. What remains
 * is (repeated XOR missing). A single set bit of that XOR distinguishes them.
 */
int[] findRepeatedAndMissing(int[] nums){
  int xor = 0;
  for(int x : nums) xor ^= x;
  for(int i = 1; i <= nums.length; i++) xor ^= i;

  int bit = xor & -xor;  // lowest set bit
  int a = 0, b = 0;
  for(int x : nums){
    if((x & bit) == 0) a ^= x;
    else               b ^= x;
  }
  for(int i = 1; i <= nums.length; i++){
    if((i & bit) == 0) a ^= i;
    else               b ^= i;
  }
  // figure out which is repeated vs missing
  for(int x : nums){
    if(x == a) return new int[]{a, b};   // a appears → repeated
  }
  return new int[]{b, a};
}
Time O(n)
Space O(1)

03Unique Paths Medium

Section · Arrays Part-III
LeetCode
Problem

On an m × n grid, count paths from top-left to bottom-right moving only right or down.

Example

Input A: m=3, n=7 · Output: 28

  1. Closed form: C(m+n-2, m-1).
  2. C(8, 2) = 28.
  3. Or DP: dp[i][j] = dp[i-1][j] + dp[i][j-1]; row/col 0 all 1.

Input B (edge — single column): m=5, n=1 · Output: 1

  1. Only down moves available → unique path; combinatorial form: C(4, 0)=1.
Mental Model
Pure combinatorics: every path takes exactly (m-1) downs and (n-1) rights — total m+n-2 moves. Choose where the downs go: C(m+n-2, m-1).
Other Approaches
Brute DFS counting paths → exponential.
DP dp[i][j] = dp[i-1][j] + dp[i][j-1] — O(m·n) time & space.
Optimal Approach
  1. Compute C(m+n-2, min(m-1, n-1)) incrementally.
  2. Multiplicative form: ans = (N · (N-1) · … (N-r+1)) / r! computed step-by-step.
Visual
flowchart LR
  P["total moves = m+n-2"] --> C["choose downs = m-1"]
  C --> R["answer = C(m+n-2, m-1)"]
Animation

Code (Java)
/**
 * Unique Paths in an m×n grid — pure combinatorics, O(min(m,n)) time, O(1) space.
 * Trick: every path has the same length m+n-2 with exactly m-1 downs. Counting the
 * orderings is C(m+n-2, m-1).
 */
int uniquePaths(int m, int n){
  int total = m + n - 2;
  int r = Math.min(m - 1, n - 1);   // pick smaller for fewer multiplications

  long ans = 1L;
  // ans = total * (total-1) * ... / 1 * 2 * ... * r
  for(int i = 1; i <= r; i++){
    ans = ans * (total - r + i) / i;
  }
  return (int) ans;
}
Time O(min(m,n))
Space O(1)

04Count Subarrays with XOR K Medium

Section · Arrays Part-IV
GFG
Problem

Count subarrays whose XOR equals a target k.

Example

Input A: nums=[4,2,2,6,4], k=6 · Output: 4

  1. Prefix XOR P[i]; subarray (l,r] has xor P[r] ⊕ P[l].
  2. Need P[r] ⊕ P[l] = k → P[l] = P[r] ⊕ k.
  3. Hashmap of prefix counts seeded with {0:1}; at each i add map[P[i] ⊕ k].
  4. Subarrays summing to xor 6: [4,2], [4,2,2,6,4], [2,2,6,4], [6] → 4.

Input B (edge — k=0): nums=[5,5], k=0 · Output: 1

  1. Prefix xor: 5, 0. Map seeded {0:1}; i=0: lookup 5 → 0; add 5. i=1: P=0; lookup 0 → 1; add. Total 1 (subarray [5,5]).
Mental Model
Prefix XOR + HashMap. A subarray a[i+1..j] XORs to k iff prefixXor[j] XOR prefixXor[i] == k, i.e. prefixXor[i] == prefixXor[j] ^ k. Count earlier prefix XORs equal to currentXor ^ k.
Other Approaches
Brute All subarrays, XOR each → O(n²).
Optimal Approach
  1. HashMap count[prefixXor] = freq; seed with {0:1}.
  2. Walk array maintaining currentXor. Add count.get(currentXor ^ k) to answer.
  3. Increment count[currentXor].

  L --> X["xor ^= a[i]"]
  X --> A["ans += map.getOrDefault(xor ^ k, 0)"]
  A --> M["map[xor]++"]
  M --> L

Code (Java)
/**
 * Count subarrays whose XOR equals targetXor — prefix-XOR + HashMap.
 * Trick: if prefixXor up to j is P and we want subarray ending at j to XOR to k, we
 * need an earlier prefix-XOR equal to P XOR k. Counting frequencies makes this O(n).
 */
int countSubarraysWithXorEqualsK(int[] nums, int k){
  Map<Integer, Integer> freq = new HashMap<>();
  freq.put(0, 1);                       // empty prefix

  int xor = 0;
  int ans = 0;
  for(int x : nums){
    xor ^= x;
    int need = xor ^ k;
    ans += freq.getOrDefault(need, 0);
    freq.merge(xor, 1, Integer::sum);
  }
  return ans;
}

05Add Two Numbers (Linked List) Medium

Section · Linked List
LeetCode
Problem

Two non-empty LLs represent integers in reverse order (one digit per node). Return their sum as a LL in the same form.

Example

Input A: L1=2→4→3 (=342), L2=5→6→4 (=465) · Output: 7→0→8 (=807)

  1. Walk both lists with running carry. Digit = (a + b + carry) % 10; carry = /10.
  2. 2+5=7; 4+6=10 → 0, carry 1; 3+4+1=8 → 8 → list 7→0→8.

Input B (edge — different lengths and final carry): L1=9→9 (=99), L2=1 (=1) · Output: 0→0→1 (=100)

  1. 9+1=10→0 carry 1; 9+0+1=10→0 carry 1; both null but carry=1 → emit final 1.
Mental Model
Classic grade-school addition: walk both lists, sum digit + digit + carry, store result mod 10, carry = sum / 10. Continue while any list or carry remains.
Other Approaches
Convert Build integers from LL, sum, build LL back — fails for big numbers.
Optimal Approach
  1. Dummy head; tail = dummy; carry = 0.
  2. While a != null || b != null || carry != 0: sum = (a?val:0) + (b?val:0) + carry; append sum % 10; carry = sum/10; advance.
  3. Return dummy.next.
Visual
flowchart LR

  B["b: 5 → 6 → 4"] --> S
  S --> R["result: 7 → 0 → 8  (342 + 465 = 807)"]

Code (Java)
/**
 * Add two numbers stored as reverse-order linked lists.
 * Trick: simulate column addition with a carry. The loop runs as long as ANY of the
 * three sources (l1, l2, carry) still has something to contribute.
 */
ListNode addTwoNumbers(ListNode l1, ListNode l2){
  ListNode dummy = new ListNode(0);
  ListNode tail = dummy;
  int carry = 0;

  while(l1 != null || l2 != null || carry != 0){
    int a = (l1 != null) ? l1.val : 0;
    int b = (l2 != null) ? l2.val : 0;
    int sum = a + b + carry;

    carry = sum / 10;
    tail.next = new ListNode(sum % 10);
    tail = tail.next;

    if(l1 != null) l1 = l1.next;
    if(l2 != null) l2 = l2.next;
  }
  return dummy.next;
}
Time O(max(n,m))
Space O(max(n,m))

06Flatten a Multi-Level Linked List Hard

Section · Linked List Part-II
GFG
Problem

Each node has next and bottom pointers; sub-lists via bottom are sorted. Flatten into one sorted bottom-linked list.

Example

Input A: heads-along-next: 5→10→19→28; their bottom-lists are 5→7→8→30, 10→20, 19→22→50, 28→35→40→45 · Output: 5→7→8→10→19→20→22→28→30→35→40→45→50

  1. Recursive merge from the right: flatten(head.next) first, then merge two sorted bottom-lists.
  2. Merge by bottom pointers: 5→7→8→30 with 10→20 → 5→7→8→10→20→30 … keep merging.

Input B (edge — single sub-list): head=3→5→7 (next=null) · Output: 3→5→7

  2. node.next = flatten(node.next).
  3. Return mergeTwoSortedBottom(node, node.next).

Code (Java)
/**
 * Flatten a multi-level sorted linked list into one sorted bottom-list.
 * Trick: recurse to flatten the right tail FIRST, then merge the current bottom-list
 * with the already-flattened tail. Each node visited once during its merge.
 */
Node flatten(Node head){
  if(head == null || head.next == null) return head;

  Node right = flatten(head.next);
  return merge(head, right);
}

private Node merge(Node a, Node b){
  Node dummy = new Node(0);
  Node tail = dummy;

  while(a != null && b != null){
    if(a.val <= b.val){ tail.bottom = a; a = a.bottom; }
    else               { tail.bottom = b; b = b.bottom; }
    tail = tail.bottom;
  }
  tail.bottom = (a != null) ? a : b;
  dummy.next = null;                       // flattened result uses bottom-pointers only
  return dummy.bottom;
}
Time O(N) total nodes
Space O(depth) recursion

07Remove Duplicates from Sorted Array Easy

Section · LL & Arrays
LeetCode
Problem

Given a sorted array, remove duplicates in place. Return the new length; the first k elements should hold the unique values.

Example

Input A: [1,1,2,2,3] · Output: k=3, prefix=[1,2,3,...]

  1. Two pointers w=1, r=1..n-1; whenever nums[r] != nums[r-1] assign nums[w++] = nums[r].
  2. r=1 same; r=2 differs → w=2; r=3 same; r=4 differs → w=3.
  3. Return w=3; prefix [1,2,3].

Input B (edge — all duplicates): [5,5,5,5] · Output: k=1

Two pointers: write marks the slot for the next unique value; read scans. Whenever a[read] != a[write-1] → copy and bump write.
Other Approaches
HashSet O(n) time but O(n) space and loses sortedness.
Optimal Approach
  1. Edge: empty → 0. Else write = 1.
  2. For read = 1..n-1: if a[read] != a[write-1] → a[write++] = a[read].
  3. Return write.

/**
 * In-place remove duplicates from a sorted array; returns the new length.
 * Trick: maintain a write pointer such that a[0..write-1] holds the unique prefix; the
 * read pointer scans the rest, copying only values different from the last unique one.
 */
int removeDuplicatesFromSortedArray(int[] nums){
  if(nums.length == 0) return 0;

  int w = 1;                                  // first slot is always unique
  for(int r = 1; r < nums.length; r++){
    if(nums[r] != nums[w - 1]){
      nums[w++] = nums[r];                    // commit new unique value
    }
  }
  return w;
}

/**
 * Greedy minimum coin/note count for a canonical denomination set.
 * Trick: with greedy-safe denominations (e.g. {1,2,5,10,20,50,100,200,500,1000}),

}

09Palindrome Partitioning Medium

Section · Recursion
LeetCode
Problem

Partition string s such that every substring of the partition is a palindrome. Return all possible partitionings.

Example

Input A: s="aab" · Output: [["a","a","b"], ["aa","b"]]

  1. Backtrack over start index i; for each j ≥ i, if s[i..j] is a palindrome, push it and recurse from j+1.
  2. i=0: "a" palin → recurse from 1: "a" palin → recurse from 2: "b" → emit ["a","a","b"]; back to 0: "aa" palin → recurse from 2: "b" → emit ["aa","b"]. "aab" not palin.

Input B (edge — empty string): s="" · Output: [[]] (one empty partition)

  1. Base case i==n emits the running partition immediately, which is the empty list.
Mental Model
Backtrack on the cut position; only accept a substring if it is a palindrome. The recursion tree explores every valid first-cut, then recurses on the remaining suffix.
Other Approaches
Palindrome-table precompute Each isPal call is O(n); precompute a 2D boolean table in O(n²) so every palindrome check becomes O(1). Build bottom-up by length: pal[i][j] = (s[i]==s[j]) && (j - i < 2 || pal[i+1][j-1]).

for(int len = 1; len <= n; len++){
  for(int i = 0; i + len - 1 < n; i++){
    int j = i + len - 1;
    pal[i][j] = (s.charAt(i) == s.charAt(j))
             && (len < 3 || pal[i + 1][j - 1]);
  }
}
// then inside backtrack: if(!pal[start][end]) continue;

Optimal Approach
  1. partition(start): if start == n → record current; return.
  2. For end = start..n-1: if s[start..end] palindrome → push, recurse(end+1), pop.

/**
 * All palindromic partitions of s via backtracking on the first cut.
 * Trick: at each level we try every possible prefix as the next piece; only palindromic
 * prefixes are taken, pruning entire subtrees of bad cuts.
 */
List<List<String>> palindromePartitioning(String s){
  List<List<String>> res = new ArrayList<>();
  backtrack(s, 0, new ArrayDeque<>(), res);
  return res;
}

private void backtrack(String s, int i, Deque<String> path,
                       List<List<String>> res){
  if(i == s.length()){
    res.add(new ArrayList<>(path));
    return;
  }
  for(int j = i; j < s.length(); j++){
    if(!isPal(s, i, j)) continue;       // prune non-palindrome

    path.addLast(s.substring(i, j + 1));
    backtrack(s, j + 1, path, res);
    path.removeLast();
  }
}

private boolean isPal(String s, int lo, int hi){
  while(lo < hi) if(s.charAt(lo++) != s.charAt(hi--)) return false;
  return true;
}

Space O(n) depth

10Rat in a Maze Medium

Section · Recursion & Backtracking
GFG
Problem

An n × n grid (0=blocked, 1=open). Starting at (0,0), reach (n-1,n-1) moving D, L, R, U. Return all unique paths in lexicographic move order.

Example

Input A: 4×4 grid (rats-in-a-maze classic) · Output: e.g. ["DDRDRR", "DRDDRR"] sorted lexicographically

  1. DFS exploring directions in order D, L, R, U so emitted paths are sorted automatically.
  2. Mark visited[r][c]=true on entry; unmark on backtrack.
  3. On reaching (n-1,n-1), append the move-string to results.

private static final int[] DR = { 1, 0, 0, -1};      // D, L, R, U
private static final int[] DC = { 0,-1, 1,  0};
private static final char[] MOVES = {'D','L','R','U'};

private void dfs(int r, int c, int[][] grid, int n,
                 boolean[][] visited, StringBuilder path, List<String> res){
  if(r == n - 1 && c == n - 1){
    res.add(path.toString());
    return;

/**
 * Median of two sorted arrays in O(log(min(m,n))).
 * Trick: binary-search a "cut" in the SMALLER array. A valid cut splits the combined
 * arrays into a left-half and right-half where every value on the left is ≤ every value
 * on the right. The median is then determined by four border values.
 */
double findMedianSortedArrays(int[] a, int[] b){
  if(a.length > b.length) return findMedianSortedArrays(b, a);

  int m = a.length, n = b.length;
  int half = (m + n + 1) / 2;       // target size of combined left side
  int lo = 0, hi = m;

  while(lo <= hi){
    int cutA = (lo + hi) / 2;
    int cutB = half - cutA;

    int leftA  = (cutA == 0)  ? Integer.MIN_VALUE : a[cutA - 1];
    int rightA = (cutA == m)  ? Integer.MAX_VALUE : a[cutA];
    int leftB  = (cutB == 0)  ? Integer.MIN_VALUE : b[cutB - 1];
    int rightB = (cutB == n)  ? Integer.MAX_VALUE : b[cutB];

    if(leftA <= rightB && leftB <= rightA){

      return (Math.max(leftA, leftB) + Math.min(rightA, rightB)) / 2.0;

      hi = cutA - 1;                         // need fewer from A
    } else {
      lo = cutA + 1;                         // need more from A
    }
  }
  throw new IllegalArgumentException("inputs not sorted");
}

      last = f.pos;

      if(node.left  != null) q.offer(new Frame(node.left,  2 * p));
      if(node.right != null) q.offer(new Frame(node.right, 2 * p + 1));
    }
    best = Math.max(best, last - first + 1);
  }
  return best;
}
Time O(n)
Space O(n)
Detailed Walkthrough

Core idea. Imagine the tree laid out like a complete binary tree (with all the missing nodes as invisible placeholders). The "width" of a level is the distance between the leftmost real node and the rightmost real node — including the gaps.

            1                ← pos 0
          /   \
         2     3              ← pos 0, 1
        / \     \
       4   .     5            ← pos 0, _, _, 3   (positions 0..3 → width 4)

Why positions 2·p and 2·p+1? Standard heap indexing. If a node sits at position p, in a hypothetical complete tree its left child sits at 2·p and right child at 2·p+1. So width = rightmostPos − leftmostPos + 1.

Why re-anchor (normalise) per level? A height-60 tree would push positions past 2^60 and overflow long. Subtract the level's first position as an offset before computing children — widths are differences, so subtracting the same value from both endpoints leaves the difference unchanged:

(rightmost − offset) − (leftmost − offset) = rightmost − leftmost

The numbers stay bounded (root of each level becomes 0 again), the widths stay correct.

Two BFS invariants the code relies on:

  1. BFS preserves left-to-right order within a level → first frame polled is leftmost, last frame polled is rightmost. That's why lastPositionThisLevel = f.pos inside the loop simply ends up holding the rightmost real node's position when the loop finishes.
  2. Children are pushed with 2·normalisedPos (NOT 2·f.pos) — that's how the offset stays small across levels.

Trace on the tree above:

Level 0: queue=[(1,0)], first=0
  poll(1,0) normalised=0, last=0
  push left(2, 0), right(3, 1)
  width = 0 − 0 + 1 = 1  →  maxWidth = 1

Level 1: queue=[(2,0),(3,1)], first=0
  poll(2,0) normalised=0, last=0 → push left(4, 0)
  poll(3,1) normalised=1, last=1 → push right(5, 2·1+1 = 3)
  width = 1 − 0 + 1 = 2  →  maxWidth = 2

Level 2: queue=[(4,0),(5,3)], first=0
  poll(4,0) normalised=0, last=0 → no children
  poll(5,3) normalised=3, last=3 → no children
  width = 3 − 0 + 1 = 4  →  maxWidth = 4   ✓

Common "wait, why?" questions

  • Does pushing 2·normalisedPos still give correct widths? Yes — within a single level every node was offset by the same amount on the previous level, so children's positions are all shifted by the same factor. The difference last − first is unaffected.
  • Could I push raw f.pos instead? Works on small trees, but on skewed/deep inputs the values double every level and overflow long. Normalisation keeps the math bounded.
  • Why is Frame static final? static so it carries no hidden reference to the enclosing class; final so it can't be subclassed. Cleaner and typed compared to Object[]{node, (Integer)pos}.

/**
 * BST iterator that emits values in inorder, with amortised O(1) next() and O(h) space.
 * Trick: only push the LEFT SPINE we need next. After popping a node, push the left
 * spine of its right subtree — each node is pushed and popped exactly once over the
 * entire traversal.
 */
class BSTIterator {
  private final Deque<TreeNode> st = new ArrayDeque<>();

  BSTIterator(TreeNode root){ pushLeft(root); }

  int next(){
    TreeNode top = st.pop();
    pushLeft(top.right);
    return top.val;
  }

  boolean hasNext(){ return !st.isEmpty(); }

  private void pushLeft(TreeNode node){
    while(node != null){
      st.push(node);
      node = node.left;
    }
  }
}

  1. Kahn's BFS: compute indegrees; queue all with indeg=0; pop, append, decrement neighbors.

/**
 * Topological sort of a DAG via post-order DFS using a stack.
 * Trick: push each vertex onto the stack ONLY AFTER all its reachable children have
 * been processed. The reverse of post-order finish times is a valid topological order.
 */
int[] topologicalSortDFS(int n, List<List<Integer>> adj){
  boolean[] visited = new boolean[n];
  Deque<Integer> st = new ArrayDeque<>();

  for(int i = 0; i < n; i++){
    if(!visited[i]) dfs(i, adj, visited, st);
  }

  int[] order = new int[n];
  int idx = 0;
  while(!st.isEmpty()) order[idx++] = st.pop();
  return order;
}

private void dfs(int u, List<List<Integer>> adj,
                 boolean[] visited, Deque<Integer> st){
  visited[u] = true;
  for(int v : adj.get(u)){
    if(!visited[v]) dfs(v, adj, visited, st);
  }
  st.push(u);                  // push AFTER all descendants
}

/**
 * Prim's MST total weight via min-heap (lazy edge skipping).
 * Trick: heap entries can become stale (a vertex may be reached cheaper later), so we
 * lazily skip popped entries for vertices already in the MST instead of decrease-key.
 */
int primMSTWeight(int n, List<int[]>[] adj){
  PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> Integer.compare(a[0], b[0])); // {weight, vertex}
  boolean[] inMst = new boolean[n];

  pq.offer(new int[]{0, 0});
  int total = 0;

  while(!pq.isEmpty()){
    int[] cur = pq.poll();
    int w = cur[0];
    int u = cur[1];
    if(inMst[u]) continue;            // stale entry — skip

    inMst[u] = true;
    total += w;

    for(int[] e : adj[u]){
      int v = e[0], wv = e[1];
      if(!inMst[v]) pq.offer(new int[]{wv, v});
    }
  }
  return total;
}

/**
 * Maximum XOR of two numbers using a bit-trie in O(n · 32) time.
 * Trick: XOR is maximised bit-by-bit greedily. For each query bit b, the OPPOSITE bit
 * (1 - b) in some prior number produces a 1 in the XOR result — descend that branch
 * whenever it exists.
 */
int findMaximumXOR(int[] nums){
  Node root = new Node();
  for(int x : nums) insert(root, x);

  int best = 0;
  for(int x : nums){
    best = Math.max(best, query(root, x));
  }
  return best;
}

private static final int HIGH_BIT = 30;             // values fit in 31 bits (non-negative)

private void insert(Node root, int x){
  Node cur = root;
  for(int bit = HIGH_BIT; bit >= 0; bit--){
    int b = (x >> bit) & 1;
    if(cur.children[b] == null) cur.children[b] = new Node();
    cur = cur.children[b];
  }
}

private int query(Node root, int x){
  Node cur = root;
  int res = 0;
  for(int bit = HIGH_BIT; bit >= 0; bit--){
    int b = (x >> bit) & 1;
    int opp = 1 - b;
    if(cur.children[opp] != null){
      res |= (1 << bit);                    // we can flip this bit → contributes 1
      cur = cur.children[opp];
    } else {
      cur = cur.children[b];
    }
  }
  return res;
}

private static class Node { Node[] children = new Node[2]; }

Space O(n · 32)
